# 参考库 · 参考图片使用规范

本文件说明各领域 `参考图片/` 文件夹的使用方式和存放规范。

---

## 文件夹结构

每个领域文件夹（A-J + Z）均包含：

```
[领域文件夹]/
├── [领域]参考档案.txt      ← 文字偏好记录（原有）
└── 参考图片/               ← 新增图片参考层
    ├── 喜爱参考/            ← 正向视觉锚点
    └── 禁忌参考/            ← 反面案例，评估时用于对比
```

---

## 喜爱参考/

**存放什么**：
- 你真正喜欢、风格理想的参考图片
- M9 审美评估器批准的优秀 AI 生成结果
- 日常收集的 Moodboard 图片

**命名规范**：
- 格式：`[编号]-[简短描述]-[来源].jpg`
- 示例：`001-低饱和咖啡摄影-Pinterest.jpg`
- 示例：`002-极简Logo设计-Behance.png`
- 示例：`003-MJ生成-通过评估-20240315.png`

**如何使用**：
```
设计任务时：@[领域]参考档案.txt + 直接上传2-3张参考图到对话
M8 会读取参考图生成更精准的 Prompt

评估时：@M9_审美评估器.txt + @参考图 + @待评估图
M9 会进行真实视觉对比而不是靠文字描述脑补
```

---

## 禁忌参考/

**存放什么**：
- 你明确不喜欢的风格案例
- M9 审美评估器判定不通过的 AI 生成结果
- 有助于「负向界定」的反面教材

**命名规范**：
- 格式：`[编号]-[问题描述]-[来源].jpg`
- 示例：`001-过度饱和俗气配色-截图.jpg`
- 示例：`002-廉价3D质感-AI生成失败.png`

**如何使用**：
```
评估时：@M9_审美评估器.txt + @禁忌参考图（可选）
M9 会检测生成结果是否与禁忌案例相似
```

---

## Z_禁忌案例/禁忌参考/

与其他领域不同，Z 的 `禁忌参考/` 存放**跨领域通用的禁忌案例**，如：
- 廉价感设计
- 过度使用渐变/阴影
- 违反基本设计原则的案例

---

## 数量建议

每个领域的 `喜爱参考/` 维持 **5-20 张**为宜：
- 太少：AI 无法建立有效的视觉模式识别
- 太多：上传到对话时 Token 消耗大，建议每次只 @ 最相关的 3-5 张

---

## 图片格式建议

- 优先使用 `.jpg` / `.jpeg`（文件较小，加载快）
- `.png` 用于需要透明背景的案例
- 单张图片建议控制在 2MB 以内
- 可用压缩工具（如 TinyPNG）批量压缩后再存入
