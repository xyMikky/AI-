## P3-001 · 字号差异 > 颜色差异

**原则**：建立信息层级时，字号差异比颜色差异更有效。大字 = 主信息，小字 = 辅助信息，无需用颜色强调。折扣数字应远超周围文字 3-5 倍。超大字体（≥90% 画面宽度）= 字体即图形。

**交叉验证**：E1b2（多语言排版字号差异建立层级） + E2a1（大折扣数字 3-5 倍字号） + E1b1（超大字体 90%+ 画面宽度） + C（主次字号 10 倍对比） + D1a（信息层级三段递减）
**覆盖域**：E × C × D = 3 域 ✅

## P3-002 · Z 形阅读路径利用

**原则**：左上角是 Z 形阅读的起点（最先被看到），右下角是终点。即使主要内容是功能教育，促销信息应放在左上角确保被看到。CTA/行动指令放在用户视线最后落点。

**交叉验证**：E5b1#070（左上角促销信息位置策略） + E2b2（水平三区版式按视线流动排列） + E3b（Tagline 放最后作情感收尾） + D1e（Z 形阅读节点布局）
**覆盖域**：E × D = 2 域 ✅

## P3-003 · 三层信息递进结构

**原则**：复杂信息应分三层递进：① 可见结构（直觉感知）→ ② 量化数字（理性验证）→ ③ 可交互验证（深度信任）。在电商中体现为：主图吸引 → 功能说明 → 用户评价。层层递进，逐步说服。

**交叉验证**：E6b3b2（技术参数三层递进信任建立） + E3a（权威认证 + 社群证明 + 风险对冲三维叠加） + E6b2b2（情感插图 + 实物信任 + 数据说服三层） + E2b2（品牌感 + 产品信度 + 购买触发三维）
**覆盖域**：E = 1 域（但原则通用性极强，保留）

## P3-004 · 功能标注的 5 点上限

**原则**：功能标注点数量应严格控制——超过 5 个标注，图面会显得杂乱而非专业。3 个标注是最佳平衡点（如三圆形功能区域地图）。标注覆盖不同层级的用户顾虑（核心功能 + 工艺验证 + 舒适细节），而非重复同一卖点。

**交叉验证**：E5b1（标注点 ≤5 个，3 个最佳） + E5b1#081（同一页只用一种标注技法） + E5b3（四角分布标注均衡感） + E6b3a1（5-6 格图标矩阵 1 秒扫读） + F2c1（暗调 + 细线箭头标注专业感）
**覆盖域**：E × F = 2 域 ✅

## P3-005 · 文字即图形 / 图形即文字

**原则**：当文字放大到足够大，它就不再是「信息」而是「图形」；当图形足够精确，它就承担了「信息」功能。「字穿越人脸」「字母间隙即街道」「文字嵌入场景材质」——模糊字与图的边界是高级设计的标志。

**交叉验证**：E1a3b1（字穿越人脸打破传统版式） + E1a3c2b（文字解体几何形态人物穿行） + E1a2a（镂空大字 + 艺术背景融合） + E1a1c（超大汉字图形化） + C（书法超大字出血/字即版式）
**覆盖域**：E × C = 2 域 ✅

## P3-006 · 移动端优先的扫读设计

**原则**：功能要点列表使用圆点/编号短语格式而非段落文字，3-5 条，每条 5-8 字主功能 + 8-15 字副说明，优先服务移动端快速扫读。成分百分比 ≥48pt 字号确保缩略图可读。

**交叉验证**：E5b1（功能要点 3-5 条短语格式） + E5b1#082（成分百分比 48pt+ 移动端可读） + E6b2b1（三段功能竖排一屏完整显示） + E5b2（功能 badge 圆点短语集合）
**覆盖域**：E = 1 域（但移动端设计通用，保留）

## P3-007 · 促销信息与品牌感并存

**原则**：促销信息不等于放弃品牌感。「生活方式主图 + 浮层促销」结构保留了主图完整氛围。分割元素（Marquee 条/色块线）可同时承担视觉分区和信息重复传达。底部全宽横条确保促销在阅读结束时被看到。

**交叉验证**：E2a2（生活方式主图 + 浮层促销保留品牌感） + E2b1（分割元素兼顾分区和信息传达） + E2b2（底部全宽横条促销位置） + E5b1（功能说明图 + 左上角保留促销）
**覆盖域**：E = 1 域（电商核心原则，保留）

## P3-008 · 图标风格必须统一

**原则**：同一页面中的图标风格必须统一（全线描或全填充，不混用）。图标选择使用「感觉类」而非「功能类」（透气 = 波浪线，弹力 = 箭头延展），降低跨语言受众的理解门槛。

**交叉验证**：E5b1#082（图标风格统一不混用） + E5a（弹性/伸展功能用简单线条符号） + D1a（颜色语义编码一致性） + E6b3a1（图标 + 关键词 + 核心数字 1 秒扫读）
**覆盖域**：E × D = 2 域 ✅

## P3-009 · 双比例输出覆盖全场景

**原则**：内容必须同时输出至少两种比例（4:5 Feed + 9:16 Story / 1:1 + 3:4）以覆盖全版面投放场景。设计时以最窄比例为基础构图，宽比例通过扩展边缘实现。

**交叉验证**：E2b1（必须双比例输出 4:5 + 9:16） + E1a3a3（写实摄影版 + 扁平插画版双轨策略）
**覆盖域**：E = 1 域（但跨平台发布通用，保留）

## P3-010 · 信息密度与受众关系

**原则**：目标受众越专业/发烧友级，越欣赏克制和极简（少信息 = 高品位信号）；目标受众越大众/价格敏感，越需要信息丰富和促销明确。品牌形象广告与转化广告的比例 ≥ 20:80。

**交叉验证**：E1b3（发烧友受众欣赏克制排斥过度解释） + E6b2b1（极简信息架构 = 设计品牌认知） + E5b3（品牌广告占比 ≥20%） + F3a（克制的广告比明显的广告更有说服力）
**覆盖域**：E × F = 2 域 ✅

## P3-011 · 阶梯/台阶构图 = 进阶叙事

**原则**：产品从前到后依次升高的台阶构图，是传达「产品线进阶感/使用步骤顺序/升级」的通用视觉隐喻。多姿态陈列（站立/倾斜/横置）在台阶上制造动态节奏感。

**交叉验证**：F2b（阶梯台构图产品线进阶感） + F2b（白色几何台阶 + 蓝色镜面条进阶感） + E6b3b1（双手托举结构层级展示） + D1a（工作流步骤递进可视化）
**覆盖域**：F × E × D = 3 域 ✅

## P3-012 · 节点/连线系统 = 复杂关系可视化

**原则**：当需要可视化「多个实体之间的关系/流程/依赖」时，节点 + 连线 + 颜色编码是最直觉的信息架构。节点内嵌数据预览、条件标签贴连线、端口设计——这些 UI 模式在任何关系型信息中均可迁移。

**交叉验证**：D1a（节点画布三栏布局） + D1d（数据管线 ML 节点编辑器） + D1e（iPaaS 集成可视化图谱） + D1b3（协同可视化流程分析） + E（M12 设计关系可视化 D3 力导向图）
**覆盖域**：D × E = 2 域 ✅

## P3-013 · 时尚/服饰 Hero 图的文字极简原则（"文字是配角"）

**原则**：在时尚/服饰类的 Hero 图、生活方式 Banner、A+ 首屏中，**文字是产品摄影的配角，不是主角**。文字总面积应 ≤20%，模特/产品应 ≥80%。一旦文字字号过大形成视觉主导，就出现「让消费者看文字而非看产品」的致命错误——消费者购买产品，不购买标题。

**两种典型错误模式**：
1. 独立白色文字面板占据画面 40-50%（= 说明书化，失去时尚感和情感吸引力）
2. 文字叠印但字号过大（= 文字盖过产品视觉，焦点错位）

**正确字号比例参考**（以画面高度为基准）：
- Logo：小（≤10% 画面宽度）
- 标题：中（1-2 行，每行高约画面高度 5-8%）
- 卖点子弹：小（3 条以内，每行高约 2-3%）
- Slogan / CTA：极小（1 行，装饰性）

**交叉验证**：NEBILITY AC3383A A+ Hero 迭代实战（V2 文字过大 → V4 文字收缩进化路径） + P1-001（情感主体 70-80% / 功能验证 20-30% 七三法则） + P3-010（目标受众越专业越欣赏克制）
**覆盖域**：E（实战推导）= 1 域 ⚠️（待第二域验证升级）
