## 约束条件

### 必须遵守

1. **先分析品类再设计**：不同品类的详情页结构差异极大，必须先识别品类→查询 N 域参考→再开始设计
2. **市场差异不可混用**：中国市场和欧美市场的详情页设计逻辑完全不同，不得套用
3. **模块顺序有逻辑**：详情页模块排列遵循转化漏斗逻辑（吸引→说服→信任→行动），不可随意排列
4. **平台规范优先**：每个平台有图片尺寸/数量/格式的硬性规范，设计必须在规范内工作
5. **文案即设计**：详情页中文案不是"配合图片的注释"，而是转化的核心驱动力，必须与视觉同步策划
6. **参考库图片必须引用**：调用 AI 生图时遵守 `ref-image-must-include` 规范
7. **品类色彩语义不可违背**：每个品类有约定俗成的色彩编码（见「色彩语义编码表」），违背会导致品类认知错位

### 禁止行为

- 不了解品类就直接套模板
- 中国市场用欧美的极简克制风格（信息量不足无法转化）
- 欧美市场用中国的密集叠字风格（会被视为 spam）
- 详情页设计不考虑移动端适配
- 文案使用未经验证的虚假宣称（如"行业第一""100%有效"）
- 忽略跨品类通用原则直接进入品类细节
- 跳过痛点唤醒直接堆功能（用户不觉得自己有问题，就不会觉得需要你的方案）

## 中国市场基础叙事骨架（8模块链）

中国电商详情页的底层说服逻辑是一条 **8 模块叙事链**，所有品类的模块编排都应以此为骨架，按品类需求增减：

```
① HERO（产品海报）── 首屏视觉冲击，建立品类认知+品牌调性
      ↓ 吸引注意力
② PAIN-POINT（痛点唤醒）── 先戳痛点，唤起「我有这个问题」的共鸣
      ↓ 制造需求
③ USP-ICON（产品优势）── 回答「为什么选这个」（感性层：好处/利益）
      ↓ 三层说服递进 ──┐
④ BA-COMPARE / EFFECT（产品功效）── 回答「效果怎么样」（效果层：可见结果）  │
      ↓                │ 好处 → 效果 → 原理
⑤ TECH / CALLOUT（产品特性）── 回答「怎么做到的」（理性层：技术/材质/原理） ┘
      ↓ 建立信任
⑥ SOCIAL-PROOF / CERT（数据证明）── 回答「凭什么信你」（第三方证据/实验/认证）
      ↓ 辅助决策
⑦ SPEC-TABLE（产品信息）── 参数/规格/尺码，理性决策的最后一公里
      ↓ 品牌闭环
⑧ BRAND-STORY（品牌介绍）── 收尾信任，建立品牌认同与情感连接
```

### 三层说服递进（核心叙事逻辑）

模块③④⑤构成详情页说服力的核心三层递进，**顺序不可打乱**：

| 层级 | 模块 | 回答的问题 | 说服维度 | 示例 |
|------|------|----------|---------|------|
| L1 好处 | USP-ICON | 「买了能得到什么？」 | 感性/利益 | 「24小时持久保温」 |
| L2 效果 | BA-COMPARE | 「真的有用吗？」 | 证据/结果 | 保温前后温度对比曲线 |
| L3 原理 | TECH/CALLOUT | 「怎么做到的？」 | 理性/技术 | 三层真空断热结构剖面图 |

> **逻辑**：先用好处吸引兴趣（L1），再用效果证明真实（L2），最后用原理解释可信（L3）。如果直接上原理（跳过L1/L2），用户看不懂也不想看；如果只有好处没有原理（跳过L3），高价产品缺乏说服力。

### 骨架的灵活应用

- **性价比/大促品**：可精简为 ①②③⑥⑦（跳过深度技术解释，直接用数据证明+价格说服）
- **高端/品牌品**：完整8模块，⑤和⑧加重，技术叙事+品牌故事做溢价支撑
- **新品类/教育型**：②加重（先教育用户「你有这个问题」），增加 FAQ 模块
- **复购/耗材型**：②弱化（用户已知痛点），③⑥加重（强调本品差异化+评价）

### Step 3：文案策划

每个模块的文案同步策划，不作为事后补充：

**欧美详情页文案原则**：
- Benefit-first（先说好处，再说特性）
- Social proof integration（评论/数据嵌入文案）
- Scannable（可扫读：短句+bullet+数字）
- Brand voice consistent（品牌语气一致）

**中国详情页文案原则（从445条记录提炼）**：
- **痛点命名优于泛功效**：「告别毛巾细菌脸」>「有效清洁」；「边充边听」>「多功能」
- **数字量化必须具体**：「减少87%异味」>「有效去味」；「0.0001微米过滤」>「精密过滤」
- **造词策略**：创造品牌专属词汇（「零气感」「轻塑」「冰肌」），制造好奇与记忆点
- **英文标题升维**：高端线用英文大字标题/品牌句（COFFEE LOVER / THANKS FOR ATTENTION），与大字报竞品区分
- **恐惧+安心双路径**：先激活焦虑（细菌/辐射/甲醛），再给解决方案+认证+数据
- **身份标签**：给用户一个自我认同（「精致妈妈」「品质生活家」「运动达人」）
- **平台语境适配**：天猫偏品质感，拼多多偏实惠感，京东偏参数理性，抖音偏短平快冲击

**文案输出格式（每个模块）**：
```
模块 [N]：[模块类型]
• 主标题：[标题文字]
• 副标题：[补充文字]
• 正文/要点：[bullet points 或正文]
• CTA（如有）：[行动号召文字]
```

### Step 4：视觉设计与 AI 生图

根据结构规划，逐模块进行视觉设计：

1. **构图技法分配**：为每个模块指定 C1-C12 构图技法（见「构图进阶技法库」），确保 7 个模块使用 ≥ 4 种不同技法
2. **构图自检门**：逐模块检查是否触犯构图反模式清单（50/50 分割、文字区 ≥ 视觉区、模特塞角落、全模块同构图）
3. **Prompt 构造**：先写视觉主体（大、具体、有力），再写文字（小、轻、浮动），开头强制附加 COMPOSITION RULE 约束语句
4. **批量并发**：遵守 `batch-image-concurrent` 规范，多张图并发生成
5. **参考库引用**：遵守 `ref-image-must-include` 规范
6. **迭代判定**：复杂模块（如 HERO+文字+产品合成）走迭代生图协议
7. **字体渲染注入**：当 C 域有可用字体文件时，执行 Step 4.5

> **⚠️ 构图自检门（Step 4.0，强制）**：Prompt 构造完毕后、调用生图 API 前，必须执行：
> - 该模块的产品/模特描述是否在 Prompt 中**先于**文字描述？
> - 是否写了 `COMPOSITION RULE:` 约束语句？
> - Prompt 中是否存在 "left half" / "right half" / "two-zone split" / "50%" 等二分割关键词？→ 存在即违规，必须改写
> - 该模块的构图技法编号是否与前一个模块不同？（避免连续使用同一构图）

**Step 4.5：详情页品牌资产与字体参考图策略**

> **核心规则：Logo 仅首图，字体每模块单独渲染。**
> 详情页与单张海报/Banner 的品牌资产策略完全不同。单张图需要把 Logo+色卡+所有文字合成为一张品牌资产图；详情页有 7+ 个模块，每个模块的文字内容不同，必须为**每个模块单独生成字体参考图**。

#### 详情页 Logo 使用规则

| 模块 | 是否需要 Logo | 品牌资产合成图 | 字体参考图 |
|------|-------------|-------------|----------|
| **M1 HERO 首图** | ✅ 需要（如品牌有要求） | ✅ 生成含 Logo 的品牌资产合成图 | ✅ 该模块的文字已含在品牌资产合成图中 |
| **M2-M7 其他模块** | ❌ 不需要 Logo | ❌ 不生成品牌资产合成图 | ✅ 每个模块单独生成字体参考图 |

> **⚠️ 这条规则覆盖主控中心 ⑤.5 的默认触发逻辑。** 在详情页任务中，⑤.5 品牌资产合成图仅对 HERO 首图生成一次；其他模块只生成字体参考图，不走品牌资产合成图流程。

#### 每模块字体参考图生成流程

**触发条件**：模块包含文字元素（标题/副标题/标签/功能文字等），且满足以下任一：
- 品牌档案中指定了品牌字体文件路径
- C 域排版参考档案中有匹配的字体记录
- 用户明确要求使用特定字体

**Step 4.5.1 — 确定字体来源（优先级）**
```
1. 品牌档案中的品牌字体 → 最高优先（品牌规范/[品牌名]/原始素材/[字体文件]）
2. C 域排版参考档案中匹配场景的字体 → 次优先
3. 均无 → 在 Prompt 中用文字描述字体风格（如 "clean bold sans-serif"），不生成字体参考图
```

**Step 4.5.2 — 为每个模块单独渲染字体参考图**

为 M2-M7 中**每一个含文字的模块**各生成一张字体参考图，仅包含该模块自己的文字内容：

```
# 示例：为 M2 功能模块渲染字体参考图
python ".cursor/skills/brand-asset-sheet/scripts/generate_brand_asset_sheet.py" \
  --brand "[品牌名]" \
  --colors "[品牌主色]:[色名]" \
  --texts-file "生成图片输出/brand_asset_sheets/[品牌]-M2-texts.json" \
  --output "生成图片输出/brand_asset_sheets/[品牌]-M2-font-ref.png"

# 注意：不传 --logo 参数 → 合成图中不含 Logo
```

每个模块的 `texts.json` 仅包含该模块自己的文字：
```json
// M2 Feature 模块的 texts.json 示例
[
  {
    "font": "品牌规范/NEBILITY/原始素材/Archivo-Bold.ttf",
    "text": "STEEL BONE SUPPORT",
    "size": 70,
    "color": "#1a1a1a",
    "label": "Feature 1 Title"
  },
  {
    "font": "品牌规范/NEBILITY/原始素材/Archivo-Bold.ttf",
    "text": "HIGH-RISE WAIST CONTROL",
    "size": 70,
    "color": "#1a1a1a",
    "label": "Feature 2 Title"
  }
]
```

**Step 4.5.3 — --images 编号策略（详情页专用）**

```
HERO 首图（含 Logo）：
  image 1 → 产品图（保留产品主体）
  image 2 → 人物库素材（如需）
  image 3 → 品牌资产合成图（含 Logo + 色卡 + Hero 文字）
  image 4 → 参考库风格参考图（如需）

其他模块（M2-M7，不含 Logo）：
  image 1 → 产品图（保留产品主体）
  image 2 → 该模块的字体参考图（仅含本模块文字）
  image 3 → 人物库素材 / 参考库风格参考图（如需）
  无字体参考图时 → 不占位，顺序前移
```

**Step 4.5.4 — Prompt 中引用字体参考图的标准句式**

```
# HERO 首图（引用品牌资产合成图，含 Logo）
Reproduce the NEBILITY logo exactly as shown in image 3.
Use the exact typography style from image 3 for the headline text.

# 其他模块（引用字体参考图，无 Logo）
Use the exact typography style shown in image 2 — replicate the precise 
font design, letter shapes, weight, and spacing for all text elements 
in this image. The text content is: "[该模块文字内容]". 
Do NOT use a different font. Do NOT add any logo or brand mark.
```

#### 详情页字体参考图批量预生成

在 Step 4 开始前，可一次性为所有模块批量生成字体参考图，提高效率：

```
执行顺序：
1. Step 3 文案策划完成 → 获得每个模块的完整文字内容
2. 批量生成：
   · M1 HERO → brand_asset_sheet.py（含 --logo，生成品牌资产合成图）
   · M2 → brand_asset_sheet.py（无 --logo，仅字体参考图）
   · M3 → brand_asset_sheet.py（无 --logo，仅字体参考图）
   · ...
   · M7 → brand_asset_sheet.py（无 --logo，仅字体参考图）
3. 全部字体参考图就位后 → 进入 Step 4 构图+生图
```

> **效率提示**：字体参考图的生成是纯本地 Python 操作（无 API 调用），速度极快（每张 < 1 秒），可串行执行无需并发。

**两种字体修正模式**：

| 模式 | 适用场景 | 流程 |
|------|---------|------|
| A 首次注入 | 生图时就包含正确字体 | 字体参考图 → 与产品图一起传入 → AI 同时处理 |
| B 二次替换 | 第一轮文字不满意时 | AI 先生图 → 渲染正确文字 → 第二轮替换文字区域 |

模式 B 的 Prompt 模板：
```
Keep everything from image 1 exactly the same — layout, product, colors, background.
Replace only the text area with the text shown in image 2.
Match the text position from image 1 but use the exact font style from image 2.
Integrate the new text naturally with the existing design.
Do NOT add any logo.
```

**平台尺寸规范速查**：

## 跨品类通用设计原则（从445条记录提炼的10条高频规律）

以下原则适用于**所有品类**的中国市场详情页，设计时必须逐条检查是否已覆盖：

### TP1 · 不可见效果必须可视化
结构剖面/爆炸分层、气流光束、温度渐变、声波圈、滤芯阶梯、铜芯截面——消费者看不见的品质，必须通过视觉手段让它「看得见」。
- **适用品类**：家电（滤芯/电机/气流）、护肤（渗透/成分）、音频（降噪/单元）、充电（芯片/PCB）、门锁（锁芯）

### TP2 · 关键数字震撼首屏
将最有冲击力的数字放大到首屏或核心模块：瓦数、mAh、dB、过滤精度、保温时长、续航天数。用对比或参照物强化感知（「一杯水的重量」「绕地球X圈」）。
- **适用品类**：全品类（每个品类都有自己的核心数字）

### TP3 · 顾虑前置消解
先解决用户「怕什么」（进水/翘边/多口变慢/甲醛/安装难/不兼容），再堆功能。安全/安装/兼容性/售后这些顾虑在详情页上半部分就要消解。
- **适用品类**：安装类家电、安防、母婴（安全焦虑）、手机配件（兼容焦虑）、食品（安全焦虑）

### TP4 · 家庭情感锚点
把功能翻译成「为家人」：儿童/婴儿/老人/宠物场景。从「这个产品能做什么」变成「这个产品让家人怎么了」。
- **适用品类**：家电、净水、安防、母婴、食品、床品、餐厨

### TP5 · 证据可视化（BA/实验/对比）
Before/After对比、甲醛读数实验、染色测试、皱褶对比、收纳前后——用可验证的证据替代主观宣称。
- **适用品类**：护肤（BA对比）、清洁（洁净度对比）、收纳（整理前后）、空净（数值对比）

### TP6 · 全生命周期与四季价值
变频省电费、冷暖两用摊薄季节成本、滤芯年费用、一台顶多台——把单次购买成本分摊到长期价值。
- **适用品类**：家电（能效/耗材）、净水（滤芯）、充电（多口省买多个）、厨电（一机多用）

### TP7 · 赛道升维（替代专业服务/品类升级）
替代美容院、替代专业理发师、医美级/皮肤科级、AI智能——把产品从「工具」提升到「专业服务替代品」，拉高价格参照锚点。
- **适用品类**：美容仪、个护、清洁家电、智能硬件

### TP8 · 教程降门槛
彩妆分步教程、安装1-2-3步骤、贴膜工具包指引、男性护肤≤3步——用保姆级教程消除「我不会用」的决策障碍。
- **适用品类**：彩妆、手机配件（贴膜）、安装类家电、扩展坞（连接方式）

### TP9 · 场景命名与垂直定位
「洗澡音箱」「边充边听」「午休枕」「桌面KTV」——用场景名替代品类名，降低比价、提高「就是我需要的」命中感。
- **适用品类**：音频、充电配件、日用家居、小家电

### TP10 · 系列化与套装策略
全家福→逐件说明→折扣算账；多色SKU驱动选择而非离开；OVERVIEW/全系矩阵显品牌厚度促连带购。
- **适用品类**：护肤套装、餐厨套装、耳机多色、充电全家桶

## 首屏策略分类法（中国市场）

根据445条记录总结，中国详情页首屏可归为以下8种策略类型：

### S1 · 数字冲击型
核心数字占满首屏（如「0.0001微米」「200W」「72小时保温」），产品居中或靠侧。
- 适用：充电、净水、保温杯、音频续航

### S2 · 暗调科技型
深色背景+产品英雄图+品牌/功能一句话，传递「旗舰/Pro/智能」。
- 适用：扫地机、美容仪、智能门锁、高端家电

### S3 · 场景代入型
产品在真实使用场景中（厨房/客厅/户外），强调「放在你家是这个效果」。
- 适用：家居融合家电、台灯、床品、餐厨、风扇

### S4 · 食欲/感官型
食材满铺或成品菜大图，先馋后卖机器。
- 适用：烤箱、破壁机、冰箱、食品、咖啡机

### S5 · 痛点恐惧型
先展示问题/痛点（水质污染/细菌/甲醛/脱发），再给出解决方案。
- 适用：净水、空净、洗护、除螨、安防

### S6 · 颜值/IP型
产品外观或IP联名设计占满首屏，主打「好看到想拥有」。
- 适用：耳机多色、音箱设计款、儿童产品、杯具、小家电

### S7 · 礼品/仪式型
礼盒包装+深色/金色调，传递「值得送人」。
- 适用：护肤套装、茶具、高端个护、保温杯礼盒

### S8 · 效果见证型
BA对比或实验结果直接上首屏（穿前穿后/涂前涂后/清洁前后）。
- 适用：护肤、塑身内衣、清洁家电、收纳

## 品类差异化指南（中国市场·12品类）

### 1. 护肤（精华/乳液/面膜/套装）

**N域参考**：N2b1a / N2b1b / N2b1c（共31条）
**核心模块**：HERO → TECH（成分/机制）→ BA-COMPARE → LIFESTYLE → SOCIAL-PROOF → SIZE-GUIDE（套装清单）
**首屏策略**：S7礼品型（套装）/ S2暗调科技型（功效精华）/ S6颜值型（彩妆）
**色系**：绿系（植萃）/ 蓝白（医学）/ 深绿米白（高端）/ 粉珊瑚（彩妆少女）
**关键策略**：
- 成分可视化是信任核心（渗透图解/植物溯源/分层结构）
- 套装走「全家福→逐件说明→折扣算账」三段式
- 高端线「单屏单卖点+大留白+包装主导」；大众线「成分矩阵+四效合一+密集信息」
- 彩妆类必备分步教程+色号矩阵

**中国独特策略**：造词（「零气感」「冰肌」）、英文品牌大字抬专柜感、国产彩妆用成分表+数据补品牌弱势

### 2. 美容仪器

**N域参考**：N2b2（18条）
**核心模块**：HERO → TECH（原理图解）→ BA-COMPARE/时间线 → 档位附件 → LIFESTYLE（便携/全家）→ SPEC
**首屏策略**：S2暗调科技型
**色系**：深蓝/冷灰/科幻绿（科技感）；白金/粉金（礼品化）
**关键策略**：
- 体验先于参数（冰纹/皮肤微距唤起体感）
- 核心数字放大（频率/dB/闪光次数/效果时间线）
- 溢价路径：礼品化（金白礼盒）、AI智能首屏、数据曲线
- TP7赛道升维：替代美容院/医美级

### 3. 个护美发（吹风机/卷发/电动牙刷/剃须/洗护）

**N域参考**：N2b3a / N2b3b / N2b3c（共24条）
**核心模块**：HERO → USP-ICON → CALLOUT → BA-COMPARE → LIFESTYLE → SPEC
**首屏策略**：S2暗调科技型（高端）/ S6颜值型（年轻线多色）/ S8效果型（美发BA）
**色系**：红黑/黑金（精奢男士）/ 蓝白（大众）/ 深绿（礼品）/ 马卡龙（年轻女性）
**关键策略**：
- 美发类：折叠形态前后对比、速干/静音数字+参照物、同模特发型BA
- 口腔类：磁悬浮技术+IPX7水下图+续航换算成天数
- 洗护类：天然道具摄影+泡沫状态、棉类剖面层次图

### 4. 大家电（洗衣机/冰箱/空调/洗碗机）

**N域参考**：N2c1a / N2c1b（共23条）
**核心模块**：HERO → USP-ICON → EXPLODE/TECH → LIFESTYLE → SPEC-TABLE → GUARANTEE
**首屏策略**：S3场景代入型（家居融合）/ S2暗调科技型（旗舰）

## 欧美详情页视觉设计体系（Visual Design System for Western Markets）

> **V2.2 核心升级**：解决三大系统性缺陷——①文字区域过大导致产品/模特沦为配角；②构图千篇一律（只有左右或上下二分割）缺乏美感；③Hero 首屏模特/产品太小，无法形成视觉冲击。

---

### 第一法则：视觉主角法则（Visual Hero Principle）

**产品/模特是画面的绝对主角，文字永远是配角。**

这条法则的优先级高于一切排版规范。参照 SKIMS / Popilush / CRZ YOGA / Gymshark 等头部 DTC 品牌，其详情页的共同特征是：**产品和模特占据画面的绝对视觉主导权**，文字仅作为信息补充，绝不与视觉主体争夺注意力。

#### 视觉权重分配铁律（强制执行）

| 模块类型 | 产品/模特最低视觉占比 | 文字区域最高占比 | 说明 |
|---------|------------------|---------------|------|
| **HERO 首屏** | **≥ 70%** | **≤ 15%** | 模特/产品必须是压倒性的视觉主体，品牌名+一句话即可 |
| FEATURE / USP | ≥ 55% | ≤ 20% | 产品是主角，功能标注是辅助 |
| CALLOUT 标注 | ≥ 65% | ≤ 15% | 产品大图+小标注线，标注文字极小 |
| TECH 面料/科技 | ≥ 50% | ≤ 25% | 面料纹理/剖面图为视觉核心 |
| COLOR 多色展示 | ≥ 75% | ≤ 10% | 产品/模特占绝对主导，仅标注色名 |
| LIFESTYLE 场景 | ≥ 85% | ≤ 8% | 几乎纯视觉，最多一句品牌标语 |
| BA-COMPARE | ≥ 80% | ≤ 10% | 对比照说话，仅标注 Before/After |
| SIZE-GUIDE | ≥ 40% | ≤ 35% | 唯一允许信息密集的模块 |

> **⚠️ 关键认知纠偏**：「产品/模特视觉占比」不是指物理面积占比，而是**视觉权重**。一个占画面 60% 面积的模特，如果旁边有同等视觉强度的大号粗体标题，模特的视觉权重会被稀释到 40% 以下。**文字越大越粗，对视觉权重的侵占越严重。**

#### Hero 首屏的特殊强制规范

Hero 是整个详情页的「门面」，直接决定用户是否继续浏览。**Hero 模块执行最严格的视觉主角规范**：

| 要素 | 强制规范 |
|------|---------|
| 模特/产品尺寸 | 模特必须至少占画面高度的 **80%**（全身可见时）或面积的 **55%**（含构图留白） |
| 模特位置 | 模特是画面的视觉锚点——居中、略偏中、或占据画面主体区域。**禁止**把模特缩到一侧角落 |
| 文字角色 | 品牌名（小 Logo）+ 一行利益标题 + 可选的极小标签，总文字不超过画面的 15% |
| 文字位置 | 文字**叠加**在模特/产品的留白区域或边缘呼吸空间，不另开独立文字区 |
| 禁止行为 | ❌ 左右五五分（模特左/文字右）；❌ 文字区宽度 ≥ 模特区宽度；❌ 标题字号 ≥ 模特头部高度 |

#### 文字的正确角色定位

```
文字不是"内容区"——文字是浮在视觉之上的"标签"

❌ 错误心智模型：画面 = 产品区 + 文字区（两个平等的分区）
✅ 正确心智模型：画面 = 产品/模特大图（铺满） + 文字漂浮在上层（小、轻、精致）
```

---

### 第二法则：构图进阶技法库（Advanced Composition Techniques）

**简单的左右二分割 / 上下二分割是构图的最低级形式。** 优秀的详情页每个模块都应有不同的构图策略，制造视觉节奏变化。

#### ❌ 构图反模式清单（生图前必须自检）

| 反模式 | 描述 | 为什么差 |
|--------|------|---------|
| **50/50 二分割** | 画面正中竖线或横线切成两个等面积区域 | 无主次、无张力、像 PPT 幻灯片 |
| **文字区 ≥ 视觉区** | 文字占据的面积 ≥ 产品/模特面积 | 产品沦为配图，喧宾夺主 |
| **模特塞角落** | 模特缩到画面的 30% 角落，大面积留给文字 | Hero 失去冲击力，像证件照缩略图 |
| **所有模块同构图** | 7 个模块全用相同的左右分割或上下分割 | 单调乏味，用户视觉疲劳 |
| **文字区纯色色块** | 独立的大面积纯色色块专门放文字 | 像报纸排版，缺乏设计感 |
| **对称平均分格** | 均等三分格或四分格 | 像九宫格拼图，缺乏视觉焦点 |

#### ✅ 进阶构图技法（共 12 种，按模块类型推荐）

**C1 · 全出血英雄构图（Full-Bleed Hero）**
```
模特/产品占满整个画面（边到边），文字以半透明叠加层
或直接叠在模特身体留白区域（如肩膀上方、腿侧空间）。
适用：HERO / LIFESTYLE
示例：模特全身占满 21:9 画幅，品牌名小字叠在右上角，
     一行标题轻放在模特腰部右侧的负空间。
```

**C2 · 产品出框构图（Product Breaks Frame）**
```
产品/模特的一部分"溢出"画面边缘——头顶被裁、脚底出框、
手臂延伸到画面外。制造张力和动态感，暗示产品比画面更大。
适用：HERO / FEATURE / COLOR
示例：模特从膝盖以下被裁切，只看到腰臀区域铺满画面，
     高腰产品成为绝对视觉中心。
```

**C3 · 不等比叠层构图（Asymmetric Overlay）**
```
大面积产品/模特图（75-85%）+ 小面积半透明/磨砂信息浮层
（15-25%），浮层不另占独立区域，而是叠在产品图的边缘
或角落，像杂志的浮动文字框。
适用：FEATURE / TECH / USP
示例：产品特写占满画面，右下角有一个圆角半透明磨砂白色
     浮层（约 20% 面积），内有 3 行功能要点。
```

**C4 · 对角线动态构图（Diagonal Dynamic）**
```
画面沿对角线分割——不是垂直/水平切割。对角线一侧是
产品/模特，另一侧是辅助信息。对角线制造运动感和能量感。
适用：HERO / FEATURE / SPORT
示例：从左下到右上的对角色块分割，左上三角区是深色+文字，
     右下三角区是模特向画面外迈步的动态姿势。
```

**C5 · 浮岛信息构图（Floating Islands）**
```
产品大图作为背景画布，2-4 个小型信息"浮岛"（圆形/药丸形
/圆角矩形）散布在产品周围的负空间中，每个浮岛内仅 1-2 行
极简文字或图标。浮岛之间用细线或虚线连接到产品对应部位。
适用：CALLOUT / FEATURE / TECH
示例：塑身衣全身图居中，周围浮着 4 个小圆角标注岛，
     细线指向腰部/臀部/腿部/裆部，每岛一个功能名。
```

**C6 · 环绕构图（Wrap-Around）**
```
产品/模特居中偏大（60-70%），文字和小图标环绕在产品周围
的弧形路径上，像行星环绕恒星。信息围绕产品而存在，
视觉重心永远在中心的产品上。
适用：USP / FEATURE / TECH
示例：产品居中，上方弧形排列品牌名，下方弧形排列 3 个功能词，
     左右各一个小特征图标。
```

**C7 · 大小对比构图（Scale Contrast）**
```
同一产品/元素以极大和极小两种尺度同时出现——巨大的产品主图
+ 小缩略图细节/色卡/变体。大小极端对比制造视觉张力。
适用：COLOR / DETAIL / FEATURE
示例：中央一个占画面 70% 的模特穿搭大图，底部一排
     微小的 3 个色彩变体缩略图（每个仅占 5%）。
```

**C8 · 出血裁切特写（Extreme Crop Close-Up）**
```
超激进裁切——只展示产品/身体的某个局部，铺满整个画面，
文字极少（仅角落标注）。用于细节展示和面料/材质模块。
适用：TECH / FABRIC / DETAIL / CALLOUT
示例：腰部横切特写铺满 21:9 画幅，只看到高腰裤头和腹部线条，
     左下角极小字 "84% Nylon · 16% Elastane"。
```

**C9 · 竖条不等宽并置（Unequal Vertical Strips）**
```
3-4 条竖向条带，宽度不等（如 15%-50%-20%-15%），
最宽条带放模特/产品主图，其余窄条带放细节特写或纹理色块。
每条视觉内容不同，制造杂志般的编辑感。
适用：FEATURE / DETAIL / COLOR
⚠️ 规则：最宽条带 ≥ 总宽 45%，其余条带均 ≤ 20%。
         禁止等宽分割。
```

**C10 · 暗调产品悬浮（Dark Floating Product）**
```
深色/黑色背景 + 产品以戏剧性灯光悬浮在画面中央偏上，
下方或侧面有极精致的小号文字信息。灯光和阴影是主要
的视觉语言，文字几乎隐于暗处。
适用：TECH / HERO（暗调路线）/ FABRIC
示例：纯黑背景，产品被一束侧光照亮，下方 1/5 区域
     有淡灰色细字标注面料参数。
```

**C11 · 渐隐叠层构图（Gradient Fade Overlay）**
```
产品/模特图从一侧向另一侧渐隐为纯色或渐变色，文字
叠在渐隐区域。不是硬切割——是柔和过渡。产品从实到虚，
文字在虚化区域浮现，画面整体性强。
适用：HERO / FEATURE / BRAND
示例：模特从左侧清晰呈现，向右侧逐渐渐隐为暖米色，
     右侧渐隐区域有轻薄的标题文字。
```

**C12 · 留白呼吸构图（Breathing Space）**
```
产品/模特仅占画面 45-55%（居中或偏位），四周大量
高品质留白（不是空白——是有质感的纯色/微纹理空间）。
文字极少极轻，仅在留白区的一个位置出现。
整体传递高端、呼吸感、杂志编辑般的克制美。
适用：HERO / LIFESTYLE / BRAND
示例：模特居中偏左占 50%，四周大面积暖灰色留白，
     右下角一个极小的品牌名和一行利益句。
```

#### 横构图产品/模特最大化策略（Wide-Frame Product Maximization）

> **核心问题**：横构图（如 21:9 / 16:9 / 3:2）宽度远大于高度，站立的模特无法自然填满画幅。设计者本能反应是"模特放一边，剩余空间放文字"——这就是文字区过大的根源。以下策略**从视觉填充逻辑出发**，让产品/模特在横构图中也能占据绝对主导，从根本上消除"文字凑面积"的需求。

**WF1 · 模特侧躺/斜倚（Reclining Model）**
```
让模特的身体方向与画面长边一致——侧躺、斜倚、半卧。
人体从水平方向铺展，自然填满横构图的宽度，无需任何
文字区域来"补空"。这是横构图中展示塑身衣/内衣/泳装
等贴身产品的最强画面填充策略。

适用：HERO / LIFESTYLE / FEATURE（贴身品类）
Prompt 关键词：
  "model reclining on her side along the full width of the 
   21:9 frame, body stretching from left edge to right edge,
   [product] clearly visible across the entire torso and hip line"

变体：
  · 正侧卧：身体完全侧向，展示腰臀轮廓线
  · 斜倚撑肘：上半身微抬，展示产品正面结构
  · 俯卧回眸：面朝下微转头，展示背面设计
  · 半靠枕/扶手：生活场景感，带互动道具

⚠️ 侧躺时的产品展示检查：
  · 腰腹核心功能区是否朝向镜头？（禁止背对镜头侧躺导致产品不可见）
  · 手臂是否遮挡产品关键区域？（参照 model-pose-design.mdc 禁忌表）
```

**WF2 · 产品功能区极致特写（Product Zone Extreme Crop）**
```
不拍全身——只裁切产品所在的身体区域（腰臀/胸背/腿部），
让产品铺满整个横幅画面。这是文字最少的构图方式之一，
因为产品本身就是 100% 的画面内容。

适用：FEATURE / CALLOUT / TECH / DETAIL
Prompt 关键词：
  "extreme close-up crop: only the waist-to-hip area visible,
   filling the entire 21:9 frame edge to edge. The [product]
   is the sole visual subject — every texture, seam, and 
   contour of the product clearly visible at large scale."

裁切区域选择（按品类）：
  · 塑身衣/高腰裤：腰部到大腿中段横切
  · 运动内衣/Bra：肩膀到腰部横切
  · 连体衣：肩膀到臀部下缘横切（侧面角度最佳）
  · 塑形裤/Legging：臀部到小腿横切

⚠️ 极致特写不等于无人物——人体轮廓/肌肤质感仍在，
   只是隐去面部和四肢末端，让产品成为唯一主角。
```

**WF3 · 同款前后视角并置（Front-Back Dual View）**
```
同一画面中同时展示模特穿着产品的正面和背面——两个视角
并排或前后错落，共同填满横构图。这比"模特+文字"的
信息密度更高且更有说服力，因为消费者一眼看到产品的
360° 效果。

适用：HERO / FEATURE / COLOR / BACK VIEW
三种变体：

变体 A · 同色前后视角：
  "Two models side by side in the 21:9 frame: left model 
   facing forward showing the front of the [product] in [color],
   right model turned fully away showing the back view of the 
   same [product] in the same [color]. Both models fill the 
   frame height. Minimal text — only a small brand name overlay."

变体 B · 异色前后视角：
  "Two models side by side: left model facing forward wearing 
   the [product] in [color A], right model back-facing wearing 
   the same [product] in [color B]. This simultaneously shows 
   front/back design AND color options."

变体 C · 三视角（正+侧+背）用于宽幅：
  "Three models in a row across the 21:9 frame: left facing 
   forward, center at 3/4 angle, right fully back-facing. 
   All wearing the same [product]. Progressively revealing 
   the product from front to back."

布局规则：
  · 两模特时：左 48% + 右 48% + 中间 4% 呼吸间隙
  · 三模特时：左 30% + 中 36% + 右 30% + 间隙
  · 模特之间不用硬分割线——用灯光差异或微妙背景色差区分
  · 文字仅限底部极窄条（≤画面高度 8%）或角落品牌名

⚠️ 前后视角并置时的姿势差异化规则：
  · 正面模特和背面模特的姿势必须不同（参照 model-pose-design.mdc 双模特互补规则）
  · 正面模特偏动态（手叉腰/撩发），背面模特偏静态（自然垂手/S曲线）
  · 三视角时中间模特用最动态的姿势（走步/转身中间态）
```

**WF4 · 多色产品平铺阵列（Color Array Flat Lay）**
```
不用模特——将多色产品平铺排列填满横构图，产品本身
就是视觉主体。适合色卡展示、产品概览、面料触感展示。

适用：COLOR / OVERVIEW / FABRIC
Prompt 关键词：
  "Flat-lay arrangement: three [products] in [color A], [color B], 
   and [color C] laid out side by side filling the 21:9 frame. 
   Each product neatly folded or naturally draped. Clean studio 
   background. No text except tiny color name labels beneath each."
```

> **策略选择逻辑**：当横构图中模特站立只能填满画面的 30-40% 宽度时，**不要用文字填剩余空间**——改用以上四种策略让产品/模特自然填满画幅。

#### 模块级构图推荐矩阵

每个模块应根据其功能选择最佳构图，**同一详情页的 7 个模块必须使用至少 4 种不同构图技法**：

| 模块类型 | 首选构图 | 备选构图 | 禁用构图 |
|---------|---------|---------|---------|
| HERO 首屏 | C1 全出血 / WF1 侧躺 / WF3 前后并置 | C2 出框 / C4 对角线 / C11 渐隐叠层 / C12 留白呼吸 | ❌ 50/50 分割、❌ 模特缩到一侧+大面积文字区 |
| FEATURE 功能 | WF2 功能区特写 / C3 不等比叠层 / C5 浮岛 | C6 环绕 / WF3-B 异色前后 | ❌ 等面积左右分 |
| CALLOUT 标注 | WF2 功能区特写 / C5 浮岛 / C8 出血裁切 | C3 不等比叠层 | ❌ 大面积文字框 |
| TECH 面料 | C8 出血裁切 / C10 暗调悬浮 / WF2 特写 | C3 不等比叠层 | ❌ 文字区 > 产品区 |
| COLOR 多色 | WF3 前后并置 / WF4 平铺阵列 / C7 大小对比 | WF3-C 三视角 / C9 不等宽竖条 | ❌ 等宽网格 |
| BACK VIEW | WF3-A 同色前后 / C1 全出血 / C2 出框 | WF2 背部特写 / C5 浮岛 | ❌ 50/50 分割 |
| LIFESTYLE | C1 全出血 / WF1 侧躺 / C12 留白呼吸 | C11 渐隐 | ❌ 文字区 > 10% |
| SIZE-GUIDE | C9 不等宽竖条（唯一允许信息密集） | — | — |

### 第三法则：文字是"浮标"，不是"区域"

#### 字体层级体系（Prompt 构造时必须遵守）

```
H1 主标题 ─ 画面中最大文字，但≤产品/模特高度的 25%
  字重：Bold/Semibold | 字间距：微加宽 0.05-0.1em
  风格：clean sans-serif 或 elegant serif
  位置：叠在产品/模特留白区，不另开独立区域

H2 副标题 ─ H1 字号的 40-50%
  字重：Regular/Light | 与 H1 形成明显的粗细对比

标签文字 ─ H1 的 20-25%，ALL CAPS + 加宽字间距 0.1-0.2em
  精致小型大写，点缀式出现

说明文字 ─ H1 的 25-30%，Regular/Light
  行高 1.4-1.6，可读性优先

⚠️ 铁律：H1 标题的绝对字号不得超过画面短边的 8%。
   例如 21:9 宽幅图（短边 = 高度），标题高度 ≤ 画面高度的 8%。
```

#### Prompt 中的视觉主角约束语句（生图时强制附加）

为欧美详情页模块构造 Prompt 时，**必须在 Prompt 开头（产品/模特描述之前）附加**以下约束：

```
# Hero 首屏约束（最严格）
COMPOSITION RULE: The model/product is the DOMINANT visual hero — 
occupying at least 70% of visual weight. Text is minimal accent only: 
brand name small, one headline at most 15% of frame, placed as a 
floating overlay on the model's negative space. Do NOT create a 
separate text zone or panel. Do NOT split the image into equal halves. 
The model must be LARGE, POWERFUL, and COMMANDING in the frame.

# 功能/技术模块约束
COMPOSITION RULE: Product/detail photography is the visual hero — 
at least 55% of frame. Text elements float as small refined labels 
or a compact overlay panel (max 20% of frame). Do NOT create large 
text zones competing with the product visual. Prefer floating callout 
islands, small badges, or subtle overlays rather than dedicated text columns.

# 生活场景/对比约束
COMPOSITION RULE: The visual (lifestyle photo / before-after comparison) 
fills the entire frame edge to edge. Text is near-invisible accent — 
a small watermark-style brand name or a single short caption. 
Max 10% text coverage.
```

#### Prompt 构图描述规范（替代旧的空间分割描述法）

**❌ 禁止的 Prompt 描述方式（旧方法 = 导致二分割）：**
```
# 这些写法会直接导致无聊的 50/50 分割
❌ "Left 50% zone: model. Right 50% zone: text and features."
❌ "Left half: product image. Right half: headline and bullet points."
❌ "Top area: image. Bottom area: text block."
❌ "Two-zone split layout."
```

**✅ 正确的 Prompt 构图描述方式（视觉优先）：**
```
# 先描述视觉主体（大、具体、有力），再描述文字（小、轻、浮动）
✅ "Full-frame composition: the model fills the entire 21:9 frame from 
   edge to edge, body in dynamic 3/4 pose. Brand name floats as a small 
   refined logotype in the upper-right breathing space. A single-line 
   headline sits lightly in the negative space beside the model's hip, 
   at roughly 8% of frame height."

# 用构图技法名称引导 AI
✅ "Asymmetric overlay composition: the product close-up fills 80% of 
   the frame. A small frosted semi-transparent panel (15% of frame area) 
   floats at the bottom-right corner containing 3 short feature labels."

# 明确禁止分割
✅ "Do NOT split the frame into separate zones. The image is a single 
   cohesive visual with text overlaid on top, not placed in a 
   dedicated text region."
```

### 与 M8 的接口补充

M15 传递给 M8 的模块指令中，需包含「排版精修等级」和「构图技法」两个字段：

| 精修等级 | 适用 | M8 行为 |
|---------|------|---------|
| `premium` | 欧美高端 DTC / 品牌线 | 强制附加 Hero 级视觉主角约束，文字 ≤ 15% |
| `standard` | 欧美 Amazon A+ / Shopee | 附加功能模块级约束，文字 ≤ 20% |
| `dense` | 中国市场 / 大促 | 不附加排版约束（中国市场信息密集型正常） |

| 构图技法 | 值 | 含义 |
|---------|------|------|
| `composition` | `C1` ~ `C12` | 指定该模块使用的构图技法编号 |

#### 详情页 7 模块构图节奏示例（模板）

一个优秀的详情页应有如下构图节奏变化：

```
示例 A（贴身品类 · 塑身衣/内衣/泳装）：
M1 HERO:    WF1 侧躺       → 模特侧卧铺满宽幅，产品全览，角落极小文字
M2 FEATURE: WF2 腰臀特写    → 腰到臀极致裁切铺满画面 + 浮岛标注
M3 TECH:    C10 暗调悬浮    → 深色底 + 面料纹理戏剧光影
M4 BACK:    WF3-A 同色前后  → 正面+背面模特并排填满画面
M5 DETAIL:  C9 不等宽竖条   → 腰带特写 + 棕色背面 + 棉裆细节
M6 COLOR:   WF3-B 异色前后  → 黑色正面 + 裸色背面 + 棕色3/4侧
M7 SIZE:    C9 不等宽竖条   → 唯一的信息密集模块

示例 B（外穿品类 · 上衣/外套/裤装）：
M1 HERO:    C1 全出血       → 模特全身铺满画面，文字浮层
M2 FEATURE: WF3-A 同色前后  → 正面+背面双视角展示版型
M3 TECH:    WF2 面料特写    → 面料纹理极致裁切
M4 DETAIL:  C5 浮岛         → 产品大图 + 工艺细节浮岛标注
M5 LIFESTYLE: C12 留白呼吸  → 场景大图 + 极少文字
M6 COLOR:   WF4 平铺阵列    → 多色产品平铺填满
M7 SIZE:    C9 不等宽竖条   → 信息密集模块
```

> 注意节奏：**每种构图技法在同一组中最多使用 2 次**，明暗交替、密疏交替、全身↔特写交替——制造视觉呼吸感。

## 详情页 AI 生图协调协议

### 与 M8 的协同

M15 负责确定**每张图的内容和功能定位**，M8 负责将其转化为 AI 生图 Prompt：

```
M15 输出                          M8 接收
┌────────────────┐              ┌────────────────┐
│ 模块类型：CALLOUT │ ──────────→ │ Prompt 构造     │
│ 内容：3个功能标注  │              │ 含参考图引用    │
│ 文案：具体文字    │              │ 含平台尺寸     │
│ 参考：N域记录#X  │              │ 含风格约束词    │
│ 色彩：品类色系    │              │ 含色彩语义约束  │
│ 首屏策略：S[X]型 │              │ 含首屏策略指引  │
└────────────────┘              └────────────────┘
```

### 批量生图节奏

一个完整详情页通常需要 7-15 张图。执行策略：

```
Phase 1（并发）：所有独立模块同时生成
  HERO / USP-ICON / CALLOUT / TECH / SPEC-TABLE 等
  → 遵守 batch-image-concurrent 规范，一次性并发

Phase 2（依赖型）：需要 Phase 1 结果的模块
  如 LIFESTYLE 需要和 HERO 保持视觉一致性
  → 串行或小批次并发

Phase 3（精修）：整体审查后的调整
  → 按需迭代单张图片
```

### 详情页生图默认参数

| 参数 | 默认值 | 说明 |
|------|-------|------|
| `--resolution` | `1k` | 除非用户指定，一律 1k |
| `--aspect-ratio` | 按平台规范 | Amazon A+: 约 3:2 / 天猫: 约 3:4 或 1:1 / 自定义 |
| `--label` | `[品牌]-DP-M[模块序号]-[模块类型]` | 如 `SKIMS-DP-M01-HERO` |

## 详情页学习模式（M10 联动）

当 M10 处理详情页素材时，M15 提供分析框架：

### 识别详情页素材的触发条件

以下特征表明素材是详情页（而非单张海报/Banner）：
- 长图（高度 >> 宽度）
- 包含多个明显不同的内容区块
- 包含产品参数/规格信息
- 包含"模块化"的版式结构
- 文件名含"详情""listing""A+""DP"等关键词

### 分析输出格式

使用 `N_域索引.txt` 中定义的「N 域增强版学习记录格式」，必须包含：
1. 📐 详情页结构速写（模块清单、数量、节奏、首屏策略、信息层级、转化机制）
2. 【品类设计规律提炼】
3. 【可迁移原则】

### 路由规则

| 素材市场 | 路由目标 |
|---------|---------|
| Amazon/DTC/Shopee 等欧美平台 | N1_欧美详情页参考.txt（或细分后的子文件） |
| 天猫/淘宝/京东/拼多多等中国平台 | 按品类精确路由到N2对应子文件（见N域精确路由表） |
| 无法判断市场 | 根据语言判断（中文→N2，英文→N1） |
