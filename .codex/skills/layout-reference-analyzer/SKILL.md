# 版式参考图分析与复刻引擎

## 概述

当用户给出一张包含多个板块的版式参考图，本 Skill 将参考图拆解为独立模块，对每个模块进行八维结构化分析，输出可直接用于生图 Prompt 的精确版式描述——让 AI 生图不再"大概像"，而是"模块级还原"。

## ⚠️ 强制自动触发规则（与 design-must-use-main-control 流程绑定）

**本 Skill 已被注册为设计流程的 ⑤.9 阶段门（见 `design-must-use-main-control.mdc`）。**  
凡设计任务满足以下检测条件，**无需用户主动说明**，必须在构造 Prompt（⑥）之前自动调用本 Skill 完成分析。

### 自动触发检测（两项均需判断）

**检测项 A：用户是否提供了版式参考图？**

下列任意情形 = 版式参考图已提供：
- 用户在对话中上传了含多个内容区块的图片（Amazon A+ 截图、DTC 落地页、电商长图、竞品详情页、多模块 Banner）
- 用户表述中出现："参考这张图""按这个风格/排版""和这张一样的版式""参考图5/图N 的排版"
- 用户同时提供了「多张产品图 + 一张版式长图」——其中的长图即为版式参考图

**检测项 B：任务是否需要生成多张模块图？**

下列任意情形 = 多模块生图任务：
- 详情页 / Amazon A+ / 产品 listing 设计
- DTC 独立站落地页
- 多模块 Banner 组
- 用户要求「一套图」「系列图」「多张风格一致的图」

**判定逻辑**：
- A 且 B → **强制触发**，在 ⑥ 构造 Prompt 之前必须先完成本 Skill Phase 1-4 分析
- 仅 A 但非 B（单张图借鉴风格）→ **建议触发**，若版式比较复杂（≥3个区域）仍应触发
- 仅 B 但非 A（无参考图）→ **不触发**，参考 N 域已有结构记录设计即可
- A 和 B 均否 → **不触发**

### 触发豁免

以下情形**不触发**，即使用户提供了图片：
- 用户明确说"只借鉴色调/氛围/风格"，无需复刻版式结构
- 用户提供的图片为单张产品图、模特图、Logo——非版式参考
- 用户明确说"跳过版式分析"

## 触发条件（兼容表述识别）

以下任意表述触发本 Skill：

| 用户表述 | 示例 |
|---|---|
| 明确说「参考这张图的版式/排版/结构」 | "参考这张 A+ 图的排版来设计" |
| 提供一张明显含多模块的长图/详情页截图 | 上传一张 Amazon A+ 截图 |
| 要求「按照参考图的风格和布局」生成系列图 | "按这个风格出一套详情页" |
| 要求「和这张图一样的排版」 | "做成和这张一样的版式" |
| **提供多张图中有一张是版式长图（自动识别）** | 用户同时传入产品图和竞品详情页截图 |

**不触发**：用户仅提供单张产品图/模特图作为素材（非版式参考），或明确说"只借鉴色调/氛围"。

---

## ⚠️ 核心原则：版式靠图，不靠文（Prompt Hygiene for Layout）

> **参考图已经承载了全部版式视觉信息。文字越多，模型越按"文字描述的版式"去生成，越偏离参考图本身。**

这个 Skill 最容易被误用的地方是——把八维分析结果当作"更详细的 Prompt 模板"写进生图指令，最后得到一份长达 500+ token 的英文版式描述。结果是：

- 模型的注意力预算被文字描述吃掉，**参考图反而被降权**
- 生成结果看起来像"Prompt 想象的版式"，而不是"参考图里的版式"
- 越详细越偏离（与 `prompt-hygiene-no-internal-ids.mdc` 铁律三同源的陷阱）

### 分析目标 ≠ Prompt 内容

| 产出物 | 去处 | 角色 |
|---|---|---|
| Phase 1-4 的八维分析报告 | **assistant 回复正文** | 给用户看 + 内部决策审计依据（让人类能确认"你看懂了参考图"） |
| 版式引用句 + 差异点清单 + 关键锚点约束 | **生图 Prompt 正文** | 交给模型的极简指令：引用版式 + 标明差异 + 锁定关键元素 |

**禁止**把八维分析报告的英文翻译直接塞进 `--prompt`。见 Phase 5 的正确写法。

### 正反对照（以本次 NEBILITY 短裤 9:16 任务为例）

❌ 错误（过度描述版，原有 Phase 5 引导出的长 Prompt）：
```
第一区 — 品牌页眉（画布顶部约 10%）
- 居中大号 NEBILITY 品牌字，精致现代无衬线字体，宽字间距，颜色 #1A1A1A，占画布宽度约 45%。
- 正下方一行细小居中大写文字：THE SCIENCE OF A PERFECT CURVE……

第二区 — 首屏分栏（画布高度约 10%-45%）
- 左右 50:50 水平分栏……
……（共 5 个区域，每个区域 5-10 行文字描述）
```

后果：生成结果完全偏离参考图 6 的黑色剪影极简美学，成了"按 prompt 文字重新设计的新版式"。

✅ 正确（极简版，靠图 4 承载版式）：
```
[版式 — 来自 image 4]
完整复刻 image 4 的竖向多区段版式模板——区段数量、堆叠顺序、各区比例、模块形状、字体层级、红色点缀使用方式、留白节奏全部保持一致。

[与 image 4 模板的差异点]
- 将所有区段的黑色剪影占位替换为 image 1、2、3 中实际拍摄的塑形短裤和模特照片。
- 将灰色文字占位替换为真实品牌标题"NEBILITY"和标语"THE SCIENCE OF A PERFECT CURVE"。
- 将矩阵区段标题从"SHAPEWEAR BODYSUIT"改为"SHAPEWEAR SHORTS"。
- image 4 中的其余元素保持不变（红色装饰下划线、4 圆形矩阵、NEW 翻页箭头、尺码表页脚）。
```

短 80%，版式靠 image 4 承载，模型生成的结果会紧贴参考图原貌。

## 核心流程

```
用户上传版式参考图
 ↓
Phase 1：全局概览 — 识别参考图整体风格基调（仅 assistant 回复）
 ↓
Phase 2：模块分割 — 从上到下逐一识别独立模块，编号（仅 assistant 回复）
 ↓
Phase 3：逐模块八维分析 — 理解版式的内部诊断（仅 assistant 回复）
 ↓
Phase 4：输出【版式分析报告】— 结构化分析，展示给用户确认（仅 assistant 回复）
 ↓
Phase 5：生成 Prompt 三件套 — ① 版式引用句 ② 差异点清单 ③ 关键锚点约束
         （⚠️ 只有三件套进 --prompt；Phase 1-4 的文字分析不进 Prompt 正文）
```

> Phase 1-4 的作用是**让 assistant 和用户都看清楚参考图的版式**（人类审计），但**模型本身靠参考图像素看版式**，不需要人类再用英文翻译一遍。

## Phase 1：全局概览

读取参考图后，首先输出整体风格判断：

```
【全局风格概览】
━━━━━━━━━━━━━━━━━━━━━━━━━
• 参考图类型：[Amazon A+ / DTC 落地页 / 电商长图 / 社媒组图 / 其他]
• 整体风格基调：[高端简约 / 促销活力 / 功能技术 / 生活方式 / 编辑杂志]
• 主色系：[列出 2-3 个主色 HEX 或色名]
• 背景色基调：[纯白 / 浅灰 / 暖米 / 深色 / 渐变 / 混合]
• 字体风格大方向：[无衬线现代 / 衬线优雅 / 混合使用]
• 图文比例（整体）：[图重 70:30 / 均衡 50:50 / 文重 30:70]
• 模块总数：N 个
• 模块间过渡风格：[硬切分 / 渐变过渡 / 共享背景 / 留白分隔]
━━━━━━━━━━━━━━━━━━━━━━━━━
```

## Phase 2：模块分割

从参考图顶部到底部，按视觉边界识别每个独立模块。

### 分割判定标准

出现以下任一信号即判定为模块边界：

| 边界信号 | 说明 |
|---|---|
| 背景色/底色变化 | 从白色区切换到有色区，或反之 |
| 明确的水平分隔线/留白带 | 两个内容块之间有明显间距 |
| 内容主题切换 | 从产品展示切换到功能说明，从模特图切换到面料细节 |
| 布局网格突变 | 从单列切换到多列，或从图文并排切换到纯图 |
| 标题/Section Heading 出现 | 新的大标题通常标志新模块开始 |

### 编号规则

```
REF-M1  首屏 / Hero 模块
REF-M2  第二模块
REF-M3  第三模块
...
REF-MN  第 N 模块
```

### 分割输出格式

```
【模块分割结果】
━━━━━━━━━━━━━━━━━━━━━━━━━
REF-M1：[简称，如"首屏Hero Banner"]（参考图顶部 ~ 约 15% 高度）
REF-M2：[简称，如"面料功能区"]（约 15% ~ 30% 高度）
REF-M3：[简称，如"多角度产品展示"]（约 30% ~ 45% 高度）
REF-M4：[简称，如"Before/After 对比"]（约 45% ~ 60% 高度）
REF-M5：[简称，如"生活场景 Lifestyle"]（约 60% ~ 75% 高度）
REF-M6：[简称，如"买家秀/社交证明"]（约 75% ~ 100% 高度）
━━━━━━━━━━━━━━━━━━━━━━━━━
```

## Phase 3：逐模块八维分析

对 Phase 2 识别出的每个模块，执行以下八个维度的深度分析。

### 八维分析框架

#### D1 · 布局网格（Layout Grid）

精确描述模块内部的空间划分方式。

| 分析要素 | 输出内容 |
|---|---|
| 列数 | 1列 / 2列 / 3列 / 4列 / 不规则 |
| 列宽比 | 50:50 / 60:40 / 33:33:34 / 25:25:25:25 等 |
| 行数 | 1行 / 2行 / 标题行+内容行 |
| 对齐方式 | 左对齐 / 居中 / 右对齐 / 两端对齐 |
| 内边距感知 | 紧凑无边距 / 适中边距 / 大量留白 |

**输出示例**：
```
D1 布局网格：4等宽列，标题行居中 + 内容行4列平分，
列间距约 2-3% 画面宽度，每列内部上图下文垂直堆叠
```

#### D2 · 视觉比重（Visual Weight）

描述模块中各元素的视觉面积占比和主次关系。

| 分析要素 | 输出内容 |
|---|---|
| 视觉主角 | 产品图 / 模特图 / 文字标题 / 图标组 |
| 主角面积占比 | 约 X% |
| 辅助元素 | 标签 / 副标题 / 图标 / 色块 |
| 图文比 | 图 X : 文 Y |
| 视觉重心位置 | 偏左 / 居中 / 偏右 / 上方 / 下方 |

**输出示例**：
```
D2 视觉比重：左侧模特为主角（约 55% 面积），右侧文字区为辅（约 35%），
留白约 10%。视觉重心偏左，模特引导视线向右流动至文字区。
```

#### D3 · 元素清单（Element Inventory）

逐一列出模块中出现的所有可见元素。

```
D3 元素清单：
- [ ] 模特全身/半身/特写 → 位置：左半/右半/居中/全幅
- [ ] 产品实物图 → 位置、尺寸占比
- [ ] 产品细节特写 → 数量、排列方式
- [ ] 主标题（H1） → 内容、位置
- [ ] 副标题（H2） → 内容、位置
- [ ] 正文描述 → 内容、位置
- [ ] 功能图标 → 数量、排列、风格（线性/填充/手绘）
- [ ] 标签/Badge → 形状（圆角矩形/圆形/药丸形）、颜色
- [ ] CTA 按钮 → 文案、形状、颜色
- [ ] 分隔线/装饰线 → 粗细、颜色
- [ ] 背景图/纹理 → 类型
- [ ] 色块/色条 → 形状、位置、颜色
```

#### D4 · 文字层级（Typography Hierarchy）

分析模块中所有文字元素的层级关系。

| 层级 | 分析内容 |
|---|---|
| H1 主标题 | 字号感知（特大/大/中）、字重（粗/中/细）、字体类型（无衬线/衬线）、颜色、大小写（ALL CAPS / Title Case / sentence case）、字间距（正常/宽松） |
| H2 副标题 | 同上 |
| Body 正文 | 同上 |
| Label 标签 | 同上 |
| Caption 图注 | 同上 |

**输出示例**：
```
D4 文字层级：
- H1: 大号粗体无衬线，ALL CAPS，深炭色，宽字间距，居中
- H2: 中号常规衬线斜体，sentence case，中灰色，居中，在 H1 正下方
- Label: 小号无衬线，ALL CAPS，装在圆角矩形标签框中，深色底白字
```

#### D5 · 配色方案（Color Palette）

```
D5 配色方案：
- 背景色：[色值或色名]
- 主文字色：[色值]
- 副文字色：[色值]
- 强调色/点缀色：[色值]（用在 CTA/标签/图标）
- 图片色调倾向：[暖色/冷色/中性/低饱和/高饱和]
```

#### D6 · 摄影/图片风格（Visual Style）

```
D6 图片风格：
- 摄影类型：[棚拍白底 / 场景拍 / 生活方式 / 平铺展示 / 3D 渲染]
- 光线特征：[柔光均匀 / 侧光戏剧性 / 自然窗光 / 高调亮白]
- 后期风格：[自然无滤镜 / 低饱和电影感 / 高对比商业 / 温暖复古]
- 裁切方式：[全身 / 半身 / 胸上 / 腰腹特写 / 产品微距]
- 模特视角：[正面 / 3/4侧 / 侧面全侧 / 背面 / 回眸]
```

#### D7 · 留白与节奏（Whitespace & Rhythm）

```
D7 留白节奏：
- 模块内留白：[紧凑 / 适中 / 宽松]（元素间距占模块面积比）
- 元素排列节奏：[均匀等距 / 密上疏下 / 居中聚拢 / 向心辐射]
- 呼吸感：[高端留白型 / 信息密集型 / 均衡型]
```

#### D8 · 模块间过渡（Transition）

```
D8 过渡方式：
- 与上一模块的过渡：[硬切换（背景色突变）/ 渐变 / 共享背景 / 大留白]
- 与下一模块的过渡：同上
- 视觉连续性手段：[统一背景色 / 统一色系 / 重复元素 / 无连续手段]
```

## Phase 5：生成 Prompt 三件套（只有这三件进 `--prompt`）

版式参考图已作为 image N 传入，模型能直接看见所有版式像素。Prompt 要做的不是**复述版式**，而是做三件事：

### 三件套结构

```
[版式 — 来自 image N]               ← 件套 ①：版式引用句（1-3 句，最短）
完整复刻 image N 的[类型词]模板——[区段数量 / 堆叠顺序 / 模块形状 / 各区比例 / 
点缀色使用方式 / 字体层级 / 留白节奏]全部保持一致。

[与 image N 模板的差异点]    ← 件套 ②：差异点清单（逐条短句）
- 将[模板中占位符 A]替换为[实际内容 A，引用 image 1/2/...]
- 将[模板中占位符 B]替换为[实际内容 B]
- 将[模板中章节标题/文字占位]改为[实际文字]

[关键锚点 — image N 中必须保留]  ← 件套 ③：关键锚点约束（可选，1-4 条）
- image N 中的[具体版式锚点 1]保持不变
- image N 中的[具体版式锚点 2]保持不变
```

### 三件套写作规则

| 件套 | 规则 | 禁忌 |
|---|---|---|
| ① 版式引用句 | 用抽象类目词（竖向多区段模板 / 左右分栏前后对比 / 四列功能矩阵），不拆解为尺寸/像素/角度 | 禁止写列宽比、模块高度百分比、圆形直径、字号层级等——这些已经在 image N 里 |
| ② 差异点清单 | 只写"模板里**没有**但画面**必须有**的内容"——实际产品/模特/品牌文字/章节改名 | 禁止描述产品的颜色/材质/结构（见 `prompt-hygiene-no-internal-ids.mdc` 铁律三），直接引用 image 1/2/3 |
| ③ 关键锚点约束 | 仅在参考图中**容易被模型弱化或替换**的关键元素上用（例：红色下划线、左右翻页箭头、分页圆点、标签形状），锁定即可 | 禁止把全部八维分析结果列为锚点——锚点是**少量关键位**，不是"面面俱到" |

### 示例（本次 NEBILITY 短裤 9:16 任务）

假设 image 1/2/3 = 产品多视角图，image 4 = 图 6 版式长图，image 5 = 色卡：

```
[版式 — 来自 image 4]
完整复刻 image 4 的竖向多区段 Lookbook 模板——区段数量与堆叠顺序、模块形状、各区比例、红色点缀使用方式、字体层级、留白节奏全部保持一致。

[与 image 4 模板的差异点]
- 将所有黑色剪影占位替换为 image 1、2、3 中实际拍摄的塑形短裤和模特照片。
- 将灰色段落/条块占位替换为真实品牌标题"NEBILITY"和标语"THE SCIENCE OF A PERFECT CURVE"。
- 将矩阵区段标题从"SHAPEWEAR BODYSUIT"改为"SHAPEWEAR SHORTS"。
- 在尺码表页脚行中填入"XS / S / M / L / XL"胶囊标签。

[关键锚点 — image 4 中必须保留]
- 每个区段标题下方的红色装饰下划线保持不变。
- 4 圆形矩阵、NEW 翻页轮播箭头、分页圆点（品牌红填充一个）在 image 4 中的位置保持不变。
- 首屏区段中的圆形模特缩略图和三个功能图标（柔软 / 舒适 / 高弹力）保持不变。
```

整段 ≤ 250 token（老版是 800+ token）。版式靠 image 4 承载，差异点靠文字承担，分工明确。

## 分析精度保障规则

### 必须做

- 读取参考图后完成完整的 Phase 1-4 分析报告，展示给用户确认"版式已看懂"
- 每个模块的 D1-D8 八维必须填写（作为内部审计依据和差异点来源）
- **Prompt 只写三件套（版式引用句 + 差异点清单 + 关键锚点）**，段落总长建议控制在 200-300 token
- 差异点清单只写"替换什么/改名什么/保留什么"，一律用 `image N` 引用实际视觉内容
- 关键锚点只锁定"容易被模型弱化或替换"的少量关键位（如品牌红装饰、箭头、圆点、Badge 形状）

### 禁止做

- ❌ **把 Phase 1-4 八维分析的英文翻译直接塞进 `--prompt`**（这是老版本的最大陷阱——文字过多会压过 image N 的视觉信号）
- ❌ 在 Prompt 中逐 SECTION 写 5-10 行英文版式描述（百分比、像素、字重、对齐方式……这些 image N 已经承载了）
- ❌ 不做模块分析就直接生图（= 无版式认知，差异点也写不准）
- ❌ 用"参考这张图的风格"一句话代替分析（太笼统，且找不到差异点）
- ❌ 在 Prompt 中只写 `use the layout from image N` 而不给差异点清单（= 模型不知道要替换占位符为什么实际内容）
- ❌ 把"4 列、列宽比 25:25:25:25、列间距 2%、圆形直径等于列宽"等**image N 本身就承载的尺寸信息**写进 prompt

## 注意事项

- 参考图分辨率较低时，配色分析以视觉感知为主，无需精确到 HEX 值
- 参考图若为竖版长图（如电商详情页），模块数可能超过 8 个，全部分析
- 版式参考图无论是中文还是英文版式，文字层级分析同样有效，Prompt 版式指令默认使用中文输出
- 不同模块可能共享同一张参考图传入（只占 1 个 image 编号），通过 Prompt 文字区分引用哪个模块的版式
