# 视频 Prompt 设计（Video Prompt Designer）

## 概述

视频/短视频生成的统一业务入口。前身是能力模块 M14_视频设计与生成 + 配套规范「可灵 / Kling 视频提示词规范」（原全局 rule kling-prompt-output-format，V8.x 起按 Skill 自包含原则迁入本 Skill `references/KLING_PROMPT_FORMAT.md`）。

## 与 ai-image-prompt-builder 的边界

| 任务 | 主入口 |
|---|---|
| 静态图（含视频首帧） | `ai-image-prompt-builder` |
| 视频/分镜/口播 | **本 Skill** |
| 视频任务需要先有静态首帧 | 本 Skill 调用 ai-image-prompt-builder 出首帧 |

## 三种工作模式

| 模式 | 触发场景 | 输出物 |
|---|---|---|
| A 完整视频设计 | 用户提供产品 + 平台 + 时长，需从零设计 | 分镜脚本 + 口播文案 + AI 视频提示词 |
| B 提示词优化 | 用户已有可灵/Gemini 提示词，需优化 | 优化后的提示词 |
| C 口播文案生成 | 用户只要口播脚本 | 六连击结构口播文案 |

## 视频提示词输出格式（摘要 · 详见 `references/KLING_PROMPT_FORMAT.md`）

> 本节是可灵 / Kling 规范的核心摘要。**输出可灵视频提示词前必读** `references/KLING_PROMPT_FORMAT.md`，该文件包含完整的密度对照表、节奏锁定句、禁止清单、自查清单等强制约束。

### 图生视频首帧精简硬门（可灵 / Kling 强制）

当用户上传了图片并要求生成可灵 / Kling 视频提示词时，上传图已经是模型可见的首帧或参考图。最终可复制提示词只描述三类信息：

1. 人物或产品接下来发生的动作
2. 镜头推进、拉回、跳切、特写等镜头变化
3. 用户明确要求的口播台词或图中不存在的新变化

不得在最终可复制提示词中复述首帧已经承载的视觉信息，包括人物外貌、服装颜色/款式、室内/户外环境、背景细节、光线氛围、画面风格、`image 1` 角色说明。
如果用户未明确要求口播，默认为纯展示模式，不写 `说"..."`。

**正确示例（单图 OOTD 纯展示，6 秒）：**

```
保持镜前自拍构图，人物轻轻侧身展示整体穿搭，抬手整理外套前襟，重心自然从一侧换到另一侧，镜头轻微推进至中景，人物抬眼看向手机屏幕并自然微笑，动作慢速、真实、流畅，衣料和头发随身体动作轻微摆动。
```

**错误示例：**

```
金发女性穿棕色西装外套和白色长裤站在温暖室内镜前，柔和自然光，高级生活方式氛围，参考 image 1 的人物和穿搭。
```

问题：复述了首帧中的人物、服装、环境、光线与 `image 1` 元信息，会稀释首帧权重。

### 负向提示词禁用规则

视频最终提示词不输出负向提示词。可灵 / Gemini / SeedDance 的视频提示词主体只写正向的动作、镜头变化、口播台词和画面中需要发生的变化。

不得在最终可复制提示词中加入 `[NEGATIVE]`、`Negative Prompt`、`负向提示词`、`不要`、`禁止`、`不得`、`避免`、`do not`、`no ...`、`never` 等负向约束句。质量边界只放在内部自查清单或诊断说明里，不喂给视频模型。

### 中文场景描述 + 英文台词混合格式

以下结构仅用于完整分镜/口播脚本展示；如果输出给可灵输入框的最终 Prompt，优先使用单段连续提示词，不添加标题、参数、解释或参考图目录。

```
【场景 1】（0:00-0:05）
镜头：[镜头描述，中文]
人物：[模特动作 + 表情，中文]
产品：[产品展示，中文]
台词："[Spoken English line, 5-8 words]"
氛围：[光线 + 色调，中文]
```

### 关键技术锚点（保留原样）

- 拍摄镜头术语：`close-up shot` / `medium shot` / `wide shot` / `over-the-shoulder` / `low-angle` / `high-angle`
- 运镜术语：`dolly in` / `pan left` / `tilt up` / `tracking shot`
- 时间码：`0:00-0:05`
- 模型参数：`--aspect-ratio 9:16`

## 镜头调度资料库：大师100镜头调度

资料文件：`references/MASTER_SHOTS_CAMERA_BLOCKING.md`

本资料库用于提升视频 Prompt 的镜头调度质量。使用原则不是照搬电影案例，而是先判断当前视频的叙事目的，再选择 1-2 个合适的镜头调度技法，把它们转译成可灵 / Gemini / SeedDance 能执行的动作与镜头语言。

### 调用时机

- 完整视频设计时：分镜脚本前必须先做一次【镜头调度选择】。
- 提示词优化时：若原 Prompt 只有动作、缺少镜头目的或镜头变化，必须用本资料库补强。
- 口播视频时：每 1-2 个口播段落最多选 1 个镜头调度技法，避免镜头抢过台词。
- 图生视频时：仍优先遵守「首帧精简硬门」，只写动作和镜头变化，不复述首帧视觉事实。

### 输出禁区

最终可复制 Prompt 中不得写入书名、案例电影名、章节号、技法编号或“参考大师镜头”等元信息。资料库只作为内部镜头设计依据，最终提示词只保留可执行的景别、运镜、演员动作和节奏。

### 镜头调度选择流程

```
明确视频目的
  ↓
判断情绪：展示 / 亲密 / 压迫 / 速度 / 揭示 / 疏离 / 权力 / 冲突
  ↓
从资料库选择 1-2 个镜头调度技法
  ↓
转译为具体提示词：景别 + 机位高度 + 运镜方向 + 主体动作 + 节奏
  ↓
按时长做密度自检，删除无法执行或过度复杂的镜头
```

### 镜头调度速查表

| 叙事目的 | 推荐镜头调度 | Prompt 转译方式 |
|---|---|---|
| 强调面部情绪 / 亲密感 | 长焦近拍、核心特写、大特写 | `close-up shot` 聚焦眼神或细微表情，背景自然虚化，镜头缓慢推进 |
| 展示人物与环境关系 | 广角远拍、固定广角、全景到特写 | 先用全景或中景建立空间，再缓慢推进到人物或产品 |
| 制造压迫 / 威胁 | 广角近拍、入侵角度、仰视推轨、固定仰拍 | 低机位或贴近主体，镜头向前逼近，前景物体被放大 |
| 表现速度 / 追逐 / 紧张 | 运动广角、运动的摄影机、传达速度、穿过人群 | 广角跟拍，前景快速掠过，镜头与主体同步移动 |
| 逐步揭示信息 | 揭示式运动、移动用作揭示、俯仰用作揭示、对角线揭示 | 镜头横移 / 后拉 / tilt up，从局部或遮挡后逐步露出关键信息 |
| 表现犹豫 / 决定 | 演员驱动摄影机、犹豫不决、短距离运动 | 摄影机随人物停顿而停顿，再随其转身或迈步恢复运动 |
| 表现孤立 / 疏离 | 固定长焦、穿过人群、打破多人场景、俯瞰后拉 | 主体被前景、人群或空旷环境包围，镜头保持克制或后拉 |
| 表现关系靠近 | 双重推进、吸引演员、对向滑动 | 两个主体从分离位置向画面中心靠近，镜头轻微推进 |
| 表现关系破裂 | 亲密关系中断、分开演员、旋转出画 | 原本同框的主体被走位或镜头后拉分开，构图留出明显间距 |
| 表现权力变化 | 权力交换、正反角度的高度、落至摄影机高度 | 通过高低机位、画面中心位置和主体大小变化表现主导权转换 |
| 增加空间层次 | 纵深演员调度、充实前景、窗框聚焦、镜面反射 | 前景 / 中景 / 背景都有动作或遮挡，镜头轻微横移增强深度 |
| 动作能量连续 | 动作转换、回到主体、暂停推进、转身剪辑 | 镜头跟随动作变化，在高潮前短暂停顿或回到主体表情 |

### 时长适配

- 3-6 秒：只选 1 个镜头调度技法，例如短距离推进、轻微横移、核心特写、演员驱动摄影机。
- 7-10 秒：可选 2-3 个镜头单元，按“建立空间 → 动作展示 → 情绪/产品特写”组合。
- 10-15 秒：可选 3-5 个镜头单元，但每个镜头只承担一个叙事目的。
- 口播视频：镜头运动服务台词节奏，优先使用缓慢推进、侧向跟拍、核心特写，少用复杂旋转和快速跳切。

### 复杂镜头降级规则

资料库中包含复杂旋转、跟拍长镜头、多人运动、假墙、镜面反射逆向运用等高级技法。用于 AI 视频 Prompt 时，除非用户明确要求电影感复杂调度，否则应降级为更稳定的表达：

| 原技法 | AI 视频稳定写法 |
|---|---|
| 复杂旋转 | 镜头围绕主体轻微弧线移动 |
| 跟拍长镜头 | 镜头稳定跟随人物向前移动 |
| 多人运动 | 人群作为背景流动，主体动作保持清晰 |
| 假墙 / 穿墙运动 | 镜头从遮挡物后横移露出主体 |
| 镜面反射逆向运用 | 镜头掠过反光表面，短暂出现反射层次 |

## 人物去待机感行为资料库

资料文件：`references/ANTI_IDLE_CHARACTER_BEHAVIOR.md`

本资料库用于降低 AI 视频人物的“待机感”。人物真实感不只来自五官和皮肤细节，更来自生活状态、行为动机、下意识反应和人物之间的反应关系。

### 调用时机

- 完整视频设计中，只要画面含人物，分镜脚本前必须补一次【人物行为动机设计】。
- 提示词优化中，若原 Prompt 只有“看镜头 / 微笑 / 站立 / 走路 / 展示产品”等泛动作，必须用本资料库补强。
- 图生视频中，仍优先遵守「首帧精简硬门」：只写首帧之后发生的小事、下意识反应、情绪落脚点和镜头变化，不复述首帧视觉事实。
- 多人物视频中，必须检查是否存在“一人行动，另一人反应”的先后关系，避免所有人物同时表演。

### 单人物公式

```
人物 + 正在做的小事 + 下意识反应 + 情绪落脚点
```

示例转译方向：不要只写“人物端着咖啡看镜头”，应写成“人物一边看向前方，一边无意识地转动咖啡杯，听到声音后眼神短暂偏移，表情带着赶时间的分神感”。

### 多人物公式

```
人物 A 行动 + 人物 B 反应 + 两者之间的情绪关系
```

示例转译方向：不要让两个人同时看镜头、同时微笑、同时做动作；应让一人先做具体行为，另一人产生停顿、看向、微笑、接话或伸手配合等反应。

### 输出禁区

最终可复制 Prompt 中不要写“去待机感”“行为动机设计”“人物太闲”等内部诊断词。只保留可执行的生活化小动作、微弱生命反应、情绪状态和互动反应。

## 六连击口播文案结构

| 段 | 时长 | 内容 |
|---|---|---|
| 1 钩子 | 0-3s | 痛点提问 / 反差陈述 |
| 2 人设 | 3-6s | "我也曾..." 共鸣 |
| 3 转折 | 6-12s | "直到我发现..." |
| 4 卖点 | 12-20s | 产品核心功能 + 视觉证据 |
| 5 信任 | 20-25s | 评价/数据/对比 |
| 6 行动 | 25-30s | CTA + 紧迫感 |

## 工作流程

### 模式 A：完整视频设计
```
读取产品信息 + 受众 + 平台 + 时长 + 风格
  ↓
（如涉及特定平台）调用 market-platform-analyzer 获取卡 A
  ↓
判断是否为图生视频：若用户上传图片，先进入「首帧精简硬门」
  ↓
调用「大师100镜头调度」资料库：按叙事目的选择 1-2 个镜头调度技法
  ↓
调用「人物去待机感行为」资料库：为每个含人物镜头补充小事、下意识反应、情绪落脚点；多人镜头补充行动与反应关系
  ↓
设计分镜脚本（按时长拆分场景）
  ↓
（如需）调用 ai-image-prompt-builder 生成静态首帧
  ↓
为每个分镜构造视频提示词（中文场景 + 英文台词）
  ↓
图生视频自检：是否只写动作、镜头、口播和新增变化；是否删除首帧人物/服装/背景复述
  ↓
生成口播文案（六连击或自定义）
  ↓
输出完整视频设计文档
```

### 模式 B：提示词优化
```
读取用户已有 Prompt
  ↓
诊断问题（图生视频是否复述首帧内容、缺镜头描述、缺镜头调度目的、台词不自然、动作不具体、人物是否只有待机式摆拍）
  ↓
调用「大师100镜头调度」资料库：补入与情绪/动作匹配的镜头调度
  ↓
调用「人物去待机感行为」资料库：把泛动作改写为有小事、下意识反应和情绪落脚点的生活化行为
  ↓
按混合格式重写
  ↓
若为可灵图生视频，压缩为单段可复制提示词，去除人物/服装/背景/光线复述
  ↓
输出对比 + 优化版
```

### 模式 C：口播文案
```
读取产品 + 受众 + 平台
  ↓
按六连击结构起草
  ↓
输出脚本 + 时间码
```

## 注意事项

- 中文场景描述 + 英文台词是可灵/Gemini 视频的最佳实践格式
- 视频提示词不要写太长，单分镜 80-150 字最佳
- 可灵 / Kling 图生视频优先遵守本 Skill `references/KLING_PROMPT_FORMAT.md`；历史模板仅作参考，凡与该规范冲突，以规范为准
- 单图首帧视频默认输出一段可直接复制到可灵的连续提示词，不加标题、`image 1` 说明、参数块或分析说明
- 含人物视频必须避免“待机式摆拍”：给人物安排正在做的小事、微弱生命反应和明确情绪落脚点；多人物镜头必须有行动与反应的先后关系
