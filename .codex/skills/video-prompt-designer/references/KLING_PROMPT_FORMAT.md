# 可灵 / Kling 视频提示词规范（video-prompt-designer Skill 内部）

> 本文件是 `video-prompt-designer` Skill 的内部规范。原位于 `.codex/rules/kling-prompt-output-format.mdc`（全局 alwaysApply rule），V8.x 起按"Skill 自包含原则"迁入本 Skill 的 `references/`，不再随每次对话强制加载。
>
> **何时读取本文件**：`video-prompt-designer` 输出最终视频 Prompt 时（特别是可灵 / Kling / 图生视频场景）必读，详见本 Skill `SKILL.md` 的「视频提示词输出格式」与「工作流程」章节。

## 触发条件

以下任一条件满足时，本规范生效：

- 用户请求生成「可灵」「Kling」视频提示词
- 用户提到「视频提示词」「短视频」「分镜」「口播」「TikTok」「Reels」「小红书视频」
- `video-prompt-designer` 输出最终视频 Prompt

## 一、图生视频首帧精简硬门

当用户上传了 1 张或多张图片用于图生视频时，图片已经是模型可见的首帧或参考图。最终可复制 Prompt 只写：

1. 动作：人物、产品、物体接下来发生什么
2. 镜头：推进、拉回、跳切、特写、轻微横移等
3. 口播：仅当用户明确要求口播或提供台词时写
4. 新变化：图中不存在、但用户明确要求发生的变化

最终 Prompt 中不得复述图片里已有的信息：

| 已在图片中可见的信息 | 最终 Prompt 写法 |
|---|---|
| 人物外貌、发型、身材 | 只写「人物」和动作 |
| 服装颜色、款式、搭配 | 只写「展示整体穿搭」「整理衣摆/外套」等动作 |
| 场景、背景、家具、地点 | 只写构图延续或镜头动作 |
| 光线、色调、氛围 | 由首帧决定，最终 Prompt 不复述 |
| `image 1` / 参考图角色说明 | 可灵最终 Prompt 不写 `image N` |

> 原则：不要用文字覆盖首帧。可灵图生视频里，首帧负责视觉事实，Prompt 只负责动作和镜头。

## 二、模型与口播判断

| 用户上传图数量 | 推荐模型 | Prompt 策略 |
|---|---|---|
| 1 张 | 可灵 3.0 标准版 | 该图作为起始帧，单段动作 + 镜头描述 |
| 2 张及以上 | 可灵 3.0 Omini | 多图作为参考，仍不复述图片内容 |

| 用户意图 | 输出模式 |
|---|---|
| 未提及口播 / 未提供台词 | 纯展示，只写动作 + 镜头 |
| 明确提及口播 / 提供台词 | 展示 + 口播，使用 `说"English line"` 格式 |

## 三、语言规则

- 动作、镜头、场景变化：中文
- 口播台词：默认英文，用户明确要求中文时才写中文
- 可保留少量镜头术语：`medium shot`、`close-up shot`、`dolly in`、`pan left`

正确：

```
人物轻轻侧身展示整体效果，镜头推进至中景，抬眼看向手机屏幕并自然微笑。
```

正确（含口播）：

```
人物整理衣摆并看向镜头说"This outfit feels so effortless"，镜头推进至中景，随后轻轻侧身展示整体效果。
```

错误：

```
金发女性穿棕色西装外套和白色长裤，站在温暖室内镜前，柔和自然光，高级生活方式氛围，参考 image 1。
```

问题：复述了首帧中的人物、服装、环境、光线，并写入 `image 1` 元信息。

## 四、分镜模式

### 镜头与内容融合写法（强制）

无论是智能分镜还是自定义分镜，**镜头描述必须与该镜头内的动作内容写在一起**，写完一个镜头单元（景别/角度/运动 + 该镜头内的动作）再写下一个镜头单元。**禁止**把所有镜头节奏单独罗列在最后一句，或把动作和镜头变化拆成两个独立段落。

错误（镜头与动作分离）：

```
人物视线落向窗外，左手拨开发丝，重心后靠。人物提杯轻啜，眼神低垂带笑。
镜头采用跳切节奏：正面中景、侧面三分之二、全景斜后侧、正面中近景。
```

正确（每个镜头单元自成一句，镜头 + 该镜头内的动作）：

```
镜头1 中景，人物视线落向窗外车流，左手轻轻拨开耳侧发丝。跳切镜头2 侧面三分之二角，人物提起咖啡杯凑近唇边轻啜，眼神低垂带笑意。
```

### 智能分镜（3-6 秒优先）

适合 3-6 秒、单一动作流的短视频。用一段连续文字自然嵌入 1-2 次镜头变化。

正确示例（6 秒 OOTD 纯展示）：

```
保持镜前自拍构图，人物轻轻侧身展示整体穿搭，抬手整理外套前襟，重心自然从一侧换到另一侧，镜头轻微推进至中景，人物抬眼看向手机屏幕并自然微笑，动作慢速、真实、流畅，衣料和头发随身体动作轻微摆动。
```

### 自定义分镜（≥ 7 秒优先 / 跳切节奏强制）

适合 ≥ 7 秒视频或用户明确要求跳切的片段。每个镜头单元写"景别/角度/运动 + 该镜头内的动作"，串联成连续提示词。

正确示例（10 秒跳切纯展示）：

```
镜头1 中景，人物视线落向窗外车流，左手轻轻拨开耳侧发丝，重心微微后靠陷进椅背。跳切镜头2 侧面三分之二角，人物双手提起咖啡杯凑近唇边小口轻啜，眼神低垂带笑意。跳切镜头3 全景斜后侧，人物放下杯子，单手沿腰侧顺势整理上衣下摆，双腿交叠换边。跳切镜头4 正面中近景，人物侧身轻轻转向窗户，抬眼望向远处随后低头看向手机屏幕，发丝随转身轻轻扫过肩膀。
```

## 五、时长规则与信息密度

### 时长上限

| 类型 | 单段上限 | 建议范围 |
|---|---|---|
| 纯展示 | 15 秒以内 | 3-10 秒 |
| 含口播 | 10 秒以内 | 3-10 秒 |

### 信息密度对齐参考表（强制）

可灵 / Kling 模型会把 prompt 里的动作节点和镜头切换**自动展开到指定时长**。当信息密度不足时，模型会用慢放、停顿、无意义微动来填满时间——这是导致"看似一切都在动但什么都没发生"的废片感的根因。

写完 prompt 后必须做一次密度自检，按下表对齐：

| 视频时长 | 镜头数 | 动作节点数 | 中文 Prompt 字数 | 推荐分镜模式 |
|---|---|---|---|---|
| 3 秒 | 1 | 1-2 | 40-80 | 智能分镜 |
| 5 秒 | 1-2 | 2-3 | 60-120 | 智能分镜 |
| 8 秒 | 2-3 | 4-5 | 100-180 | 自定义分镜 |
| 10 秒 | 3-4 | 5-7 | 140-240 | 自定义分镜 |
| 15 秒 | 4-5 | 7-10 | 200-320 | 自定义分镜 |

> "动作节点"指一个可被肉眼识别的独立动作单元（如"提杯轻啜"是 1 个节点；"低头看手机然后微笑"是 2 个节点）。
> 跳切节奏可在同等时长下塞更多镜头数，但单个镜头内部动作要更克制（每个镜头 1-2 个动作节点即可）。

### 节奏锁定句（≥ 7 秒视频强制末尾追加）

≥ 7 秒视频在最后追加一句明确节奏，防止模型自动慢放填充：

- 跳切型：`整体节奏快速干脆，每个镜头 X-X 秒，动作实拍质感，无停顿无慢放。`（X 由总时长除以镜头数推算）
- 连续型：`动作节奏自然流畅，无停顿无慢放，整体保持实拍真实感。`

### 含口播时长反推

含口播时先写台词，再按词数反推时长：

- 快速语速：英文词数 ÷ 3.3
- 极速语速：英文词数 ÷ 4.0

## 六、最终输出格式

给用户的最终可复制 Prompt 默认只输出一段正文。不要在正文前后添加：

- 标题
- 参数块
- 参考图目录
- `image 1` 角色说明
- 分析说明
- 负向提示词
- `[NEGATIVE]`、`Negative Prompt`

如果用户明确要求"解释原因 / 给多个版本 / 分镜卡片"，可以在 Prompt 外做说明，但必须清楚区分"可复制 Prompt 正文"。

## 七、禁止清单

### 7.1 内容/格式禁止

- 在图生视频 Prompt 中描述首帧/参考图已有的人物、服装、场景、背景、光线、色调
- 使用 `image 1` / `image 2` / `参考图中的...` 作为可灵最终 Prompt 的正文
- 将图生图 Prompt 写法套用到图生视频 Prompt
- 用户未提口播时主动添加 `说"..."`
- 动作/镜头描述大段英文
- 输出负向提示词或自然语言负向约束
- 使用 `cinematic`、`8k`、`Unreal Engine`、`CGI`、`perfect body`、`bodybuilder`、`flawless figure`

### 7.2 无效元描述禁止（不喂给模型的"自言自语"）

最终 Prompt 中**禁止**出现以下三类元描述——它们对视频模型没有视觉指令意义，只会浪费 token 并稀释真正的动作/镜头权重：

| 类别 | 错误示例 | 处置 |
|---|---|---|
| 视频类型/风格自述 | `10 秒抓拍感短视频`、`一段日常 vlog 风格视频`、`这是一支 OOTD 短片` | 删除。视频时长由用户在可灵参数里设置，"抓拍感"应转为具体动作（如 `动作随机自然，眼神不看镜头`） |
| 首帧延续声明 | `人物自然坐姿延续首帧`、`画面延续上传图片`、`保持首帧构图`（孤立成句时） | 删除。可灵图生视频里"延续首帧"是默认行为，无需声明；如需明确锁定构图，改写为具体动作（如 `保持镜前自拍构图，人物轻轻侧身`） |
| 模型/参数/任务自述 | `使用可灵 3.0 标准版生成`、`本视频用于 TikTok 投放`、`目标受众为欧美女性` | 删除。这些信息属于上下文，不进入最终可复制 Prompt |

判断方法：把句子从 Prompt 中删掉后，**对模型的视觉输出有任何影响吗**？没有 → 该句是无效元描述，必须删除。

### 7.3 镜头与内容分离禁止

禁止把所有镜头变化单独罗列成一句（如 `镜头采用跳切：正面中景、侧面、全景`），必须按"镜头单元 + 该镜头内动作"融合写法逐个串联（详见第四节）。

## 八、输出前自查

最终 Prompt 交付前逐项检查：

- [ ] 用户上传图时，Prompt 是否只写动作、镜头、口播、新变化
- [ ] 是否删除了人物外貌、服装款式、场景背景、光线氛围复述
- [ ] 是否没有 `image N`、参考图目录、标题、参数块、分析说明
- [ ] 用户未要求口播时，是否没有 `说"..."`
- [ ] 是否无任何无效元描述（视频类型自述 / 首帧延续声明 / 模型参数自述）
- [ ] 镜头与内容是否按"镜头单元 + 该镜头内动作"融合写法逐个串联
- [ ] **信息密度自检**：数一下镜头数和动作节点数,是否对齐第五节参考表？字数是否在区间内？
- [ ] **节奏锁定句**：≥ 7 秒视频是否在末尾追加节奏锁定句？
- [ ] 动作和镜头是否清晰、短促、可执行
- [ ] 是否无负向提示词和禁用词
