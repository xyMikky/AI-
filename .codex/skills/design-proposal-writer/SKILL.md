# 设计提案与汇报（Design Proposal Writer）

## 概述

设计方案的提案文档撰写专家。前身是能力模块 M7_设计提案与汇报。

## 边界

- 仅做**提案文档**输出
- 不做实际设计执行
- 不替代演讲/PPT 美术设计（执行设计 → 对应 Skill）

## 工作流程

```
Step 1：背景洞察（行业现状/品牌现状/竞争格局）
Step 2：问题定义（核心痛点 + 设计能解决的部分）
Step 3：设计策略（方向选择 + 决策依据）
Step 4：视觉方案展示（Logo/配色/排版/物料示例）
  ├─ 可调用 design-relation-visualizer 生成关系图谱嵌入
  └─ 可调用已完成的设计稿截图引用
Step 5：ROI 论证（预期效果 + 量化指标）
Step 6：推进计划（时间表 + 关键节点 + 协作方式）
```

## 提案标准结构（10 部分）

1. 封面（项目名 + 客户 + 日期）
2. 议程
3. 背景洞察
4. 问题定义
5. 设计策略
6. 视觉方案
7. 应用场景
8. ROI 论证
9. 推进计划
10. 致谢与 Q&A

## 与其他 Skill 的关系

| Skill | 关系 |
|---|---|
| `brand-logo-designer` | 协同：品牌焕新提案常组合 |
| `design-relation-visualizer` | 工具：生成可视化嵌入提案 |
| `non-ai-design-reviewer` | 协同：方案展示前可先审 |

## 输入输出

**输入**：项目背景 + 客户/受众 + 已完成设计方案

**输出**：完整提案文档（Markdown / PPT 大纲 / 汇报稿）

## 注意事项

- 客户提案语气需正式、有逻辑、量化论证
- 详细模板见 `归档/v7.9/能力模块/M7_设计提案与汇报.txt`
