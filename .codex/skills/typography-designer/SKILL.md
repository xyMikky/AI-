# 排版与字体设计（Typography Designer）

## 概述

排版字体设计的双模式 Skill。前身是能力模块 M3_排版与字体设计。

## 双模式触发契约

| 模式 | 触发信号 | 工作流 | 主产出 |
|---|---|---|---|
| **独立模式** | "字体规范/字体层级/排版规范/中英文混排方案" | Step 1-5 | 完整字体层级规范（中/英/数 × 数字/印刷）|
| **Step 0 协同** | 被生图 Skill 声明协同 + 画面含文字 | Step 0 轻量 | 【排版决策卡 E】|

## Step 0 协同必读卡 D

协同模式下，**必须先读取 color-system-designer 输出的【配色决策卡 D】**才能确定文字色值。

## 独立模式（Step 1-5）

```
Step 1：行业/品牌调性诊断
Step 2：字体调研（中文/英文/数字字体方向）
Step 3：主字族 + 字重选型（如 Inter Variable / 思源黑体 / Gilroy）
Step 4：字号层级（H1/H2/H3/Body/Caption × 移动端/桌面端/印刷端 三套尺寸）
Step 5：排版黄金规则（行高/字间距/段落间距/中英文混排间隔）
```

## Step 0 协同模式（轻量交接卡）

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【排版决策卡 E】　typography-designer 协同输出
• 排版来源：[品牌字体规范 / 用户指定 / 调性推导]
• 字体方案：中文 [字体] / 英文 [字体]
• 字重层级：H1 [字重] / H2 [字重] / Body [字重] / Badge [字重]
• 文字色值：主 #XXXXXX / 辅 #XXXXXX / 强调 #XXXXXX（来自卡 D）
• Prompt 排版关键词：[英文关键词，如 modern sans-serif, bold weighted, condensed]
• 品牌一致性：✅ / ⚠️ / N/A
• 下游衔接：→ ⑦ Prompt 构造
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 独立 → 协同桥接

- 主字族 + 字重 → 卡 E 的 `font_family` / `weight`
- H1/H2/Body 字号级差 → 卡 E 的 `hierarchy`
- 排版黄金规则 → 卡 E 的 `typography_rules`

## 与其他 Skill 的关系

| Skill | 关系 |
|---|---|
| `color-system-designer` | 协同：先读卡 D 取文字色值 |
| `ai-image-prompt-builder` | 下游消费者（Step 0 模式）|
| `detail-page-designer` | 下游消费者（Step 0 模式）|
| `print-poster-designer` | 协同：印刷物料独立模式 |
| `ui-ux-designer` | 协同：UI 字体规范 |
| `brand-spec-learner` | 数据源：品牌字体加载 |

## 输入输出

**输入**：调性 + 中英文需求 + 输出端（数字/印刷）+ 卡 D（如协同）

**输出**：
- 独立：完整字体层级规范
- 协同：【排版决策卡 E】

## 注意事项

- 协同模式必须**先读卡 D** 才能产卡 E
- 模块不做二次仲裁
- 详细 Step 1-5 框架见 `归档/v7.9/能力模块/M3_排版与字体设计.txt`
