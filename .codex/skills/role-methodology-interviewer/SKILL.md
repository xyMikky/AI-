# 角色方法论访谈萃取师（Role Methodology Interviewer）

## 概述

把"现实中某个岗位的人怎么干活"这件隐性知识，通过**大量提问**显性化、结构化，沉淀成 AI 可调用的角色方法论知识。

这是给"角色层"补课的 Skill：系统已按**任务**路由（设计/生图/详情页…），但缺"**角色怎么思考和决策**"的知识。本 Skill 通过访谈真人专家填补这一层，产出未来可升级为「能力线业务 Skill」（产品企划/供应链成本/运营上市等）的原料。

**核心理念**：专家的价值在"判断标准"和"隐性经验"，不在"流程步骤"。流程能查到，判断标准只能问出来。

但**纯开放式提问**让被访谈者"对着空白作答"，回忆慢、容易卡壳、答得零散，逐题打字效率极低。因此本 Skill 采用**选项辅助式访谈**：

> **每一个问题都用 `request_user_input` 选项卡呈现**，AI 基于角色 / 行业 / 品牌方法论**预生成 4-8 个候选答案**作为选项，并**始终附带「不清楚 / 不确定」+「其他（请补充）」两个兜底项**——让被访谈者以"勾选 + 少量补充"的方式快速作答，再由 AI 对所选项做钻取追问把深度补回来。
>
> **选项负责提速和唤醒记忆，「其他」+ 追问负责保留深度和个性。** 候选选项只是"可能的答案"供选择，不是替用户拍板——用户勾了才算数，没勾的选项不得写入萃取结果。

## 使用场景

| 用户语义 | 是否激活 |
|---|---|
| "我想访谈一下我们的采购，把他的方法论记下来" | ✅ |
| "帮我设计一套问题问运营怎么做推广" | ✅ |
| "把质检员的经验萃取成知识" | ✅ |
| "我找现实中的角色聊，你帮我提问汇总" | ✅ |
| "帮我做个海报" | ❌ → 设计类 Skill |
| "学习这个品牌的素材" | ❌ → brand-spec-learner |

## 与其他 Skill 的边界

| Skill | 边界 |
|---|---|
| `design-requirement-guide` | 那个是问"用户这次想要什么设计"；本 Skill 是问"某个岗位平时怎么工作"。一次性 vs 知识沉淀 |
| `brand-spec-learner` | 那个学的是已有素材文件；本 Skill 萃取的是人脑里没成文的方法论 |
| `ai-persona-architect` | 那个用萃取出的方法论去**造 Skill**；本 Skill 负责**产原料** |
| 设计/生图类 | 本 Skill 不执行设计，只沉淀方法论 |

## 模式约定（重要 · 自动切 Plan/Ask 只读模式）

### 关键机制

- 在 Codex 中，**Plan 模式即只读的 Ask 模式**——只能提问、读取，不能写文件，正好适合访谈。
- **Agent 可以自动切入**：激活本 Skill 后，**第一件事就是调用 `SwitchMode → plan`**，无需用户手动切。这与 `design-requirement-guide` 的做法一致。
- 落盘时再调用 `SwitchMode → agent` 切回，写入知识库文件。

### Step 0：自动切到 Plan（Ask）模式

激活本 Skill 后立即执行：

```
SwitchMode → plan   （explanation：进入只读访谈模式，纯提问不写文件）
```

切换后简单说明一句，再进入 Step 1 开始提问：

```
已切到 Plan（Ask 只读）模式开始访谈，过程只提问不改文件。
访谈结束、你确认萃取摘要后，我会自动切回 Agent 模式落盘。
```

### 两阶段模式映射

| 阶段 | 模式 | 切换方式 | 说明 |
|---|---|---|---|
| 访谈（Step 1-4） | **Plan / Ask（只读）** | `SwitchMode → plan`（自动） | 只提问、追问、复述确认，不写文件 |
| 落盘（Step 5） | **Agent** | `SwitchMode → agent`（自动） | 写入知识库文件 |

### 容错与例外

- **用户已在 Plan / Ask 模式**：跳过切换，直接进入 Step 1。
- **用户明确要求留在 Agent 模式 / 边聊边记**：尊重选择，不强切；访谈期间**仍只提问、不写文件、不臆造答案**，结束后直接落盘。
- **切换需用户确认（Plan/Agent 切换需经用户同意）**：若用户拒绝切换，按上一条留在当前模式继续。
- ⚠️ 绝不替被访谈者编造答案。**预生成的候选选项是"供选择的可能答案"，不算编造；但未被用户勾选的选项一律不得当作答案写入。** 用户没回答的维度标注"待补充"，不得虚构经验或数据。

## 选项辅助式提问规范（核心机制 · 全程强制）

本 Skill 的每一次提问，无论是分叉卡（Step 1）还是方法论深挖（Step 3），都**必须**用 `request_user_input` 选项卡承载。规则如下：

### 1. 每题结构：候选选项 + 「不清楚」+「其他」双兜底

- 每个 question 必须给 **4-8 个候选选项**，覆盖该问题在该角色 / 行业里的常见答案分布。
- **每题末尾必带两个兜底项**（缺一不可）：
  - **「不清楚 / 不确定」**：这块不归我管 / 我确实不掌握。给被访谈者一个**合法的"我不知道"出口**，避免不掌握的人被迫硬编、污染方法论结果。
  - **「其他（请补充）」**：我有答案，但不在上面选项里。
- 两者语义不同：「其他」= 有答案但没覆盖，**要追问拿内容**；「不清楚」= 没有答案，**不硬逼、标待补充**（见第 6 条）。
- 一个 `request_user_input` 调用可一次抛 **1-3 个 question**（按维度成组），避免一次性轰炸。

### 2. 选项从哪来（按优先级生成）

1. `references/QUESTION_BANK.md` 里该问题的**候选选项库**（已为高频问题预置）
2. 该角色专属题组 + `品牌规范/[品牌]/业务方法论/` 里的流程 / 工作法
3. 该岗位的行业常识（如运营常看的指标、设计师常见卖点优先级）
4. 上一题用户的回答（动态生成下一题选项，做到"顺着答案走"）

### 3. 不同题型的选项设计

| 题型 | 选项怎么给 | 是否多选 |
|---|---|---|
| 流程 / 步骤类（D3） | 列出典型步骤，让用户勾选+排序补充 | ✅ 多选 |
| 判断标准 / 取舍类（D4） | 列出常见判断依据 + 「凭经验说不清」+ 其他 | 视情况 |
| 数值 / 阈值类 | 给区间档位（如"1-2 天 / 3-5 天 / 一周以上"）+「说不准」+ 其他 | 单选 |
| 工具 / 指标类（D5） | 列常用工具 / 指标 + 其他 | ✅ 多选 |
| 坑 / 异常类（D7） | 列高频坑 + 其他 | ✅ 多选 |
| 隐性经验 / 直觉类（D9） | 给几个候选方向引导回忆，**强制配「其他」让其展开** | 视情况 |

### 4. 追问也用选项卡（深度补偿）

选项提速会牺牲深度，靠**追问把深度补回来**——而追问同样用选项卡：

- **钻取追问**：用户选了某判断标准 → 下一张卡问"为什么是这个标准"，给几个候选原因 + 其他。
- **量化追问**：用户选了模糊档 → 下一张卡给更细的数值区间逼成具体数字。
- **反例法**：把"什么情况你会不这么做"做成选项卡，列常见破例场景 + 其他。
- 凡涉及"判断标准（D4）"和"隐性经验（D9）"，**至少追加一轮选项卡追问或强制走「其他」展开**，不能只停在第一层勾选。

### 5. 「不清楚」的处理（防污染核心）

被访谈者不一定掌握每一个维度（如设计师对供应链账期、采购对广告指标）。**强制硬逼回答 = 让人编 = 污染方法论**。因此：

- 用户选「不清楚 / 不确定」时，**不追问、不诱导、不换个说法再逼**，直接跳到下一题。
- 该维度 / 该问题在萃取摘要中标注 **"待补充（对方不掌握）"**，**绝不**用预设选项或 AI 推测填充。
- 若某维度被访谈者整体选「不清楚」，说明这块该问别的角色——在摘要里记录"建议改访谈 [更对口的角色]"。
- 「不清楚」是合法答案，不是失败；它本身就是有价值的信息（界定了这个角色的知识边界 D1）。

### 6. 红线

- **选「其他」必须追问**，拿到真实内容再记录，不得用任何预设选项替代用户没说的话。
- **选「不清楚」不得追问硬逼，更不得替对方填答案**——标"待补充"即可。
- 用户勾了才算数；**没勾的选项不得写入萃取摘要**。
- 选项措辞保持中立，不诱导；不得把"AI 觉得对的答案"做成唯一像样的选项。

## Step 1：锁定对象与范围（用 request_user_input）

用 `request_user_input` 选项卡分叉，三问（注意：选项卡是全程提问方式，不止此阶段）：

1. **访谈哪个角色？**（选项给出 14 个新品流程角色 + "其他"）
2. **本场深度？**
   - Quick：只补充某一个点（1 维 / 几个问题）
   - Standard：单角色单场，9 维走一遍
   - Deep：多场分阶段深访，含关键事件回溯
3. **优先聚焦维度？**（9 维多选，默认全选）

## Step 2-3：选项辅助式访谈（核心）

> 提问方式严格遵循上文「选项辅助式提问规范」：每题一张 `request_user_input` 选项卡（4-8 候选 + 「不清楚」+「其他」），勾选为主、补充为辅；判断标准 / 隐性经验类必追问；选「不清楚」不硬逼、标待补充。

### 9 维萃取框架

| 维度 | 要挖到什么 |
|---|---|
| D1 角色边界 | 职责范围、不负责什么、和谁的边界最容易混 |
| D2 触发与输入 | 什么事/什么信号启动他的工作、需要哪些前置信息 |
| D3 标准流程 | 一件典型任务从头到尾的真实步骤（不是理想流程） |
| D4 决策判断 | **每个步骤靠什么标准做决定**（核心金矿） |
| D5 工具与数据 | 用什么系统/表格/数据源、看哪些指标 |
| D6 协作与交接 | 上游给什么、交给下游什么、接口处常出什么问题 |
| D7 异常与坑 | 最容易翻车的地方、踩过的坑、怎么补救 |
| D8 衡量标准 | 他的工作好不好由什么衡量、KPI、被骂最多的点 |
| D9 隐性经验 | 老手和新手的差距、说不清但很关键的直觉 |

> 详细每维 5-10 个问题 + 14 角色专属题组见 `references/QUESTION_BANK.md`。

### 6 种萃取技巧（提问时必须混用 · 全部用选项卡承载）

1. **关键事件法**：把"挑一个最近真实案例"做成选项卡——选项列出几个可能的典型场景让用户认领（如"最近一次主图设计 / 一次详情页 / 一次返工"）+ 其他，认领后再顺着这一单逐题深挖。
2. **钻取追问（why-why）**：用户勾了某判断标准后，下一张卡问"为什么是这个标准"，给候选原因 + 其他；连追 2-3 层逼出底层逻辑。
3. **反例法**：把"什么情况下你会**不**这么做 / 这条规则什么时候破例"做成选项卡，列常见破例场景 + 其他。
4. **量化追问**：模糊档位 → 用选项卡给更细的数值区间逼成具体数字（"差不多"→给"1-2 / 3-5 / 5 以上"档；"快"→给天数档）。
5. **教学法**：把"带新人先教什么 / 最怕他犯什么错"做成多选选项卡，列候选要点 + 其他，逼出优先级排序。
6. **复述确认**：每 1-2 维把听到的内容复述一遍，用一张"是否准确"的选项卡（准确 / 需修正某项 + 其他）让用户校正。

### 提问纪律

- **每题必用 `request_user_input` 选项卡，必带候选选项 + 「不清楚 / 不确定」+「其他（请补充）」**，禁止用纯文字段落让用户对着空白逐条打字。
- **不强制回答**：被访谈者不掌握的维度选「不清楚」即可，不硬逼、不替填，标"待补充"，避免硬编污染结果。
- 一次抛 1-3 题（一个 request_user_input 内 1-3 个 question），避免一次性轰炸。
- 选项是"候选答案"供选择，**不是替用户拍板**；用户勾了才算数，没勾的不写入萃取结果。
- 选「其他」必须追问拿到真实内容再记录，不得用预设选项替代用户没说的话。
- 判断标准（D4）、隐性经验（D9）类**不能只停在第一层勾选**，至少追加一轮选项卡追问或强制走「其他」展开。
- 用户答不上来的，标"待补充"，不替他想。
- 跟着回答走——若用户在"其他"里提到新坑 / 新流程，临时插入追问，不死守题序。

## Step 5：落盘规范

- 用户确认萃取摘要后，调用 `SwitchMode → agent` 切回写入模式。
- 输出文件：`品牌规范/[当前品牌]/业务方法论/角色方法论/[角色].md`
  - 无激活品牌时，写入 `参考库/K_业务流程/角色方法论/[角色].md`（无则创建）
- 按 `templates/methodology_output_template.md` 结构填充。
- 增量访谈：同角色已有文件则**追加/更新**对应维度，保留访谈日期与版本，不覆盖历史。

## 三档深度对照

| 深度 | 场次 | 覆盖 | 适用 |
|---|---|---|---|
| Quick | 1 场 / 几问 | 1 维补充 | 已有文件，补某个坑/某条标准 |
| Standard | 1 场 | 9 维各 3-5 问 | 第一次系统访谈一个角色 |
| Deep | 多场 | 9 维深挖 + 多个关键事件 + 跨阶段交接 | 核心角色（产品工程师/采购/运营）建权威 SOP |

## 注意事项

- 本 Skill 的产物是**给 AI 调用的知识**，写法要"可判断、可执行、可量化"，不是访谈速记流水账。
- **全程用 `request_user_input` 选项卡（候选选项 + 「不清楚」+「其他」）提问**，让被访谈者勾选为主、补充为辅，避免逐题打字的低效；深度靠对所选项的追问补回（见「选项辅助式提问规范」）。
- **不强制回答**：「不清楚」是合法答案，对方不掌握的维度标"待补充"，绝不替填——硬逼会让人编、污染方法论。
- 隐性经验（D9）和判断标准（D4）是最高价值，访谈时间应向这两维倾斜，且必须追问到底、不能只停在勾选。
- 萃取出的多角色方法论，后续可交给 `ai-persona-architect` 升级为能力线业务 Skill。
- M13（需求引导）豁免：本 Skill 自带提问流程，不再前置 design-requirement-guide。
