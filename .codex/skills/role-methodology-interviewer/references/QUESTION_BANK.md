# 角色方法论访谈题库（Question Bank）

> 本文件是 `role-methodology-interviewer` 的访谈弹药库。
> 分四部分：① 9 维通用框架题 ② 6 种萃取技巧的提问句式 ③ 14 个新品流程角色的专属题组 ④ 候选选项库。
> 访谈时：通用框架打底，混用萃取技巧追问，叠加角色专属题。

---

## ⚡ 提问方式（强制 · 全程选项卡）

> 详见 `SKILL.md`「选项辅助式提问规范」。本题库里所有问题，**都必须转成 `request_user_input` 选项卡再问**，而不是直接把问句甩给用户打字。

把题库里的一道问题转成选项卡的标准做法：

1. 取问题作为 `prompt`。
2. 从「第四部分 · 候选选项库」对应维度取候选项，再叠加**该角色 / 行业 / 品牌方法论**的具体答案，凑齐 **4-8 个 option**。
3. **末尾固定加两个兜底 option**：「不清楚 / 不确定」+「其他（请补充）」。
4. 多选场景设 `allow_multiple: true`（流程步骤 / 工具 / 指标 / 坑 类）。
5. 用户选完后，对判断标准 / 隐性经验类**再追加一张选项卡追问**（钻取 / 量化 / 反例），不能只停在第一层；但选「不清楚」时**不追问、不硬逼**，标"待补充"跳过。

> 选项是"候选答案"供唤醒记忆和提速，**不是替用户拍板**；用户没勾的选项不得写入萃取结果。
> 「其他」= 有答案没覆盖，**必须追问拿真实内容**；「不清楚」= 对方不掌握，**不硬逼、不替填、标待补充**（防止硬编污染方法论）。

## 第一部分 · 9 维通用框架题

### D1 角色边界

- 你这个岗位，一句话概括是负责什么？
- 哪些事**不归你管**但别人经常以为归你管？
- 你和上下游谁的职责边界最容易扯皮？通常怎么界定？
- 一天/一周里，你的时间大致怎么分配？

### D2 触发与输入

- 什么事情发生，会启动你的工作？（订单？通知？某个节点？）
- 开工前你必须先拿到哪些信息/材料？缺哪样会卡住？
- 信息从谁那里来？格式是什么（系统/表格/口头）？
- 你怎么判断"可以开始了"还是"条件还不够"？

### D3 标准流程

- 一件最典型的任务，从你接手到交付，真实走哪几步？
- 哪几步是必须的，哪几步实际经常被跳过？
- 顺利的话多久完成？通常卡在哪一步最久？
- 理想流程和你实际干的，差在哪？

### D4 决策判断（核心金矿）

- 第 X 步你要做个决定，你**靠什么标准**判断该选 A 还是 B？
- 这个标准是写在制度里的，还是你自己摸出来的？
- 有没有一个数字阈值？多少以上/以下你就会怎么做？
- 拿不准的时候，你看哪几个信号来拍板？
- 你和新手在这一步的判断，最大差别是什么？

### D5 工具与数据

- 你每天离不开哪些系统/表格/工具？
- 你最常看哪几个数据/指标？看到什么数值会警觉？
- 有没有你自己维护的私人表格/清单/模板？里面记什么？
- 哪些信息现在只能靠脑子记，没有系统支持？

### D6 协作与交接

- 上游交给你的东西，理想状态应该长什么样？经常缺什么？
- 你交给下游的东西，对方最在意什么？
- 交接环节最容易出问题的地方是哪？怎么提前规避？
- 跨部门拉通时，你一般找谁、用什么方式最有效？

### D7 异常与坑

- 这个岗位最容易翻车的是哪几件事？
- 讲一次你印象最深的"出问题"经历，后来怎么补救的？
- 哪些坑是新人几乎一定会踩的？
- 有没有"看起来没事、其实快出事"的预警信号？

### D8 衡量标准

- 你的工作干得好不好，老板/公司用什么衡量？
- 你被挑毛病最多的是哪方面？
- 你自己心里的"做到位"标准是什么？（可能比 KPI 更高）
- 什么结果出现，你会觉得这次做砸了？

### D9 隐性经验（最高价值）

- 干这行最关键、但最难教给别人的能力是什么？
- 有没有一些"说不清为什么、但就是这么判断准"的直觉？
- 老手和新手做同一件事，差距主要体现在哪？
- 如果只能给后来人留 3 句话，你会留什么？

## 第二部分 · 6 种萃取技巧的提问句式

| 技巧 | 触发时机 | 句式范例 | 选项卡怎么做 |
|---|---|---|---|
| 关键事件法 | 对方答得太抽象 | "挑一个你最近真实处理过的，从头讲到尾。" | 选项列几个可认领的典型场景（最近一单 / 一次返工 / 一次大改）+ 其他 |
| 钻取追问 | 出现一个判断/标准 | "为什么是这个数？" → "那这个又是怎么定的？" | 给候选原因 4-6 个 + 其他，连追 2-3 层 |
| 反例法 | 一条规则说得太绝对 | "什么情况下你会**不**这么做？" | 列常见破例场景 + 其他 |
| 量化追问 | 出现模糊词 | "差不多=具体多少？""快=几天？" | 给数值区间档位（1-2 / 3-5 / 5 以上）+「说不准」+ 其他 |
| 教学法 | 想逼出优先级 | "带新人你先教什么？最怕他犯哪个错？" | 多选列候选要点 + 其他 |
| 复述确认 | 每 1-2 维结束 | "我复述一下你看对不对：……" | 选项给"准确 / 需修正某项"+ 其他 |

> 模糊词黑名单（听到必上量化选项卡）：差不多、大概、比较快、还行、看情况、一般来说、应该、尽量。

## 第三部分 · 14 角色专属题组

> 角色映射见 `品牌规范/星驰/业务方法论/新品开发到上市全流程.md`。每组 = 通用 9 维之外的"行业特定追问"。

### 1. 产品工程师（阶段1-2）
- 你怎么把一个模糊需求变成可落地的产品定义？
- 产品草图阶段，你最先确定哪几个参数？
- 怎么判断一个需求"值得做"还是"伪需求"？
- 结构/材料/功能取舍时，你的优先级排序是什么？

### 2. 专利工程师（阶段1）
- 专利排查你查哪些库、用什么关键词策略？
- 怎么判断"有侵权风险"到"必须规避"的程度分级？
- 外观/结构/功能专利，哪类最容易踩？
- 给产品团队的风险提示，你怎么说他们才会重视？

### 3. 包装设计师（阶段3/6）
- 包装方案评估你看哪几个硬指标（成本/结构/合规/运输）？
- 量产样和产前样确认，你重点盯什么？
- 包装最容易在哪个环节出问题（打样/量产/运输）？
- 成本和视觉冲突时怎么取舍？

### 4. 商务（阶段3-5）
- 供应商确认你看哪些条件？怎么排序？
- 报价里哪几项最容易被低估/漏算？
- 和供应商谈判，你的底线和让步顺序怎么定？
- 开票/合规这块，哪些坑最致命？

### 5. 采购（阶段3/5/6）
- 选供应商你的评分维度和权重是什么？
- 采购订单下单时机怎么定（库存/账期/MOQ）？
- 退料/退货发生时，你的处理流程和止损原则？
- 怎么判断一个供应商"能长期合作"？

### 6. 原材料仓仓管（阶段6-7）
- 收料验收你检查哪几项？什么情况拒收？
- 库存怎么盘、怎么防呆防错？
- 发料给生产，怎么保证不发错/不缺料？
- 最容易出账实不符的环节在哪？

### 7. 质检员（阶段6）
- 质检标准从哪来？你怎么把标准变成可操作的检验动作？
- 良品/不良品的判定，哪些是硬性、哪些靠经验？
- 抽检比例和判定规则怎么定？
- 放过一个不良品 vs 错杀一个良品，你怎么权衡？

### 8. 生产部（阶段7）
- 接到生产订单后，排产你看哪些约束？
- 生产领料和产能怎么匹配？
- 良率波动时你怎么定位原因？
- 交期紧张时，质量和速度怎么平衡？

### 9. PMC（阶段5-7）
- 要货计划你怎么测算（销量预测/安全库存/提前期）？
- 采购申请和生产订单的节奏怎么排？
- 物料齐套你怎么盯、缺料怎么提前预警？
- 计划被打乱（插单/缺料/延期）时怎么重排？

### 10. 成品仓仓管（阶段7）
- 成品入库验收看什么？
- 发货申请到出库，你的核对流程？
- FBA 货件创建你盯哪些细节（标签/装箱/时效）？
- 库存周转和呆滞品你怎么管？

### 11. 运营（阶段5/8-10）
- 新建 listing 你的搭建顺序和必填要素？
- 关键词策略怎么定、怎么分阶段主推？
- 广告分析你看哪几个指标？什么数据触发调价/关词？
- 推广目标没达成时，你的诊断顺序是什么？
- 客户反馈怎么分级、哪些必须回流到产品？

### 12. 事业部负责人（阶段1/4-5/10）
- 立项时你拍板"做/不做"看哪几个核心判据？
- 定价评审你最关注什么？什么情况一票否决？
- 订单评估你怎么平衡风险和机会？
- 项目暂停/中止的红线是什么？

### 13. 设计师（阶段8）
- 接到设计需求，你先确认哪些信息？
- 主图/产品图你怎么定卖点的视觉优先级？
- 和运营/产品对需求时最容易误解的点？
- 什么样的需求你会判定"返工风险高"？

### 14. 剪拍师（阶段8）
- 视频拍摄方案你先定哪几件事（脚本/分镜/场景）？
- 评测视频 vs 广告视频，你的拍法差别？
- 拍摄最容易翻车的环节？
- 和设计/运营协作，素材交付标准怎么对齐？

### 15. 站外推广专员（阶段9）
- 站外渠道你怎么选、怎么排优先级？
- 推广计划分阶段怎么定、预算怎么分配？
- 效果怎么归因、什么指标判断渠道值不值得继续？
- 站外和站内推广怎么配合节奏？

## 第四部分 · 候选选项库（生成 request_user_input 选项的弹药）

> 下列是各维度的**通用候选选项池**，做选项卡时从这里取通用项，再叠加该角色 / 行业 / 品牌的具体答案，凑齐 4-8 个，**末尾固定加「不清楚 / 不确定」+「其他（请补充）」两个兜底项**。这些是"可能的答案"，不是标准答案，用户勾了才算数；选「不清楚」标待补充、不硬逼。

### 通用维度候选池

| 维度 | 候选选项池（按需取用，可多选） |
|---|---|
| D2 触发与输入（开工前必须拿到的信息） | 需求单 / 工单、产品实拍图或样品、卖点与排序、规格尺寸、竞品参考、品牌规范、交付时间节点、目标平台与受众、预算、其他 |
| D3 标准流程（典型步骤） | 接需求、确认信息、找参考 / 竞品、出草稿 / 方案、内部对齐、初稿交付、收反馈、修改、终稿交付、归档、其他 |
| D4 决策判断（拿不准时看的信号） | 看竞品怎么做、看历史数据 / 转化、看品牌规范、问运营 / 产品、凭经验直觉、看平台规则、按老板偏好、其他 |
| D5 工具与数据（常用工具 / 指标） | 工具：PS / AI / Figma / 剪辑软件 / 协作文档 / 素材库；指标：点击率 / 转化率 / A-B 测试 / 改稿次数 / 退货率 / 其他 |
| D6 协作交接（最常缺 / 最易误解） | 卖点没排序、素材不全 / 不清、需求口头没落文、风格没说清、时间太赶、改了不通知、验收标准不一致、其他 |
| D7 异常与坑（高频翻车点） | 理解偏需求、合规 / 规则违规、尺寸 / 格式错、过度设计、卖点没突出、反复返工、版本管理乱、其他 |
| D8 衡量标准 | 点击 / 转化数据、改稿次数、上级 / 运营满意度、按时交付、风格统一度、零事故 / 零违规、其他 |
| 数值 / 时长档位（量化追问通用） | 当天内 / 1-2 天 / 3-5 天 / 一周以上 / 说不准 / 其他 |
| 数量 / 比例档位（量化追问通用） | 1 个 / 2-3 个 / 4-6 个 / 6 个以上 / 看情况 / 其他 |

### 设计师角色选项卡示例（阶段 8 · 可直接套用）

以"D4 主图卖点视觉优先级怎么定"为例，转成 `request_user_input`：

- prompt：一张主图要放很多卖点，你**靠什么定哪个卖点放 C 位、哪个缩小或砍掉**？（可多选）
- options：
  1. 运营指定的主推卖点
  2. 转化数据 / A-B 测试结果
  3. 竞品都在强调的点
  4. 产品最差异化的功能
  5. 受众最痛的痛点
  6. 凭经验直觉
  7. 不清楚 / 不确定
  8. 其他（请补充）
- allow_multiple: true
- 追问卡（紧接着，仅当用户没选「不清楚」时）：你选的这个排序，主要是"为什么是它最优先"？给候选原因（数据支撑 / 运营要求 / 转化最高 / 最打动人 / 不清楚 / 其他），逼出底层逻辑。

> 其余维度按「通用维度候选池」+ 设计师专属题组（第三部分 #13）现场组合即可。换其他角色时，把专属题组（#1~#15）替换为对应角色的题，候选池叠加该岗位常识。
