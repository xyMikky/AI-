# 个人档案学习引擎（Profile Spec Learner）

## 概述

使用者**基础信息**的解析与结构化沉淀引擎。是「个人中心」的基础信息入口，对称于品牌规范的 `brand-spec-learner`——后者让智能体"认识一个品牌"，本 Skill 让智能体"认识一个人"（岗位、职责、偏好、技能、协作关系）。

**定位（V9.0）**：本系统从「AI 设计师助手」扩展为全公司各部门、各岗位、每个人的可成长辅助助手。本 Skill 负责"使用者是谁"这一层常驻上下文。

## 边界（互斥声明）

| 任务 | 归属 |
|---|---|
| 岗位 / 偏好 / 技能 / 协作关系等**基础信息** | ✅ 本 Skill |
| 工作日志归档 / 工作摘要刷新 / 待办状态 | ❌ → `work-log-manager` |
| 品牌专属规范（色彩/字体/Logo...） | ❌ → `brand-spec-learner` |
| 参考库通用素材 | ❌ → `reference-library-learner` |
| 任何设计执行 / 生图 | ❌ → 设计类 Skill |

## 触发条件

| 用户语义 | 是否激活 |
|---|---|
| "初始化个人档案" / "创建使用者 [名]" | ✅ 新建模式 |
| "更新我的信息" / "设置岗位" / "更新个人偏好" | ✅ 增量模式 |
| **档案为空 + 用户发起任意实质任务**（由强制绑定门触发）| ✅ **onboarding 绑定模式（自动）** |
| "记录工作日志" / "归档日志" | ❌ → work-log-manager |
| "学习品牌 [名]" | ❌ → brand-spec-learner |

**M13（design-requirement-guide）豁免**：本 Skill 是系统维护/信息录入任务，流程固定，跳过需求引导。

## ⚡ onboarding 绑定模式（档案为空时自动激活）

本系统设定为「必须绑定使用者」。当激活使用者的 `个人档案.txt` 为空（完整度 0/4，或姓名/部门/岗位三项全为 `[待填写]`）且用户发起实质任务时，由 `personal-hub-loading.mdc`「强制绑定门」自动调度本 Skill 进入 onboarding 模式：

1. 一句话说明：本助手需先认识你才能更好地辅助
2. 用 **request_user_input** 一次性收集【最小身份信息】（**只问 3 项，不超出，避免劝退**）：
   - 姓名 / 昵称
   - 部门
   - 岗位
3. 写入：`个人档案.txt`（身份概览）+ `基础信息/岗位与职责.txt`（基本信息），完整度 ≥ 1/4
4. 更新 `个人中心主控.txt` 注册表状态（待录入 → 活跃）
5. 标记「已绑定」→ **交还控制权，继续用户原本的任务**

> 偏好 / 技能 / 协作关系 / 工作日志属于增量补全，**不在 onboarding 阶段强求**，后续按需补。
> 用户若回答「匿名 / 先跳过」→ 不写入，本会话按匿名进行，不再追问（见强制绑定门「降级与不打扰原则」）。

## ⚠️ 技术约束（中文路径）

工作区路径含中文，所有文件操作必须用 .NET IO 方法，禁用 `Get-ChildItem`：

```powershell
$root = $PWD.Path
$hub  = Join-Path $root "个人中心"
$user = "默认用户"                      # 或主控激活的使用者
$userDir = Join-Path $hub $user

# 检查使用者目录
[System.IO.Directory]::Exists($userDir)

# 列出已注册使用者
[System.IO.Directory]::GetDirectories($hub) | ForEach-Object { [System.IO.Path]::GetFileName($_) }

# 从模板创建新使用者（含空目录）
$template = Join-Path $hub "_个人目录模板"
robocopy $template $userDir /E /NFL /NDL /NJH /NJS /NC /NS

# 写文件（UTF-8）
[System.IO.File]::WriteAllText($targetPath, $content, [System.Text.Encoding]::UTF8)
```

## 核心流程

```
触发（初始化 / 更新个人档案）
  ↓
Step 1：确定使用者 + 检查目录
  ├─ 已存在 → 增量更新模式
  └─ 不存在 → 从 `个人中心/_个人目录模板/` 复制初始化
  ↓
Step 2：获取信息来源
  ├─ 扫描 `个人中心/[使用者]/00_待学习/`（简历/岗位说明/偏好清单等）
  └─ 或直接读取本轮对话中用户提供的信息
  ↓
Step 3：按维度解析 → 写入 `基础信息/` 四文件
  ├─ 岗位 / 部门 / 职责 / OKR / 边界 → `岗位与职责.txt`
  ├─ 沟通 / 工作方式 / 设计 / 工具 / 不喜欢的事 → `个人偏好.txt`
  ├─ 技能矩阵 / 常用工具 / 在学 / 可协助 / 需协助 → `技能与工具.txt`
  └─ 上下级 / 高频协作 / 跨部门接口 / 升级规则 → `协作关系.txt`
  ↓
Step 4：刷新 `个人档案.txt`（<3KB 速查）+ 更新文件状态表（⬜→✅）
  ↓
Step 5：更新 `个人中心/个人中心主控.txt` 使用者注册表
  （档案完整度 X/4、最后更新日期、状态）
  ↓
Step 6：清理 `00_待学习/` 已处理素材（保留 .txt 说明）
  ↓
输出【个人档案更新摘要】
```

## 写入目标文件（维度路由表）

| 维度 | 写入文件 | 判定"已填充"标准 |
|---|---|---|
| 岗位与职责 | `个人中心/[使用者]/基础信息/岗位与职责.txt` | 至少含岗位名 + 1 条核心职责 |
| 个人偏好 | `个人中心/[使用者]/基础信息/个人偏好.txt` | 至少含回复风格 / 1 条明确不喜欢的事 |
| 技能与工具 | `个人中心/[使用者]/基础信息/技能与工具.txt` | 技能矩阵至少 1 行 |
| 协作关系 | `个人中心/[使用者]/基础信息/协作关系.txt` | 至少含直属上级或 1 个高频协作对象 |

**档案完整度** = 已填充维度数 / 4。

> 写入时**保留各模板文件已有的章节骨架**，只替换 `[待填写]` 占位与追加内容，不破坏结构。

## `个人档案.txt` 刷新规则（<3KB 速查层）

`个人档案.txt` 是每次会话强制加载的速查层（见 `personal-hub-loading.mdc`），只放高频摘要，详细内容留在 `基础信息/` 与 `工作摘要.txt`：

- 身份概览（姓名/部门/岗位/汇报关系/入职日期/一句话定位）
- 工作上下文速查（核心项目/状态/本周重点/待办 Top3）—— 若 `工作摘要.txt` 已有则同步要点，否则留待 work-log-manager 填充
- 个人偏好速查（沟通风格/输出语言/工具偏好/不喜欢 AI 做的事）
- 规范文件路径状态表
- 成长记录摘要（岗位变化/里程碑追加一行）

必须 <3KB；超出说明信息密度过低，应下沉到 `基础信息/` 对应文件。

## 个人中心目录结构（模板）

```
个人中心/[使用者]/
├── 个人档案.txt              ← always-load, <3KB
├── 个人索引.txt              ← 内导航
├── 00_待学习/                ← 基础信息素材入口（简历/岗位说明等）
├── 基础信息/
│   ├── 岗位与职责.txt
│   ├── 个人偏好.txt
│   ├── 技能与工具.txt
│   └── 协作关系.txt
└── 工作日志/                 ← 由 work-log-manager 维护
    ├── 工作摘要.txt
    ├── 日志索引.txt
    ├── 00_待归档/
    └── 按年/                 ← 首次归档时创建
```

## ❌ 禁止行为

- 编造用户未提供的岗位 / 职责 / 协作关系（不掌握的维度留 `[待填写]`）
- 把工作日志内容写进基础信息文件（→ work-log-manager）
- 把个人信息写进品牌规范目录
- 多使用者场景下混用 A 的档案更新 B 的目录
- 用 `Get-ChildItem` 列举中文路径（编码问题，必用 .NET IO）
- 让 `个人档案.txt` 超过 3KB

## ✅ 正确行为

- 新使用者从 `_个人目录模板/` 复制初始化
- 按维度路由写入 `基础信息/` 四文件，保留模板章节骨架
- 刷新 `个人档案.txt` 速查层 + 更新主控注册表完整度
- 信息不足时明确询问用户，而非填假数据
- 完成后输出【个人档案更新摘要】（含完整度 X/4 + 各文件状态）

## 与其他 Skill / 规则的关系

| 对象 | 关系 |
|---|---|
| `work-log-manager` | 互补：本 Skill 管"使用者是谁"，后者管"使用者在做什么" |
| `brand-spec-learner` | 镜像：品牌 vs 个人，结构对称 |
| `personal-hub-loading.mdc` | 配套加载规则：本 Skill 的产物是其 Always-Load 内容 |
| `个人中心主控.txt` | 注册表载体：新建/更新使用者后在此登记 |
