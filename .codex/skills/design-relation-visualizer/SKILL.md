# 设计关系可视化（Design Relation Visualizer）

## 概述

将设计系统/品牌体系/配色逻辑/参考档案/模块协作等关系转化为**单个可双击打开的交互式 HTML 文件**。前身是能力模块 M12_设计关系可视化（V2.0 浅色系主题）。

**与 logic-chain-diagnostic 的边界**：本 Skill 是"设计层面的可视化"（用户视角，给客户看），logic-chain-diagnostic 是"系统层面的健康度"（架构师视角，给运维看）。

## 使用场景

| 场景 | 用户表述 |
|---|---|
| 设计复盘 | "梳理一下我这次设计的决策链路" |
| 品牌体系梳理 | "把整个品牌视觉系统画一张图" |
| 设计沟通 | "做一张图给客户讲清楚配色逻辑" |
| 参考库总览 | "把所有参考档案的关系画出来" |
| 模块依赖 | "M1-M18 之间的调用关系图" |

## 核心能力

1. **设计语义提取**：从文档/规范/参考档案识别核心节点（10-200 个之间）
2. **关系建模**：父子包含 / 并列同类 / 引申扩展 / 因果依赖
3. **自包含 HTML 工程**：D3.js（CDN）+ JSON 数据 + CSS + JS 全打包成单文件
4. **7 项交互**：单击高亮三级分层 / 双击聚焦 / 右键追问 / 搜索 / 布局切换 / 画布缩放 / 清除

## 输出规范

| 项 | 要求 |
|---|---|
| 文件 | 单个 .html，双击在浏览器打开即可运行 |
| 大小 | 通常 50KB-500KB，无外部资源依赖 |
| 主题 | V2.0 浅色系（参考 D 域 Neumorphism + B4 中性） |
| 节点数 | 10-200 |
| 交互 | 7 项必须全部实现 |

### 单击节点高亮规范（280ms 过渡）

| 层级 | 圆圈变化 | 文字变化 | 连接线变化 |
|---|---|---|---|
| 直接关联（一级邻居） | 1.4× 放大、橙色描边 | 加粗、放大 | 高亮主色、加粗 |
| 间接关联（二级邻居） | 1.1× 放大、淡橙色边 | 略加粗 | 半透明高亮 |
| 无关节点 | 0.4 透明度 | 灰化 | 0.15 透明度 |

## 工作流程

### Step 1：识别可视化对象

- 用户提供文本描述 → 解析为节点+边
- 用户引用文件（如品牌档案）→ 读取后解析
- 用户说"模块关系图" → 直接用 M1-M18 + 工具 Skill 注册表

### Step 2：节点分层

- 一级（系统/品牌核心，3-7 个）
- 二级（子系统/分类，10-30 个）
- 三级（具体元素/细节，根据复杂度）

### Step 3：关系建模

- **父子包含** → 实线
- **并列同类** → 虚线
- **引申扩展** → 点划线
- **因果依赖** → 带箭头实线

### Step 4：生成 HTML 工程

参考 V2.0 浅色系模板，将节点/边数据序列化为 JSON 嵌入 HTML，输出到用户指定路径或 `生成结果输出/visualizations/` 默认路径。

## 与其他 Skill 的关系

| Skill | 关系 |
|---|---|
| `logic-chain-diagnostic` | 完全互斥：本 Skill 是设计可视化，前者是系统诊断 |
| `brand-spec-learner` | 可读取品牌档案作为可视化数据源 |
| `reference-library-learner` | 可读取参考档案三层索引作为数据源 |
| 其他业务 Skill | 不被调用，独立工具 |

## 注意事项

- **唯一产出是 HTML 文件**，不是文字分析报告
- 不参与三卡交接体系（A/B/C/D/E/F），独立于设计执行链路
- 节点数 < 10 时建议改用普通文字描述，> 200 时建议拆分多张图
- 浅色系主题：白底、奶油暖灰、橙色克制点缀，不要暗黑
