# 提示词卫生 · 四条铁律

**核心认知：提示词的每一个 token 都在和参考图竞争模型的注意力。** 多余的元信息、目录说明、对参考图已承载内容的文字复述，都会稀释模型对"图像里到底长什么样"的读取权重，导致色偏、结构漂移、细节丢失。

四条铁律：① 禁内部标识 · ② 禁图片角色目录块 · ③ 禁文字复述参考图已承载的视觉属性 · ④ 优先正向表述。

---

## 铁律一：禁内部标识污染

提交给生图模型的 Prompt 是给模型读的**视觉指令**，不是给系统读的审计日志。任何用于调度、溯源、模块互引、分类编号的标识符，一律不得出现在最终 Prompt 正文或注释中——模型不认识这些符号，只会当成噪声，或错误地试图把它"画"出来。

### 禁止出现的内部标识（通用清单）

| 类别 | 示例 | 问题 |
|---|---|---|
| 案例 / 编号 | `X-001`、`P-003`、`案例#12` | 模型可能误解为尺码/型号 |
| 流程 / 模块标签 | `M16`、`Step 3`、`阶段⑥`、`Pre-Gen` | 与视觉无关 |
| 规则 / 文件名 | `xxx.mdc`、`SCENE-5`、`rule 2.1` | 内部引用 |
| 分类代号 | `T1`、`B1`、`SI-2`、`P7` | 姿势/场景代号 |
| 元注释短语 | `anti X-001`、`规避陷阱`、`防遮挡对策` | 审计标记 |
| 行内注释 | `# 对抗过紧`、`// 防头发遮挡`、`【仅给自己看的话】` | 模型可能把注释当文字画进画面 |

### 正确做法：把"为什么"翻译成"画面要什么"

需要规避某个历史问题时，把约束转成**纯视觉语言**，描述目标状态，然后把内部编号丢掉。

```text
# ❌ 污染版（带内部代号）
【取景构图 — 规避 X-001】产品占画面 55-65%...

# ✅ 干净版（纯视觉语言）
产品躯干区域占画面 55-65%，取景下边界落在髋线、上边界落在肩线以上，产品完整可见。
```

内部溯源信息写到 Prompt **之外**（你的回复正文 / 任务记录），不进 Prompt。

## 铁律二：禁图片角色目录块

不要在 Prompt 开头写一段"图片角色目录"：

```text
# ❌ 错误（前置目录块 + 后文又重复一遍）
[图片角色目录]
image 1 = 产品参考（带网纱设计的连体衣）
image 2 = 仅作模特身份参考——仅借鉴面部...
image 3 = 仅作场景参考——空间布局...
```

问题：① 它把参考图又用文字描述了一遍（构成铁律三的过度描述）② 占用本该给视觉指令的 token ③ 后文用 `image N` 内联引用时会再说一次，纯重复。

### 正确做法：内联引用，用到时一句话说清角色

```text
# ✅ 正确（内联）
产品与 image 1 中完全一致。
使用 image 2 中人物的面容与发型——服装和背景以各自对应的图为准。
按 image 3 重建环境——墙面、地板、家具，不复制其中的人物。
```

**例外**：当传入 **≥5 张图**且其中 ≥2 张容易混淆角色（如两张都是人物，一张身份参考、一张姿势参考），可保留**极简版**一行目录（每张一句、绝不复述画面内容）：

```text
image 1 = 产品。image 2 = 面部参考。image 3 = 姿势参考。image 4 = 场景。image 5 = 色卡。
```

4 张及以下始终禁用目录块。

## 铁律三：禁文字复述参考图已承载的视觉属性

参考图里**已经有**的视觉信息（颜色、材质、结构、纹理、廓形、**版式、构图、模块排布、场景材质**），不要用文字再命名一次。只说"保持/来自 image N"。

### 最典型的翻车——颜色词汇化

```text
# ❌ 错误（给颜色起名字）
产品像素精确保留 image 1 中的巧克力棕色调。

# ✅ 正确（指向图片，不命名）
产品像素精确保留 image 1 中的颜色——色相、饱和度、明度完全一致。
```

为什么 "chocolate brown" 有害：模型对该词有**自己的先验色分布**（偏橙红？偏冷？偏黑？），会和 image 1 的实际像素打架，生成一个"既像图又像文字"的中间色——这就是色偏的根源。参考图里的 RGB 像素已经是最精确的颜色定义。

### 适用范围（不止颜色）

| 参考图已承载 | ❌ 文字重命名 | ✅ 图片锚定 |
|---|---|---|
| 颜色 | `巧克力棕` / `暖焦糖色` | `image 1 中的精确颜色` |
| 材质 | `网纱育克配莱卡主体` | `image 1 中的面料与分片结构` |
| 结构 | `高立领短袖设计` | `image 1 中的领口、袖型、廓形` |
| 发色/肤色 | `自然金棕波浪卷` / `白皙肤色` | `image 2 中的发色/肤色` |
| 场景材质 | `条纹木地板配白长椅` | `image 3 中的地板、长椅、墙面` |
| 版式/模块 | `5 区堆叠：顶部 10%、主视觉 35%(50:50)...` | `复刻 image 4 的多区块结构——区块数量/顺序/比例一致` |
| 装饰元素 | `右上 220px 圆形插图、4 列 8px 圆角缩略图` | `image 4 中的圆形插图与 4 格缩略图行` |

### 文字只负责三件事

1. **指向图片**：`the X from image N` / `复刻 image N`
2. **补充方向性信息**（图里没有的）：`— 展示背面视角` / `— 特写腰部接缝`
3. **锁定易被弱化的关键锚点**：`保留 image N 中的红色下划线 — 位置精确一致`

> 同一条铁律的另一面：有色卡图同时传入时，可用 HEX 精确引用色卡上"确实存在"的方块——那不是对产品图颜色的文字重命名（详见 COLOR_AND_BACKGROUND.md）。

## 铁律四：优先正向表述

告诉模型"画面要呈现什么"，而不是"不要出现什么"。多数主流模型对正向指令的执行比负向约束更稳定——模型要先激活被禁的概念再压制它，反而增加输出概率。

| ❌ 负向约束 | ✅ 正向表述 |
|---|---|
| 不要改变背景 | 背景与 image N 保持一致 |
| 不要改变产品颜色 | 产品色精确保留 image 1 原色 |
| 禁止降饱和 / 偏色 | 色彩保持完全饱和，色相与 image 1 精确一致 |
| 不要让头发遮挡肩部 | 发尾梳至一侧，双肩完整可见 |
| 不要裁太紧 | 取景下边界落在髋线，主体完整可见 |
| 不要有杂物 | 画面中只有产品与纯白背景 |

**例外（允许保留负向，但需与正向配对）**：
- 没有精确正向等价的尺寸边界：`取景不超过腰线以上`
- 大写技术信号词：`BACKGROUND ONLY` / `PRODUCT ONLY`（非自然语言负向）
- 某些模型的 negative prompt 字段（这是模型机制，不是正文负向句）

自检：通篇搜 `不要/禁止/不得/no/not/never/avoid`，每处判断能否换成正向等价句。

## 长度健康度（四铁律的综合体现）

| 任务类型 | 理想长度 | 超长信号 |
|---|---|---|
| 单产品换背景/换角度 | 80–150 token | >200 = 在复述产品 |
| 含版式参考的多模块图 | 200–300 token | >400 = 在复述版式 |
| 含场景参考的全身人物图 | 150–250 token | >350 = 在复述场景材质 |

超长 → 回头查：是不是把参考图里看得见的东西又用文字说了一遍？
