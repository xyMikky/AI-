# 提交前自检清单

交付提示词前逐项核对，**不自检不交付**。建议把自检结果以表格形式附在交付里，让用户可验证。

---

## A. 提示词卫生（四铁律 · 见 FOUR_IRON_LAWS.md）

- [ ] **无内部标识**：无案例编号 / 流程标签 / 规则文件名 / 分类代号 / `规避XX` 等元注释
- [ ] **无行内注释**：无 `#`、`//`、`【仅给自己看的话】`（模型会把注释当文字画出来）
- [ ] **无图片角色目录块**（传入图 ≤4 张时）；≥5 张且易混淆时仅保留极简一行目录
- [ ] **不复述参考图视觉**：无 `巧克力棕`/`网纱面料`/`金棕发色`/逐模块版式描述等文字重命名，改为 `the X from image N`
- [ ] **正向表述**：搜 `不要/禁止/no/not/never/avoid`，能转正向的已转；保留的负向已与正向配对

## B. 参考图引用（见 REFERENCE_IMAGE_PROTOCOL.md）

- [ ] `--images` 每张图都在 Prompt 中用 `image N` 引用
- [ ] 每个 `image N` 都标注了**借鉴维度**（外貌/产品/版式/色调/Logo/背景/姿势）
- [ ] 无 `the reference image`/`上传的图` 等笼统指代
- [ ] 无"传了图却不引用"的情况（不引用就从 `--images` 移除）
- [ ] 含场景参考图时，写全 S1-S5 五要素（角色声明/空间层次/光线锚定/融合锚定/排除项）
- [ ] 已附【图片引用完备性自检】表

## C. 色彩纪律（画面含产品照片时 · 见 COLOR_AND_BACKGROUND.md）

- [ ] 含产品照片 → 写了句式 A（产品色独立性）
- [ ] 有色卡图 → 写了句式 B（每个 HEX 标 BACKGROUND/PRODUCT/DECORATIVE/TEXT ONLY）
- [ ] 含产品特写/缩略图 → 写了句式 C（特写防色调牵引）
- [ ] **禁句扫描**：无 `色调向…统一`/`降饱和以匹配`/`与背景融合`/`色彩统一`等牵引句
- [ ] 颜色用 HEX 或 `image N` 锚定，未给颜色起文字名

## D. 背景策略（见 COLOR_AND_BACKGROUND.md）

- [ ] 背景色已明确（HEX 或 `image N`），未含糊
- [ ] 背景符合项目默认策略，或已说明偏离原因（品牌/用户指令）
- [ ] 参考图带有色背景时，已确认是继承还是改白底（未盲目继承）

## E. 结构与语言（见 PROMPT_STRUCTURE.md）

- [ ] 六要素覆盖完整（主体/动作/环境/构图/光线/风格）
- [ ] 结构标签用英文 `[TAG]`
- [ ] Prompt 主体语言跟随用户工作语言
- [ ] 技术锚点原样保留（image N / HEX / 参数 / 路径 / 品牌名 / 大写约束词）
- [ ] 品牌名只在合法位置（参考图声明 / `[LOGO REFERENCE]`）
- [ ] 删了 `品牌DNA/体现品牌` 等元描述，但保留了 1-2 个抽象调性词

## F. 长度与参数

- [ ] 长度在健康区间（单图 80-150 / 版式 200-300 / 场景 150-250 token），超长已回查过度描述
- [ ] `--aspect-ratio` / `--resolution` 已给（用默认时已说明）
- [ ] img2img / txt2img 选择正确（有参考图=img2img，无=txt2img）

## G. 任务拆解

- [ ] ≥3 维度同时变 / 多图实景合成 → 已考虑拆成多步提示词序列（先洗背景→再合成等），而非硬塞一条

---

## 自检输出格式（附在交付末尾）

```text
【提交前自检】
A 卫生四铁律：✅
B 参考图引用：✅（image 1/2/3 均引用并标注维度，见完备性自检表）
C 色彩纪律：✅ / N/A（无产品照片）
D 背景策略：✅ 纯白 #ffffff（项目默认）
E 结构语言：✅ 英文标签 + 中文主体 + 锚点保留
F 长度参数：✅ ~XXX token，1:1 / 2k
G 任务拆解：✅ 单图直出 / 拆为 N 步
```
