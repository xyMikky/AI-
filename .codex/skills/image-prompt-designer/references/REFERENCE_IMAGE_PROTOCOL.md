# 参考图引用协议（img2img 多图必读）

## 模型行为关键事实

> **传入但未在 Prompt 中用 `image N` 显式引用的图片 = 完全不存在。**

底层模型（Nano Banana / Gemini / 多数 img2img 模型）**不会自动理解** `--images` 中每张图的用途。如果你传了 5 张图但 Prompt 只引用了 `image 1` 和 `image 3`，那么 `image 2/4/5` 会被**完全忽略**——等于白传，甚至可能让模型把未引用图的特征随机混进结果。

**传入图片 ≠ 模型会参考它。Prompt 中写明 `image N` + 借鉴维度 = 模型才会参考。**

---

## 正确引用的三要素（缺一不可）

1. **`--images` 传入**：图片通过参数上传
2. **`image N` 编号标注**：Prompt 中用 `image 1` / `image 2` / ... 指向该图
3. **借鉴维度说明**：明确说从该图借鉴**什么**（外貌 / 产品结构 / 版式 / 色调 / Logo / 背景 / 姿势）

```text
# ✅ 三要素齐全
保留 image 1 中的产品原样。使用 image 2 中人物的面容与身材。借鉴 image 3 的对角分割版式。

# ❌ 缺编号（模型不知哪张图对应什么）
保留参考图中的产品。使用模特的外貌。应用版式风格。

# ❌ 缺借鉴维度（模型随机混合所有图特征）
参考 image 2。参考 image 3。

# ❌ 笼统指代（禁用）
参考 the reference image / the uploaded photo
```

编号规则：`image 1` 通常给用户主图（要保留的主体），后续图依传入顺序递增。建议总图数 ≤ 4 时模型理解最清晰；>4 张时每张角色要更明确地差异化标注。

---

## 过度描述陷阱（Reference Over-Description Anti-Pattern）

**当参考图已传入时，禁止在 Prompt 中用文字重新描述该图已承载的视觉内容**——无论产品、模特、场景、构图、版式。文字描述会与图片像素竞争，文字越多越细，结果越偏离参考图。

### 三大形态

#### ① 产品过度描述
```text
# ❌ 重新列举产品特征
保留 image 1 中的连体衣——保留可调肩带、网纱面板、底部开裆、防滑蕾丝边及整体廓形。

# ✅ 极简指向
保留 image 1 中的连体衣原样。
# 需强调角度时只加方向词：
保留 image 1 中的连体衣原样——展示背面视角。
```

#### ② 版式过度描述（多模块/详情页最严重）
```text
# ❌ 逐模块复述（5 模块 × 每段 5-10 行 = 300-800 token）
模块 1 — 顶部品牌区(顶部 10%)：居中品牌字，无衬线，宽字距...
模块 2 — 主视觉分栏(10%-45%)：左右各 50%...
模块 3 — 产品矩阵(45%-70%)：4 列网格...

# ✅ 三件套（版式引用句 + 差异点清单 + 关键锚点，约 200 token）
[版式 — 来自 image 4]
完整复刻 image 4 的竖向多模块结构——模块数量与顺序、形状、比例、点缀用法、排版层级一致。
[与 image 4 的差异替换项]
- 占位剪影替换为 image 1/2/3 的实拍产品与模特。
- 占位文字替换为实际文案。
[关键锚点 — 必须从 image 4 精确保留]
- 红色强调下划线、圆形插图、分页点——位置与 image 4 精确一致。
```

#### ③ 场景过度描述
```text
# ❌ 重命名场景材质
搭建浅橡木地板、哑光白墙、中世纪极简白木长椅的场景...

# ✅ 指向场景图 + 声明层次与光线
image 5 仅作场景参考——按 image 5 重建环境，保持前景/中景/背景景深，光线方向、硬度、色温与 image 5 一致。
```

### 判定原则

| 情形 | 做法 |
|---|---|
| 参考图里**已有**的（色/材质/结构/版式/构图/场景材质） | 只指向图片，不复述 |
| 参考图里**没有**的补充信息（角度/焦点/展示状态/占位符替换内容） | 可以用文字 |
| 参考图**易被弱化**的关键元素（品牌装饰/箭头/圆点/特殊形状） | 少量锚点句锁定 |
| 有色卡图时的 HEX 硬约束 | 可写 HEX（见 COLOR_AND_BACKGROUND.md） |

## 场景参考图五要素（含场景库/环境图时必写）

仅写 `背景来自 image N` / `same vibe` 容易变成壁纸拼贴。含场景参考图时写全五要素：

- **S1 角色声明**：`image N 仅作场景参考（SCENE REFERENCE ONLY）`，人物与产品以各自图为准
- **S2 空间层次**：前景 / 中景 / 背景的景深关系
- **S3 光线锚定**：主光方向 + 硬度 + 色温与 image N 一致
- **S4 融合锚定**：接触面 + 投影（模特/产品在何处落地、阴影方向）
- **S5 排除项**：不引入场景图里的无关人物 / 文字 / Logo

## 图片引用完备性自检（提交前必做）

逐一核对：`--images` 中每张图是否都在 Prompt 中被 `image N` 引用并标注了借鉴维度。

```text
【图片引用完备性自检】
--images 传入 N 张：
  image 1: [文件名] → 引用？✅ 借鉴：[维度]
  image 2: [文件名] → 引用？✅ 借鉴：[维度]
  image N: [文件名] → 引用？✅ / ❌ 未引用
```

- 全部 ✅ → 可交付
- 任一 ❌ → 二选一：① 补充该图的 `image N` 引用 + 借鉴维度；② 从 `--images` 移除该图（不需要就不传）

**禁止"传了图却不引用"**——这不是无害冗余，模型能"看到"所有图，没有明确指令时会不可预测地混入特征。

---

## 组图（一次生成多张）

- 同一批次可共享同一组参考图，**不要求**每张都用不同参考图。
- **但每张图的 Prompt 都必须各自写出 `image N` 引用**——即使 5 张图都用同一张参考图。
- 允许不同图借鉴不同维度（图1 借版式、图2 借色调）。
