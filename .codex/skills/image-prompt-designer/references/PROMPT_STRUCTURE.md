# 提示词结构 · 六要素 · 语言 · token 效率

## 一、六要素公式（任何生图任务的骨架）

一条完整的生图提示词通常覆盖六个维度，缺哪个补哪个：

| 要素 | 回答什么 | 示例关键词 |
|---|---|---|
| 1. 主体 Subject | 画面主角是谁/什么 | 产品、模特、套装部件 |
| 2. 动作/状态 Action | 主体在做什么/呈现什么状态 | 穿戴、陈列、走步、展示背面 |
| 3. 环境 Environment | 在哪、什么背景 | 纯白棚拍、客厅场景、户外 |
| 4. 构图/镜头 Composition | 怎么取景 | hero 陈列、低角度仰拍、3/4 侧、特写 |
| 5. 光线 Lighting | 什么光 | 均匀柔和棚拍光、自然窗光、戏剧硬光 |
| 6. 风格/调性 Style | 什么气质 | 干净电商感、editorial 高级感、暖色生活感 |

文生图（txt2img）六要素全靠文字；图生图（img2img）中已被参考图承载的要素**只指向 image N、不复述**（见 FOUR_IRON_LAWS.md 铁律三）。

## 二、结构标签：用英文方括号分块

复杂提示词用方括号标签分块，模型更易解析各段职责。**标签用英文**（更省 token、模型识别更稳），标签内的描述用用户工作语言。

推荐标签集：
```
[BRAND TONE]   [COLOR PALETTE]   [SCENE REFERENCE]   [MODEL & PRODUCT]
[PRODUCTS]     [PACKAGING]       [LAYOUT]            [SCENE INTERACTION]
[CAMERA & FOCUS]  [POSE]  [LIGHTING]  [BACKGROUND]  [TEXT AREA]  [FRAMING]
[LOGO REFERENCE]  [NEGATIVE]
```

```text
# ✅ 英文标签 + 用户语言内容
[CAMERA & FOCUS]
微俯视 hero 陈列视角，整组清晰对焦、高解析细节。

# ❌ 中文标签（多耗 token）
[镜头与焦点]
微俯视 hero 陈列视角...
```

简单单图任务（如"换白底"）不必强行套标签，自然语言一段话即可。

## 三、默认语言策略

| 内容 | 语言 |
|---|---|
| Prompt 主体描述 | **跟随用户工作语言**（用户用中文 → 写中文主体） |
| 结构标签 | 英文 `[TAG]` |
| 技术锚点 | **原样保留，永不翻译** |

技术锚点清单（恒不翻译、不省略）：
- 参考图编号 `image 1` / `image 2` / `image N`（禁止写成"第一张图"）
- HEX 色号 `#bf192e`（可用"色名 #hex"括注，但不能省 HEX）
- 参数 `--aspect-ratio` / `--resolution` / `--quality` 等
- 文件路径、品牌名、Logo 文字
- 色彩硬约束词 `BACKGROUND ONLY` / `PRODUCT ONLY` / `DECORATIVE ELEMENTS ONLY` / `TEXT ONLY`
- 标准视觉术语可"译文 + 英文括注"，如 `低角度仰拍 (low-angle shot)`

> 为什么不一律用英文：多数现代多模态模型对主流语言理解相当；用用户语言便于用户校对修改，技术锚点保留则保证模型精确执行。除非用户明确要求英文 Prompt。

## 四、品牌名与抽象调性词的处理

### 品牌名只放两个合法位置
1. `image N = 品牌 Logo / 品牌资产图` 的参考图声明里
2. `[LOGO REFERENCE]` 块（仅当用户明确要求渲染 Logo）

**非法位置**：内容描述句、调性描述、场景描述。模型不认识"某品牌调性"是什么视觉——它只会试图把品牌名当文字画出来，或忽略。

```text
# ❌ 品牌名在描述句里
NEBILITY 品牌调性：精致、赋能，整体要体现品牌 DNA...

# ✅ 删掉品牌名与元描述，保留具体调性词
调性：精致、自信、赋能。
```

### 元描述短语要删
`品牌调性:` / `品牌核心:` / `品牌 DNA` / `体现品牌XX` / `符合品牌XX` 这类对模型无视觉语义的短语删掉，只留后面的具体形容词。

### 但要保留 1-2 个抽象调性词
不要把抽象调性词删光只剩冷冰冰的视觉清单——`精致 / 自信 / editorial / effortless / premium / refined / empowering` 等在时尚/广告训练数据里有语义激活，能唤起模型的审美模式。保留 1-2 个核心调性词即可。

## 五、token 效率自检

- 结构标签是否英文？
- 品牌名是否只在合法位置？
- 是否有 `品牌DNA/体现品牌` 等无视觉语义的元描述？删。
- 是否仍保留了 1-2 个抽象调性词？（别删光）
- 总长是否超出长度健康度？（见 FOUR_IRON_LAWS.md：单图 80-150 / 版式 200-300 / 场景 150-250 token）

## 六、模型差异速查（跨模型迁移时参考）

| 模型族 | 偏好 | 负向提示词 | 备注 |
|---|---|---|---|
| Nano Banana / Gemini Image | 自然语言句子、`image N` 多图引用强 | 写进正文正向化为主 | 多图合成、保真编辑强；长描述会稀释参考图 |
| Midjourney | 短语堆叠 + 风格词 + `--ar`/`--stylize` 参数 | `--no xxx` | 关键词权重 `::`；偏艺术化 |
| Flux / SDXL | 自然语言 + 风格 token | 独立 negative prompt 字段 | SDXL 负向字段有效；Flux 偏自然语言 |
| DALL·E | 完整自然语言句子 | 正向化为主 | 不擅长精确多图引用 |
| Seedream / 国产文生图 | 中文自然语言友好 | 视实现 | 中文 token 友好 |

**通用不变量**（所有模型都成立）：四铁律、参考图引用三要素、色彩角色隔离、正向表述优先。模型差异只影响"语法外壳"，不影响这些底层原则。
