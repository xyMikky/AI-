# 星驰新品开发到上市全流程
> 来源：星驰业务流程图 `原始素材/星驰-新品开发到上市全流程-原始流程图.pdf`
> 归档日期：2026-06-01
> 用途：作为星驰品牌新品项目的跨部门流程地图、阶段门判断与设计介入节点依据
> 关系：本流程是《产品开发工作法》（目的/目标/第一性原理/五看三定/六大原则/七大步骤）的**落地执行版**——方法论解决“怎么想”，本流程解决“怎么跑”。

---

## 一、参与角色（泳道）

产品工程师、专利工程师、包装设计师、商务、采购、原材料仓仓管、质检员、生产部、PMC、成品仓仓管、运营、事业部负责人、设计师、剪拍师、站外推广专员。

---

## 二、流程总览

```text
项目启动会 → 需求收集/五看 → 专利排查 → 产品定义/策划评审
→ 竞品收集/测试 → BOM 拆解 → 供应商确认 → 样品生产/测试/评审
→ 审厂 / 质检标准确认 → 运费估算/报价/定价/利润核算 → 定价评审会
→ 产品发布内部评估 → 发布会/订单收集/后端同频会 → 要货计划
→ 采购申请/生产订单/采购订单 → 包装设计/确认量产样/供应商跟线
→ 收料/品质验检/物品入库 → 生产领料/包装生产/成品入库
→ 发货申请/审批/销售出库/物流发货/创建 FBA 货件
→（设计线）设计需求/图片交付/视频交付
→ 新建 listing/上架/评测准备 → 积累评论 → 启动推广（站内+站外）
→ 广告分析调控/客户反馈处理/页面优化 → 新品推广总结/项目总结
```

---

## 三、阶段拆解与责任角色

### 阶段 1 · 立项与需求（产品工程师 / 专利工程师 / 商务 / 采购）

1. 项目启动会，判断是否启动产品工程。
2. 需求收集 + 五看（行业趋势 / 市场 / 客户 / 竞对 / 自己）。
3. 专利排查、专利风险审查。
4. 产品定义、产品草图、产品策划评审。

阶段门：策划评审**不通过 → 产品重定义**；严重时**项目暂停 / 中止 / 结束**。

### 阶段 2 · 产品定义与竞品（产品工程师 / 产品经理）

1. 竞品收集、竞品测试。
2. 产品定义更新。
3. BOM 拆解。

阶段门：**通过 → 进入供应链**；不通过回到产品重定义。

### 阶段 3 · 供应链与打样（采购 / 商务 / 质检 / 包装设计师）

1. 采购收集供应商、供应商确认、HS code 查询、供应商开票确认、采购复审。
2. 样品生产、样品测试、样品评审。
3. 包装方案评估、包装费用。
4. 审厂、质检标准确认。

阶段门：样品评审 / 包装方案**不通过 → 返工重打样或重定义**。

### 阶段 4 · 成本定价（商务 / 事业部负责人）

1. 运费估算、包装费用。
2. 产品报价、产品定价、利润核算。
3. 定价评审会。
4. 产品发布内部评估。

阶段门：定价 / 利润**不通过 → 回退调整成本、供应商、包装或定义**。

### 阶段 5 · 订单与生产准备（事业部负责人 / 运营 / PMC / 采购）

1. 运营项目启动会、发布会、订单收集、后端同频会、订单评估。
2. 要货计划、采购申请单、生产订单、采购订单。
3. 采购退料 / 退货发货（异常路径）。

### 阶段 6 · 量产与质检（包装设计师 / 供应商 / 质检员 / 仓管）

1. 包装设计、确认量产样 / 产前样、供应商跟线、生产中心确认。
2. 收料通知、收料、品质验检。
3. 良品 → 物品入库；不良品 → 退料 / 退货处理。

### 阶段 7 · 生产入库与发货（生产部 / 原材料仓 / 成品仓 / PMC / 运营）

1. 生产领料、原材料仓发货、包装生产、成品入库。
2. 提交发货申请 → 审批发货申请。
3. 销售出库、物流发货、创建 FBA 货件。

### 阶段 8 · 设计与内容（设计师 / 剪拍师）

1. 产品优化需求、设计需求提交、视频拍摄方案。
2. 图片交付、视频交付。

> 设计介入点：HERO/主图/详情页用于 listing；视频用于评测、广告与站外推广。

### 阶段 9 · 上市与运营（运营 / 站外推广专员）

1. 新建 listing、产品上架、评测准备、listing 完成搭建。
2. 积累评论、启动推广。
3. 推广计划（分阶段主推关键词）、站外推广、广告分析和调控。
4. 客户反馈处理、页面优化。

阶段门：推广目标**未达成 → 需求管理 / 问题管理**循环；**达成 → 总结**。

### 阶段 10 · 复盘闭环（运营 / 事业部负责人）

1. 新品推广总结、项目总结。
2. 问题与优化需求回流到产品优化，进入下一轮迭代。

## 四、关键评审节点（阶段门清单）

| 阶段门 | 判断标准 | 通过去向 | 不通过去向 |
|---|---|---|---|
| 是否启动产品工程 | 机会/资源/风险 | 进入需求收集 | 项目结束 |
| 产品策划评审 | 是否符合产品定义与价值 | 竞品测试/定义更新 | 产品重定义 |
| 样品评审 | 样品+测试+质检+包装 | 成本报价 | 重打样/重定义 |
| 包装方案评估 | 成本/结构/合规 | 量产准备 | 重新设计包装 |
| 定价评审会 | 报价/利润/运费/包装 | 发布内部评估 | 回退成本环节 |
| 产品发布内部评估 | 商业可行性 | 发布会/订单 | 暂停/中止 |
| 品质验检 | 良品/不良品 | 入库 | 退料/退货 |
| 发货申请审批 | 发货合规 | 销售出库/FBA | 驳回 |
| 推广目标复盘 | 是否达成 | 项目总结 | 需求/问题管理 |

## AI 助手使用方式

当用户围绕星驰提出新品项目推进、阶段判断、卡点诊断、责任分工、上市与复盘任务时，应优先调用本流程，并与《产品开发工作法》配合：

1. 先用《产品开发工作法》判断“目的/目标/卖点/策略是否清晰、可量化”。
2. 再用本流程判断“当前处于哪个阶段、卡在哪个阶段门、下一步该谁做什么”。
3. 若任务属于设计/内容，按上表对接对应设计 Skill，并明确该设计服务于哪个业务阶段。
