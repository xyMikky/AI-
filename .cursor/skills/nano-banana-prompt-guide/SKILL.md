# Nano Banana 提示词专家指南

## 概述

RH 全能图片PRO-图生图的底层模型是 Google **Nano Banana Pro**（`gemini-3-pro-image-preview`，即 RunningHub 的 `rhart-image-n-pro/edit` 端点）。本 Skill 提供该模型的完整提示词写作规范，与 `rh-image-pro-img2img` skill 配合使用，实现高质量图像生成。

**使用流程**：本 Skill 负责构造提示词 → `rh-image-pro-img2img` Skill 负责执行 API 调用。

---

## ⚡ 语言规范（默认中文，最高优先级）

**用户是中文使用者。本 Skill 中出现的所有英文 Prompt 示例仅作为"结构骨架"参考，实际输出给用户或提交给 API 的 Prompt 默认使用中文。**

### 规则要点

1. **Prompt 主体用中文**：生图命令的 `--prompt` 参数内容默认中文书写
2. **技术锚点保留原样**（禁止翻译）：
   - 参考图编号：`image 1` / `image 2` / `image N`（模型机器语义标识，翻译会破坏引用机制）
   - HEX 色号：`#bf192e` / `#e8e4dd`
   - API 参数：`--aspect-ratio 3:4` / `--resolution 2k` / `--label`
   - 文件路径、品牌名、Logo 文字
   - 色彩硬约束词：`BACKGROUND ONLY` / `PRODUCT ONLY` / `DECORATIVE ELEMENTS ONLY` / `TEXT ONLY`
3. **标准化英文视觉术语**可用"中文 + 英文括注"形式保留锚点：
   - `低角度仰拍 (low-angle shot)`
   - `浅景深 (shallow depth of field)`
   - `自信直视镜头 (confident direct gaze)`
   - `棚拍干净中性光 (clean neutral studio lighting)`
4. **本文档下方所有英文示例**均作为结构参考；实际执行时请按上述规则翻译为中文版本再输出

### 示例对照

**✅ 正确（中文主体 + 技术锚点）：**
```
保留 image 1 中的塑身连体衣原样，不改变颜色、结构和面料。
使用 image 2 中的人物作为模特——精确保留她的面部特征、发色和身材比例。
低角度仰拍 (low-angle shot)，自信直视镜头 (confident direct gaze)，嘴角带轻柔浅笑。
9:16 画面比例，2K 分辨率。
```

**❌ 错误（过度翻译，破坏模型识别）：**
```
保留第一张图中的塑身衣原样。使用第二张图中的人物...
```
(模型看到"第一张图"不会关联 `image 1`，参考图引用机制失效)

### 例外情况

- 用户明确要求"用英文 Prompt"→ 尊重用户，切换为英文
- 复用 P 域存量英文案例 → 在【P 域复用摘要】中同时展示"原英文骨架"和"新中文 Prompt"供用户核对，实际提交中文版

## ⚡ 正向表述原则（核心写作规范）

**告诉模型"要什么"，而不是"不要什么"。** 当代主流生图模型（Nano Banana Pro / Gemini 3 Pro Image / Seedream 等）对正向指令的理解远比负向约束更稳定——模型需要先激活你禁止的视觉概念，再试图压制它，这反而会增加输出该概念的概率。

### 转换对照表

| ❌ 负向约束（旧写法） | ✅ 正向表述（新写法） |
|---|---|
| 不要改变背景 | 背景与 image N 保持一致 |
| 不要改变产品颜色 | 产品色精确保留 image 1 原色 |
| 禁止降饱和 | 色彩保持完全饱和 |
| 禁止偏色 | 色相与 image 1 精确一致 |
| 不要让人物脸部改变 | 人物面部特征与 image 2 精确一致 |
| 禁止文字出现在背景上 | 背景保持干净，文字仅出现在底部信息区 |
| 不要让头发遮挡肩部 | 发尾梳至一侧，双侧袖口完整可见 |
| 不要裁切太紧 | 取景范围保持腰线至肩线以上，产品躯干完整可见 |
| 不要让产品颜色向背景靠拢 | 产品色独立于背景色，保持其自身色调与饱和度 |
| 不要让产品消失在背景中 | 产品作为画面视觉焦点，与背景形成清晰对比 |

### 执行规则

1. **先写目标状态，再补边界条件**：`产品躯干占画面 55-65%，取景下边界落在髋线` 比 `不要裁切太紧也不要太松` 更精确
2. **用"保持""精确一致""完整可见"等正向动词**代替"禁止""不要""不得"
3. **边界情况用精确数值/锚点描述**：`取景下边界落在髋线`（精确）比 `不要裁太多`（模糊负向）更有效
4. **仅在负向表述是唯一准确表达方式时**（极少数情况），才允许保留——此时与正向约束配对使用

### 例外：`PRODUCT ONLY` / `BACKGROUND ONLY` 等色彩角色标记词

这些全大写标记词属于**技术约束信号**而非自然语言，模型将其视为硬分区指令，保留原样有助于强化色彩角色隔离，不属于本规范要替换的范畴。

## ⚡ 品牌名与抽象调性词的处理原则（核心写作规范）

**模型只能渲染它训练数据中见过的 token。品牌专有名词和抽象调性词的训练数据覆盖程度完全不同，必须区别对待。**

### 三类 token 的分层

| Token 类型 | 训练数据覆盖 | 对生图的作用 | 处理策略 |
|---|---|---|---|
| **具体视觉词汇**（哑光黑色面料 / 柔和漫射光 / 浅景深 / #1a1a1a） | ✅ 大量覆盖 | 直接映射为像素 | **主力使用** |
| **抽象调性词**（精致 / 自信 / 赋能 / 真实力量 / editorial / effortless） | ✅ 有覆盖（时尚/广告摄影语料） | 激活特定审美模式，概率性正向 | **保留使用**，正向收益大于噪声 |
| **品牌专有名词**（NEBILITY / Popilush / 自造品牌名） | ❌ 无覆盖 | 未知 token，对视觉无贡献，还可能拖累相邻有效描述 | **从描述句中移出** |

### 核心原则：品牌名从描述句里拆出去，调性词独立存在

**品牌名只应出现在两个位置**：
1. `image N = 品牌 Logo/品牌资产合成图`（参考图声明中，告诉模型去看图理解品牌）
2. `[LOGO REFERENCE]` 块（仅当用户明确要求渲染 Logo 时）

**品牌名不应出现在**：内容描述句、调性描述、影像风格描述中。

### 对照示例

**❌ 低效写法（品牌名拖累调性词）：**
```
[品牌规范 — NEBILITY]
NEBILITY 品牌调性：精致、赋能、自信。
体现 NEBILITY「女性真实力量」品牌核心。
```
→ 问题：`NEBILITY` 是未知 token，模型不理解它，还会吞噬相邻"精致、赋能、自信"的激活权重。

**✅ 高效写法（调性词独立 + 品牌交给参考图）：**
```
[BRAND TONE]
精致、赋能、自信的视觉调性。干净克制，editorial 质感，体现女性真实力量。
色彩与字体参考 image 4（品牌资产合成图）。
```
→ 调性词直接激活；品牌的具体颜色、字体由 image 4 视觉参考图承载。

### 抽象调性词的合法使用范围

以下抽象词在时尚/广告摄影训练数据中有明确语义激活，属于**应该保留**的有效调性词：

| 品牌调性 | 可用抽象词（中英皆可） |
|---|---|
| 赋能/自信 | 自信、赋能、真实力量、independent、empowering、confident |
| 精致/高端 | 精致、克制、editorial、premium、refined、polished |
| 轻松/真实 | effortless、casual、unposed、natural、candid |
| 运动/功能 | athletic、dynamic、performance、energetic |
| 柔软/亲密 | soft、intimate、warm、cozy |

> ⚠️ 不要堆砌：每张图选 2-3 个最契合的调性词即可，堆 10 个反而互相稀释。

### 真正无效的抽象描述（应该删除）

部分描述既不是具体视觉指令，也不是训练数据中常见的摄影语汇，属于纯噪声：

| ❌ 无效描述（应删除） | 原因 |
|---|---|
| 「品牌调性」「品牌核心」「品牌精神」 | 元描述，不是视觉词汇 |
| 「体现品牌 DNA」「诠释品牌哲学」 | 抽象哲学描述 |
| 「符合品牌定位」「契合品牌人群」 | 市场营销术语 |
| 品牌 slogan 的文字复述（除非需要渲染为画面文字） | 非视觉指令 |

## ⚡ 结构标签的语言策略

**结构标签用英文，内容描述用中文。**

Nano Banana / Gemini 对**英文结构标签**有更稳定的区块识别能力——训练数据中大量 prompt 使用英文方括号标签作为"硬分区信号"，模型能可靠地将其识别为配置块边界。中文方括号标签会被当作普通文本处理，区块隔离效果弱。

### 标签对照表

| ❌ 中文标签（区块识别弱） | ✅ 英文标签（区块识别强） |
|---|---|
| `[品牌规范]` | `[BRAND GUIDELINES]` |
| `[整体版式]` | `[OVERALL LAYOUT]` |
| `[场景参考]` | `[SCENE REFERENCE]` |
| `[模特与产品]` | `[MODEL & PRODUCT]` |
| `[场景互动]` | `[SCENE INTERACTION]` |
| `[镜头与焦点]` | `[CAMERA & FOCUS]` |
| `[姿势要素]` | `[POSE]` |
| `[文字区域]` | `[TEXT AREA]` |
| `[关键取景]` | `[FRAMING]` |
| `[配色方案]` | `[COLOR PALETTE]` |
| `[产品色独立声明]` | `[PRODUCT COLOR DISCIPLINE]` |

### 完整写法示例

```
[BRAND TONE]
精致、赋能、自信的视觉调性。高端电商摄影质感。
色彩与字体参考 image 4。

[SCENE REFERENCE — from image 1]
image 1 仅作为户外环境参考——地中海风格露台：木质躺椅、蓝白藤编椅、圆形边桌、遮荫大树、远处海天日落。
完整重现 image 1 的空间层次：前景地毯与家具，中景白色矮墙，远景柔和海天渐变。
光线与 image 1 一致：右上方暖色夕阳斜光，同样的光线方向打在模特皮肤与衣物上。

[MODEL & PRODUCT]
欧美白皙女性模特，约 25-30 岁，精致自然妆容，浅金棕色长发。
穿着黑色高弹连体塑身衣——收腰塑腹设计，面料哑光细腻。

[SCENE INTERACTION — T1 + T2]
模特右手轻搭画面左前侧蓝白藤编椅的椅背顶部……

[CAMERA & FOCUS]
低角度仰拍 (low-angle shot)，聚焦腰腹收紧区域与腿部线条。
浅景深 (shallow depth of field)，背景家具与远景柔和虚化。
```

### 规则总结

1. **结构标签全部用英文**（即使整句描述是中文）
2. **调性词、视觉描述、互动描述用中文**（你能看懂、能修改）
3. **技术锚点保持英文**（`image N` / HEX / 摄影术语括注）
4. **品牌名不进内容描述**，交给参考图（image N）承载

## 提示词基础结构

### 图片编辑公式（图生图必用）

```
【编辑动作】+ 【编辑对象】+ 【目标效果】+ 【保留内容】
```

**完整六要素公式**（复杂场景）：

```
[主体 Who/What] + [动作/状态 Doing] + [环境/背景 Where] +
[细节 Details] + [风格 Style] + [技术参数 Technical]
```

示例：
```
[主体] 一位 20 多岁的亚洲年轻女性，
[动作] 双手叉腰自信站立，
[环境] 身处明亮极简的精品店，白色墙面，
[细节] 温暖的午后阳光透过大落地窗洒入，
[风格] 专业商业时装摄影风格，
[参数] 9:16 比例，2K 分辨率，高品质。
人物外貌与 image 1 中的参考图保持一致，不做任何修改。
```

## 九大场景提示词模板

### 场景一：背景替换

**纯色背景（最常用）**
```
移除当前背景，替换为纯白色背景，
主体边缘保持干净，专业产品摄影风格，
画面中不出现文字或水印。
```

**场景背景（精准版）**
```
主体与 image 1 中完全一致，服装/发型/表情均保持原样。
仅替换背景为[新背景：例如，户外公园，柔和自然光，绿树成荫]。
主体边缘与新背景自然融合，主体身上的光线与新环境保持一致。
```

**常用室外背景替换词库**：
- 城市街道：`现代都市街道，暖色午后光线，城市背景虚化`
- 自然公园：`户外公园，柔和自然光，绿树，黄金时段光线`
- 海边沙滩：`热带海滩，金色夕阳，清澈碧绿海水，白色沙滩`
- 商业街区：`高端购物街区，明亮日光，干净的人行道`
- 咖啡馆外：`惬意的室外咖啡馆露台，欧式风格，温暖的环境光`

### 场景三：光线与氛围调整

**更换时间氛围**
```
将整体光线氛围调整为[黄金时段夕阳 / 蓝调黄昏暮色 / 明亮正午阳光]，
相应调整色温，主体构图保持不变。

**添加戏剧性光效**
```
添加戏剧性棚拍光效，左上方单主光，右侧柔和补光，
深色戏剧感背景，专业人像摄影风格。

**常用氛围词**：
- 柔和自然光：`柔和漫射自然光，均匀照明`

```

### 场景六：图片内嵌文字（营销素材）

> ⚠️ **重要**：本场景模板不得附加 `no text overlays` 负向约束。该约束仅适用于纯产品摄影场景，在图文版式场景会直接抑制文字生成。

> 最佳实践：先指定文字再配图——`"First render the text '[文字内容]', then create the surrounding visual."`

#### ⚠️ 决策点：用户是否提供了版式参考图？

| 情形 | 走哪条路 |
|---|---|
| **有版式参考图**（详情页/A+截图/竞品长图等含多个板块的参考图） | **走 6.0（默认首选）** —— 用三件套让参考图承担版式，Prompt 只做差异化 |
| 无版式参考图，需要从零构造版式 | 走 6A-6E —— 用文字描述版式结构 |

> 把 6A-6E 的"详细版式文字描述"用到有版式参考图的任务里 = 陷入**参考图过度描述陷阱**（见 `ref-image-must-include.mdc`）：文字会压过参考图，生成结果偏离参考图原貌。

#### 6.0 · 版式参考图复刻（有版式参考图时的默认写法）⭐

**适用**：用户提供了含多个板块的版式参考图（Amazon A+ 详情页、DTC 落地页、电商长图、多模块 Banner 组、九宫格样板等），要求按此版式复刻。

```

**✅ 正确写法（只指向图片 + 只写差异 + 只锁锚点）**：

### 场景七：人物一致性（系列图）

系列图生成时，每一张图的 prompt 都必须遵循以下模板：

```
使用高细节推理确保准确性：
[主体描述——不重述想保留的外貌细节],
[动作/姿势描述],
面部特征、发色和身材比例与 image 1 中的参考图保持一致，
产品与 image 1 中完全一致，外观保持原样，
[背景描述],
光线方向和质感与参考图保持一致，
[比例] [分辨率]。
```

**一致性核心原则**：
> 想保留什么元素，就引用参考图，**不要用文字重新描述它**。文字描述会让 AI 按自己理解重新生成，结果必然偏离原图。
>
> **产品保留是最高优先级**：无论是服装、鞋包、饰品、配件还是任何其他产品，只要目标是"保留参考图中的产品外观"，就**绝对不要在 Prompt 中描述产品特征**。任何描述（材质、颜色、结构、形状、细节等）都会引导 AI 按文字理解重新生成一个"符合描述"但不符合原图的版本。正确做法是传入多角度产品图，仅用 image 编号引用，让视觉信息完全替代文字描述。

| 元素 | 错误写法 | 正确写法 |
|------|---------|---------|
| 人物外貌 | 详细描述发色/五官 | `面部特征与 image 1 中的参考图保持一致` |
| 服装/产品（通用） | 描述产品的任何特征（材质/颜色/结构/细节，如：黑色压缩衬衫、金项链、皮质托特包等） | `产品与 image 1、2、3 中完全一致` — **不加任何额外描述，传多角度图让视觉说话** |
| 背景 | 重新描述背景细节 | `背景风格与 image 1 保持一致` |
| 光照 | 重新描述光线参数 | `光线方向和质感与参考图保持一致` |

### 场景八：参考库参考图的差异化引用

当使用参考库各域 `原始素材/` 中的图片（如 `参考库/F_摄影风格/原始素材/`、`参考库/E_平面与海报/原始素材/`）作为风格参考时，需要在 Prompt 中明确指定**从哪张图借鉴什么**，避免模型混淆不同图的角色。

**核心规则**：多张参考图按 `--images` 传入顺序，在 Prompt 中以 `image 1`、`image 2`、`image 3` 等称呼。

---

**基础模板（主体图 + 1 张风格参考图）**：

```
[主体元素] 与 image 1 中完全一致，外观保持原样。
借鉴 image 2 的[借鉴维度]——具体参考[具体描述借鉴的特征]。
[其他补充指令]。
```

---

**六种差异化引用方式**：

| 借鉴维度 | 写法示例 |
|----------|-------------|
| 色调 / 色彩分级 | `借鉴 image 2 的色调处理——使用同样的[暖金色调 / 低饱和哑光 / 冷蓝色调]质感` |
| 光线质感 | `借鉴 image 2 的光线质感——具体参考[柔和漫射自然光 / 戏剧单光源棚拍光 / 温暖逆光晕染]` |
| 构图方式 | `借鉴 image 2 的构图方式——[主体居中英雄构图 / 三分法放置 / 对称版式]` |
| 整体氛围 | `整体氛围与 image 2 保持一致——[简洁通透 / 情绪感沉郁 / 温暖生活感]` |
| 材质 / 质感 | `材质和质感渲染风格参考 image 2` |
| 风格类型 | `视觉风格借鉴 image 2——[平面插画 / 3D 渲染 / 编辑摄影]美学` |

---

**三图并用示例**（主体 + 构图参考 + 色调参考）：

```
使用高细节推理确保准确性：
产品与 image 1 中完全一致，外观保持原样。
借鉴 image 2 的构图版式——具体参考产品居中英雄构图，四周留有充裕留白。
借鉴 image 3 的色调和光线氛围——使用同样的柔和暖色调和漫射自然光质感。
专业商业摄影风格，高品质，2K 分辨率。
```

---

**参考库图片引用快速决策**：

```
问：这张生图需要参考参考库里的图吗？
├── 有明确的"风格对标"或"这种感觉"的需求 → YES，从参考库找对应领域图片
│   ├── 只借鉴1个维度（如只借色调）→ 2张图：主体图 + 1张审美参考
│   └── 借鉴2个以上维度        → 3张图：主体图 + 构图参考 + 色调/光线参考
└── 完全新建，无风格对标        → 只用主体图或直接文生图
```

### 场景九：多轮对话迭代编辑

> 此场景在 Cursor 对话中进行，无需重复上传图片，模型保持上下文。

```
第 1 轮：[上传图片] 将这张图调整为[建立基础效果]。
第 2 轮：在上一轮效果的基础上，进一步[细化风格]，其他内容保持不变。
第 3 轮：保留[上一轮效果]，同时增加[新的调整]。
第 4 轮：[最终微调]，其他内容保持不变。
```

#### 字体注入效果预期与局限

| 方面 | 预期效果 |
|------|---------|
| 字体风格方向 | AI 会明显参考渲染图的字体风格（衬线/无衬线/字重/气质） |
| 字母形状精确度 | 近似但非 100% 复刻 — AI 会"参考"而非"复制粘贴" |
| 模式 B 精确度 | 高于模式 A（因为文字区域已确定，替换更精准） |

## 🚨 已知陷阱与避坑指南

### 陷阱一：数量词触发九宫格拼图

**症状**：生成的单张图片变成 3×3 或 4×4 的缩略图拼贴，而不是一张完整画面。

**根本原因**：单张图的 Prompt 中出现了九宫格/系列/数量相关的描述，Gemini 将其理解为"在一张图里排列展示多张图"。

**正确做法：从 Prompt 中彻底移除所有九宫格相关描述。**

不需要加"不要拼图或九宫格"这类禁止语，这会浪费文本编码器空间，且本身也在描述"九宫格"概念。只需保证单张 Prompt 本身没有任何网格/系列/数量排列的描述即可。

```
❌ 会触发九宫格的写法（在单张 Prompt 中出现）：
"12 张图中保持相同的模特形象"
"生成系列图中全部 9 个姿势"
"以九宫格形式展示产品"

✅ 正确写法（直接描述这一张图的内容，不提数量或排列）：
"面部特征与 image 1 中的参考图保持一致"
```

**每次生成系列图前必查**：
- `□` Prompt 中是否出现"全部[数字]张图"？→ 必须删除
- `□` Prompt 中是否出现"[数字]张系列图""九宫格""网格""拼图"等词？→ 必须删除
- `□` 每张图的 Prompt 只描述这一张图的画面内容，与"一共要生成几张"无关

---

### 陷阱二：数量词触发多主体图

**症状**：本意一张产品图，结果出现多个产品排列。

```
❌ "展示产品的 3 个颜色款式"
✅ "仅展示[具体颜色]款，单个产品居中"

❌ "同时展示正面和背面视角"
✅ "仅展示正面视角"（拆分为两个独立 Prompt）
```

---

### 陷阱三：手和脸失真

```
在 Prompt 末尾加入：
"手部和面部比例准确，无变形，真实人体比例"
```

---

### 陷阱四：描述产品导致产品外观被重新生成

**症状**：生成图中产品与参考图明显不同——结构、材质、细节全部偏离，像是"按描述画出来的新产品"。

**原因**：Prompt 中对产品的任何文字描述都会让 AI 放弃参考图，转而按文字理解重新生成。描述越具体，偏离越严重。**这一规律适用于所有产品品类，不限于服装**。

```
❌ 触发重新生成（服装）：
"一件带硅胶垫的黑色压缩上衣：2 块胸垫、6 块腹肌面板"

❌ 触发重新生成（饰品）：
"一条带圆形钻石吊坠的金色细链项链"

❌ 触发重新生成（鞋）：
"白色皮革球鞋，厚底，侧边拉链细节"

❌ 触发重新生成（包）：
"一款结构感黑色皮质托特包，金色五金件，顶部双提手"

✅ 所有品类的统一安全写法：
"产品与 image 1、2、3 中完全一致，外观保持原样"
```

**核心规则**：产品有参考图 → 不描述产品 → 只引用编号。无论是服装、鞋、包、饰品、配件、3C 产品……一律如此。

## 常用正向保护指令

```
内容保护：  主体的服装与面部与 image 1 保持一致
           产品/角色/物体与 image 1 中完全一致，外观保持原样
           保留原始配色方案，仅改变风格

质量保证：  手部和面部比例准确，无变形
           主体清晰锐利，无模糊或失焦区域
           画面中不出现文字或水印

单图保证：  每张图的 Prompt 只描述该张图的内容，不出现数量词或排列描述
```
