# 非 AI 设计稿评审（Non-AI Design Reviewer）

## 概述

人工设计稿（非 AI 生成）的评审与优化建议输出。前身是能力模块 M6_设计评审与优化（V7.8 边界）。

## 边界（V7.8 评估三模块严格分工）

| 维度 | 本 Skill | `ai-image-logic-checker` | `ai-image-aesthetic-scorer` |
|---|---|---|---|
| 工作对象 | **非 AI 设计稿**（Figma/印刷/UI 截图/品牌手册） | AI 生图 Prompt + 结果 | AI 生图结果 |
| 工作时机 | 用户主动请求 | 生图 API 前后 | M16 放行后 |
| 工作目标 | 5 维设计规范评审 | 常识/逻辑 | 9 维审美 + 场景 |
| 激活方式 | **仅**人工 | 自动 | 自动 + 可人工 |

**硬性边界**：
- **不进 AI 生图链路**（V7.8 强约束）
- 用户上传 AI 生图结果说"评审一下" → 应路由到 `ai-image-aesthetic-scorer`
- 用户上传 Figma 设计稿说"评估这张" → 路由到本 Skill

## 适用对象（明确清单）

✅ Figma / Sketch / Adobe XD 设计稿  
✅ 印刷物料（折页 / 名片 / 海报印刷文件）  
✅ VI 手册 / 品牌规范文档  
✅ UI / APP 界面截图（人工设计稿）  
✅ 设计师作品集  
✅ 实拍 + PS 合成的物料

❌ AI 生图结果（用 ai-image-aesthetic-scorer）  
❌ AI Prompt 文本（用 ai-image-aesthetic-scorer 模式 C）

## 5 维评审框架

| 维度 | 检查项 |
|---|---|
| **视觉表现** | 美学水准、视觉冲击力、整体气质 |
| **版式构图** | 网格、对齐、留白、层级、视觉引导 |
| **品牌一致性** | 与品牌规范的色值/字体/Logo/语气是否一致 |
| **可读性** | 文字大小、对比度、行距、扫读流畅度 |
| **规范合规** | 印刷出血、色域、文件格式、平台尺寸/安全区 |

## 工作流程

### Step 1：识别物料类型

判断稿件属于哪一类（Figma/印刷/UI/手册），决定调用哪些参考依据：
- 印刷物料 → 加载印前规范、印刷工艺约束
- UI 设计稿 → 加载平台 HIG / Material Design
- 品牌物料 → 加载品牌规范主控

### Step 2：5 维评分

每维 1-5 星 + 依据 + 改进建议。

### Step 3：输出评审报告

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【设计评审报告】　non-ai-design-reviewer 输出
• 物料类型：[Figma 稿 / 印刷物料 / UI / 品牌手册]
• 当前品牌：[品牌名] / 无
─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─
• 视觉表现：★★★★ — [一句话依据]
• 版式构图：★★★★ — [依据]
• 品牌一致性：★★★ — [依据]
• 可读性：★★★★★ — [依据]
• 规范合规：★★★★ — [依据]
─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─
• 主要改进建议（≤ 5 条）：
  1. [具体改进，附理由]
  2. ...
• 印前/平台合规警告（如有）：
  - [必须修复的硬性问题]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 触发词识别

| 触发语 | 是否进入本 Skill |
|---|---|
| "评审这个 Figma 稿" | ✅ |
| "印刷前帮我检查一下" | ✅ |
| "看看 VI 手册有没有问题" | ✅ |
| "评审这张图"（上下文是 AI 生图结果） | ❌ → ai-image-aesthetic-scorer |
| "检查一下这张生图" | ❌ → ai-image-aesthetic-scorer |
| "和我品牌规范一致吗" + 非 AI 稿 | ✅ |
| "这个 Logo 设计稿" | ✅ |

## 与其他 Skill 的关系

| Skill | 关系 |
|---|---|
| `ai-image-logic-checker` | 互斥：本 Skill 不评 AI 生图 |
| `ai-image-aesthetic-scorer` | 互斥：物料类型决定路由 |
| `brand-spec-learner` | 协同：评审品牌物料时读品牌档案 |
| `brand-logo-designer` | 协同：评审 Logo 稿时参考其设计逻辑 |

## 注意事项

- 用户主动触发才激活；自动调度时主控中心**不会**路由本 Skill
- 印前评审有硬性警告（出血/色域/字体外发）时必须明确标红
- 详细 5 维评审标准见 `归档/v7.9/能力模块/M6_设计评审与优化.txt`
