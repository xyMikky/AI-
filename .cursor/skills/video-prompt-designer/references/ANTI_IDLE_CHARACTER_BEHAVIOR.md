# 人物去待机感行为设计指南

## 核心判断

AI 人物显假，常常不是因为五官不够真实，而是因为人物没有生活状态。真实的人不会像雕塑一样停在画面里，他通常正在被某件事推动，同时顺便做一个小动作。

视频人物提示词必须避免“人物待机感”：只站着、只看镜头、只摆姿势，但没有行为动机、情绪落脚点或微弱生命反应。

## 设计原则

### 1. 不只写动作，要写行为动机

动作本身不等于真实。真实感来自观众能感觉到人物正在经历某件事。

| 摆拍式动作 | 更真实的行为写法 |
|---|---|
| 端着咖啡 | 一边看向前方，一边无意识地转动咖啡杯 |
| 站在电梯里 | 轻轻抬眼看楼层按钮，手指搭在包带上 |
| 等车 | 低头回消息，听到声音后短暂抬头看向路边 |
| 拍照前站定 | 拍照前突然拨一下刘海，再自然放下手 |

### 2. 给人物安排“顺便做的小事”

小事不需要抢主线，但要让人物像是在生活里自然发生。

常用小事：

- 回消息、看通知、划手机屏幕
- 转杯子、咬吸管、整理杯套
- 抠手指、捏衣角、拉袖口、调整包带
- 拨刘海、把头发别到耳后
- 看电梯楼层、看红绿灯、看车流
- 走路时短暂分神，随后回到镜头或目标方向

### 3. 加入微弱生命反应

当画面需要安静、电影感或情绪停顿时，不要硬塞大动作，优先加入弱反应。

可用反应：

- 轻微眨眼
- 吸气带来的肩膀轻微起伏
- 眼神短暂偏移后回到焦点
- 嘴角很轻地变化
- 手指轻微摩挲物件
- 重心自然从一侧转到另一侧

### 4. 单人物公式

```
人物 + 正在做的小事 + 下意识反应 + 情绪落脚点
```

示例：

```
人物站在路边等车，低头回完一条消息，拇指停在屏幕上，听到远处车辆声后短暂抬眼看向街口，肩膀随着一次轻微吸气自然起伏，表情带着一点赶时间的分神感。
```

### 5. 多人物公式

多人物不要让每个人都同时表演。关键是“一人行动，另一人反应”。

```
人物 A 行动 + 人物 B 反应 + 两者之间的情绪关系
```

示例：

```
左侧人物低头快速整理购物袋，右侧人物没有同步表演，只是看着她的动作短暂笑了一下，然后伸手接过其中一个袋子，两人的反应自然错开，像真实生活里的临时配合。
```

## Prompt 写法模板

### 单人物

```
人物正在[场景中的小事]，同时出现一个下意识反应：[眨眼 / 吸气肩膀起伏 / 眼神短暂偏移 / 手指摩挲物件]。动作不是摆拍，而是像人物正在经历[情绪或情境]，节奏自然、克制、生活化。
```

### 多人物

```
人物 A 先做[具体行为]，人物 B 不同步表演，而是对这个行为产生[看向 / 停顿 / 微笑 / 接话 / 伸手配合]的反应。两人的动作有先后关系和情绪呼应，像真实互动，不像同时摆拍。
```

### 图生视频首帧

图生视频仍必须遵守首帧精简硬门：不要复述首帧里的外貌、服装、背景和光线，只写接下来发生的动作、反应和镜头变化。

```
人物先低头看了一眼手机，拇指轻划屏幕，随后像听到声音一样短暂抬眼看向画面外，轻轻眨眼并把头发别到耳后，肩膀随一次自然吸气微微起伏，镜头缓慢推进到中近景，动作克制、真实、生活化。
```

## 自检清单

生成视频人物 Prompt 前检查：

- 是否给人物安排了一个“正在做的小事”？
- 是否能看出这个动作背后的行为动机？
- 是否加入了至少一个微弱生命反应？
- 情绪是否有落脚点，而不是空泛地“看起来有感觉”？
- 多人物是否存在行动与反应的先后关系？
- 图生视频是否删除了首帧已承载的视觉复述？

## 禁止倾向

- 只写“人物看向镜头、自然微笑、站立不动”
- 把真实感完全押在五官、皮肤、发丝等外貌细节上
- 多人物同时做动作、同时看镜头、同时表演
- 为了“有动作”加入没有动机的大幅肢体表演
- 安静情绪镜头中让人物完全静止无反应
