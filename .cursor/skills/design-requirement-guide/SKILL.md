# 设计需求引导（Design Requirement Guide）

## 概述

所有设计任务的**第零步**——把用户的口头诉求（无论清晰或模糊）转化为可执行的需求简报。前身是能力模块 M13_设计需求引导（V2.1）。

**核心机制**：强制 Plan 模式 + 强制 AskQuestion 选项卡 + 三档自适应深度。

## 使用场景

凡是设计类任务（品牌/Logo/海报/详情页/AI 生图/视频/UI/包装）都自动激活，**无论用户需求多明确**。

### 豁免条件（跳过本 Skill）

| 条件 | 原因 |
|---|---|
| 学习任务（reference-library-learner / brand-spec-learner） | 流程固定 |
| 系统维护（ai-persona-architect / logic-chain-diagnostic） | 执行性任务 |
| 可视化任务（design-relation-visualizer） | 辅助工具 |
| 同会话同任务的迭代修改 | 已确认过 |
| 用户明确说"跳过引导/直接开始/直接生成" | 尊重明确意图 |

## 工作流程

### Step 1：调用 SwitchMode → Plan

切换到 Plan 模式（只读）。这是核心动作。

### Step 2：判断引导深度

| 信号 | 深度 | 轮次 |
|---|---|---|
| "帮我把背景换成白色" / "做成 1:1 主图" | Express | 1 |
| "帮我设计一张海报" / "做个详情页" | Standard | 2-4 |
| "不知道怎么开始" / "帮我想想" | Deep | 5-8 |

### Step 3：使用 AskQuestion Q&A 选项卡（强制）

**禁止纯文字追问**。每个待确认项必须通过 `AskQuestion` 工具呈现为选项卡：
- 每道题至少 2 个选项
- 每道题最后必须含"其他（请说明）"兜底
- 多选项可勾选 `allow_multiple: true`
- 单次最多 3 道题（避免轰炸）

### Step 4：输出对应产物

**Express 输出【快速需求确认】**：
```
━━━━━━━━━━━━━━━━━━━━━━━━━
【快速需求确认】design-requirement-guide Express
• 任务理解：[一句话]
• 当前品牌：[品牌名] / 无
• 关键参数：
  - [参数]：[值]
• 待确认项：[≤ 3 项]
以上理解是否正确？确认后立即开始执行。
━━━━━━━━━━━━━━━━━━━━━━━━━
```

**Standard / Deep 输出【设计任务简报 D】**：
- 任务类型 / 核心目标 / 受众 / 风格方向 / 约束 / 推荐激活 Skill / 备注

### Step 5：用户确认 → 切回 Agent 模式

调用 `SwitchMode → Agent`，主控中心读取简报继续路由。

## 三档判断快速表

| 用户输入特征 | 深度 |
|---|---|
| 具体图片 + 单一操作（换背景/加文字） | Express |
| 明确设计类型 + 受众 + 风格方向 | Express 或 Standard |
| 有设计类型但缺受众/风格/目的 | Standard |
| 只有模糊感受或宽泛诉求 | Deep |
| 用户主动说"先聊聊" / "帮我理清" | Deep |
| ≥ 3 个维度同时变更 | Standard 至少 |

## 与其他 Skill 的关系

| Skill | 关系 |
|---|---|
| 主控中心 | 上游：调度本 Skill |
| 所有业务 Skill（除豁免清单） | 下游：读取本 Skill 输出 |
| `market-platform-analyzer` | 后续：场景信息进入 K1-K4 分析 |
| `ai-image-prompt-builder` 等执行类 | 后续：根据简报路由 |

## 输入输出

**输入**：用户原始诉求（任意清晰度）

**输出**：
- 【快速需求确认】（Express 模式）或
- 【设计任务简报 D】（Standard/Deep 模式）
- Plan 模式 → Agent 模式切换信号

## 注意事项

- 即使是明确的小任务（如"换白色背景"）也要用 Express 走一次，**确保不会"确认的同时就动手"**
- Plan 模式下无法调用执行工具，强制阻挡过早动手
- Q&A 选项卡是用户体验关键，禁止退化为纯文字
- 同会话内对同一任务的迭代修改可豁免（避免烦扰）
