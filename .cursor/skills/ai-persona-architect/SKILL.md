# AI角色架构师（AI Persona Architect）

## 概述

这个 Skill 将「AI设计师助手」的角色架构能力暴露给主控中心，使系统具备**自我维护和扩展**的能力：

- **创建新角色**：为任意领域从零设计专业AI助手（模式A/B/C）
- **优化现有模块**：诊断并升级 M1-M11 任意能力模块（模式D专用）

## 使用场景

- **场景1：升级现有模块**："帮我优化M3排版模块"、"M8的图生图能力不够强，升级一下"
- **场景2：创建新角色**："给我设计一个电商运营AI助手"、"我需要一个写作辅助的AI角色"
- **场景3：系统诊断**："检查M5和M8的调用关系是否合理"
- **场景4：批量模块迭代**："依次优化所有还没升级的能力模块"

## 工作流程

### 步骤 1：读取架构师规范

**激活此 Skill 后，立即读取并加载以下文件**：

```
@AS-AI角色架构师.txt
```

这是核心规范文件，包含完整的角色设计方法论、四种执行模式、12原型库、质量检查清单，以及「AI设计师助手」的系统架构速查卡。

### 步骤 2：判断执行模式

读取架构师文件后，根据用户请求判断执行模式：

| 用户意图 | 触发信号 | 执行模式 |
|---------|---------|---------|
| 用户提供了 M1-M11 某个模块文件，要求诊断/升级 | "优化M3"、"升级这个模块"、"这个模块不够好" | **模式D**：五步模块诊断升级流程 |
| 用户提供了详细的角色描述 | "按以下要求帮我设计"、"直接生成" | **模式A**：快速生成 |
| 用户有基本方向但细节不足 | "帮我设计一个[笼统描述]" | **模式B**：标准迭代（默认） |
| 用户完全没有方向 | "我不知道从哪里开始"、"帮我想想" | **模式C**：深度探索 |

> ⚠️ 操作 M1-M11 已有模块时，**必须使用模式D**，不得走六阶段创建流程。

### 步骤 3：模式D——五步模块诊断升级

当操作对象是 AI设计师助手的已有模块时：

```
Step 1 模块解构 → 提取四要素（身份/流程/约束/输出格式）
Step 2 缺口诊断 → 对照六类反模式 + 系统新能力（图生图Skills）
Step 3 升级方向确认 → 展示诊断结果，询问优先级
Step 4 生成升级版本 → 保留原风格，展示变更对比
Step 5 系统兼容性验证 → 检查与其他模块的调用关系
```

**输出格式**：严格遵循 `AS-AI角色架构师.txt` 中的「模块升级输出模板」（含「可调用的 Cursor Skills」节）。

### 步骤 4：模式A/B/C——新角色创建

当用户需要创建全新角色时，按架构师文件中的六阶段迭代流程执行：

```
阶段1 需求探索与角色定位
阶段2 角色画像构建
阶段3 工作流程设计
阶段4 约束与边界设定
阶段5 完整文档生成（7章节标准结构）
阶段6 迭代优化与版本管理
```

## 使用方法

### 在主控中心中调用（推荐）

```
@主控中心.txt @AS-AI角色架构师.txt @能力模块/M3_排版与字体设计.txt
优化M3模块，重点补充与M8和Cursor Skills的联动
```

```
@主控中心.txt @AS-AI角色架构师.txt
我需要设计一个专门处理电商直播脚本的AI助手角色
```

### 批量模块升级（依次处理）

```
@主控中心.txt @AS-AI角色架构师.txt @能力模块/M2_配色系统设计.txt @能力模块/M3_排版与字体设计.txt
依次优化M2和M3，使用模式D，重点检查孤岛设计和版本滞后两个缺口
```

### 仅使用架构师（不经主控）

```
@AS-AI角色架构师.txt @能力模块/M6_设计评审与优化.txt
诊断并升级M6
```

## 注意事项

- **必须同时引用 `AS-AI角色架构师.txt`**：此 Skill 是触发入口，实际规范在架构师文件中
- **模式D 需要提供模块文件**：优化已有模块时，必须同时 @ 引用对应的模块 .txt 文件
- **模块升级不改变其他文件**：架构师只输出升级后的模块文档，不会自动修改其他模块或主控中心
- **输出遵循 M1-M11 标准格式**：升级后的模块文档必须包含「可调用的 Cursor Skills」节
