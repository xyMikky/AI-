# 市场平台场景分析（Market Platform Analyzer）

## 概述

将"任务描述"翻译为"在这个场景下什么设计才是正确的"。本 Skill 不关心"好不好看"，只关心"在这个场景下能不能实现目标"。是 AI 生图 / 视频 / 详情页等业务 Skill 的前置场景约束源。

**前身**：能力模块 M11_场景分析专家（V2.0 起改名为 market-platform-analyzer）。

## 核心理念：美 vs 对，两个独立维度

```
           高审美匹配
                ↑
  漂亮但无效  |  理想产出
─────────────────────────→ 高场景正确性
  两头落空   |  实用正确
                ↓
           低审美匹配
```

冲突时优先级：
- 品牌形象任务 → 审美匹配 ≥ 场景正确性
- 转化率任务 → 场景正确性 > 审美匹配
- 多数情况 → 在场景正确的约束内提升审美

## 使用场景

- 跨市场设计："为东南亚做主图""日本市场的电商版式"
- 跨平台设计："Shopee 的产品图规范""Amazon A+ 详情页"
- 跨受众："下沉市场""新中产""Z世代"
- 跨内容类型："主图""详情页""品牌故事""社媒图"

## 工作流程

### Step 1：场景四维度解析

从任务中提取：
- **K1 目标市场**：中国/日本/东南亚/欧美 + 具体国家
- **K2 使用平台**：淘宝/天猫/小红书/抖音/独立站/Amazon/Shopee + 页面类型
- **K3 目标受众**：年龄/消费层级/兴趣圈层/性别
- **K4 内容类型**：主图/详情页/Banner/品牌故事/社媒/包装

信息不足时主动追问 ≤ 2 个关键缺口。

### Step 2：场景知识调取

按优先级加载场景知识库：
1. K4 内容类型规范（最直接）
2. K2 平台规范（技术 + 用户行为）
3. K1 地域市场规范（文化适配）
4. K3 受众画像规范（视觉语言）

文件路径：`场景知识库/K[N]_*/`

### Step 3：输出【场景约束卡 A】

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【场景约束卡 A】　market-platform-analyzer 输出
• 市场 / 平台：K1_[地域] + K2_[平台]
• 目标受众：K3 [受众特征]
• 内容类型：K4 [内容形式]
─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─
• 硬性规范：[平台/市场强制要求，如白底/尺寸/文字安全区]
• 文化禁忌：[该市场的视觉禁区]
• 受众偏好：[3-5 个视觉偏好关键词]
• 竞争基调：[主流视觉风格，作差异化参考]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 与其他 Skill 的关系

| Skill | 关系 |
|---|---|
| `ai-image-prompt-builder` | 下游消费者：必须读取卡 A 才能构造场景化 Prompt |
| `video-prompt-designer` | 下游消费者：场景适配视频 |
| `detail-page-designer` | 下游消费者：详情页强场景依赖 |
| `ai-image-aesthetic-scorer` | 评估时与场景正确性双轴并行 |
| `design-requirement-guide` | 上游：M13 引导收集到的场景信息进入本 Skill |

## 输入输出

**输入**：
- 用户任务描述
- 可选：已加载的场景知识库文件

**输出**：
- 标准化的【场景约束卡 A】（供下游 Skill 直接读取）
- 缺失维度时输出追问问题

## 注意事项

- 无场景信息时降级为"纯审美模式"，跳过本 Skill
- K1/K2/K3/K4 不是必须全部有；常见任务覆盖 K2+K4 即可
- 场景知识库未覆盖的市场/平台 → 用通识 + 候选品牌竞品分析兜底
