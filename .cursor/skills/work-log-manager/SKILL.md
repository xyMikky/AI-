# 工作日志管理引擎（Work Log Manager）

## 概述

使用者**工作日志**的归档与状态萃取引擎。是「个人中心」的行为记录入口——profile-spec-learner 让智能体知道"使用者是谁"，本 Skill 让智能体知道"使用者在做什么、做过什么、接下来要做什么"。

通过持续归档工作日志并滚动刷新 `工作摘要.txt`，智能体能够掌握使用者**入职至今的工作脉络 + 当前实时状态 + 待办**，从而在任意任务中给出贴合上下文的辅助。

## 边界（互斥声明）

| 任务 | 归属 |
|---|---|
| 工作日志归档 / 工作摘要刷新 / 待办状态 | ✅ 本 Skill |
| 岗位 / 偏好 / 技能 / 协作关系等基础信息 | ❌ → `profile-spec-learner` |
| 品牌 / 参考库素材学习 | ❌ → brand-spec-learner / reference-library-learner |

## 触发条件

| 用户语义 | 模式 |
|---|---|
| "记录工作日志" / "归档日志" | 单条 / 待归档区扫描 |
| "导入历史日志" / 一次性粘贴多条 | 批量历史导入 |
| 对话中直接描述今天做了什么 + "记一下" | 对话直录 |
| "更新工作摘要" / "我现在的工作状态" | 仅刷新/读取摘要 |
| "更新我的岗位 / 偏好" | ❌ → profile-spec-learner |

**M13（design-requirement-guide）豁免**：信息录入任务，流程固定。

## ⚠️ 技术约束（中文路径）

```powershell
$root = $PWD.Path
$user = "默认用户"
$logDir = Join-Path $root "个人中心\$user\工作日志"
$pending = Join-Path $logDir "00_待归档"
$year    = Join-Path $logDir ("按年\" + (Get-Date).Year)

# 列待归档文件（禁用 Get-ChildItem）
[System.IO.Directory]::GetFiles($pending)

# 创建年份目录
[System.IO.Directory]::CreateDirectory($year)

# 写归档日志（UTF-8）
[System.IO.File]::WriteAllText($target, $content, [System.Text.Encoding]::UTF8)
```

## 三种工作模式

### 模式 A：单条归档
用户把单条日志放入 `00_待归档/`（或对话提供一条），归档为一个文件。

### 模式 B：批量历史导入
用户一次性粘贴入职至今的多条日志（用 `---` 分隔），逐条拆分归档。归档后 `工作摘要.txt` 应覆盖"入职至今的项目脉络"。

### 模式 C：对话直录
用户在对话中口述当天工作，无需先建文件，本 Skill 直接结构化归档。

## 核心流程

```
触发（记录 / 归档 / 导入工作日志）
  ↓
Step 1：确定使用者 + 校验目录存在
  └─ 不存在 → 提示先「初始化个人档案」（profile-spec-learner）
  ↓
Step 2：获取日志来源
  ├─ 扫描 `工作日志/00_待归档/`
  └─ 或读取对话粘贴内容（单条 / 多条 --- 分隔）
  ↓
Step 3：逐条解析为标准格式（提取日期/主题/内容/产出/待办/标签）
  ↓
Step 4：按 `按年/YYYY/` 归档
  → 命名 `YYYY-MM-DD-主题.txt`（同日多条追加 -01 / -02）
  ↓
Step 5：更新 `日志索引.txt`（按时间 + 按主题 + 统计三处）
  ↓
Step 6：刷新 `工作摘要.txt`（核心产物，见下）
  ↓
Step 7：清理 `00_待归档/` 已处理文件（保留使用说明.txt）
  ↓
输出【工作日志归档摘要】
```

## 单条日志标准格式（归档到 按年/YYYY/）

```
# [YYYY-MM-DD] [主题]

> 编号：W-XXX | 归档日期：YYYY-MM-DD | 标签：#xx #xx

## 今日工作内容
- [做了什么]

## 产出 / 进展
- [可交付物 / 进度变化]

## 遇到的问题 / 卡点
- [如有]

## 下一步 / 待办
- [后续动作]

## 关联
- 项目：[所属项目]
- 协作：[涉及谁]
```

## `工作摘要.txt` 刷新规则（核心产物 · always-load）

`工作摘要.txt` 是每次会话强制加载的工作状态层（见 `personal-hub-loading.mdc`）。每次归档后**必须重算并覆盖**，让智能体掌握实时状态：

| 章节 | 内容来源 |
|---|---|
| 当前工作状态 | 最新若干条日志的进行中 / 等待中事项 |
| 近期已完成（2-4 周） | 近期日志的产出汇总 |
| 待办与下一步 | 各日志"下一步"汇总 + 按 P0/P1 排优先级 |
| 长期目标 / 季度重点 | 跨日志识别的主线项目 |
| 风险与卡点 | 日志中反复出现或未解决的卡点 |
| AI 助手须知 | 最不能耽误的事 / 用户压力 / 建议主动提醒项 |

刷新原则：
- 待办状态随新日志变化（完成的从待办移到"近期已完成"）
- 头部 `> 最后更新 | 基于日志数：N` 同步更新
- 批量导入后，摘要要能讲清"入职至今 → 当前"的项目脉络

## 工作日志目录结构

```
个人中心/[使用者]/工作日志/
├── 工作摘要.txt              ← 核心产物，always-load，滚动刷新
├── 日志索引.txt              ← 全部日志索引（时间/主题/统计）
├── 00_待归档/                ← 新日志粘贴区
│   └── 使用说明.txt          ← 保留，不清理
└── 按年/
    └── YYYY/                 ← 归档原始日志
        └── YYYY-MM-DD-主题.txt
```

## ❌ 禁止行为

- 编造用户未提供的工作内容 / 进度 / 待办
- 归档后不刷新 `工作摘要.txt`（= 智能体仍不知道最新状态，破坏核心价值）
- 把工作日志写进 `基础信息/` 或品牌规范目录
- 清理 `00_待归档/使用说明.txt`
- 用 `Get-ChildItem` 列举中文路径

## ✅ 正确行为

- 三种模式自适应（单条 / 批量 / 对话直录）
- 每条日志结构化为标准格式归档到 `按年/YYYY/`
- 归档后必刷新 `日志索引.txt` + `工作摘要.txt`
- 批量导入时让摘要覆盖"入职至今"完整脉络
- 完成后输出【工作日志归档摘要】

## 与其他 Skill / 规则的关系

| 对象 | 关系 |
|---|---|
| `profile-spec-learner` | 互补：本 Skill 管"在做什么"，后者管"是谁" |
| `personal-hub-loading.mdc` | 配套：`工作摘要.txt` 是其 Always-Load 内容；归档后刷新才生效 |
| `个人档案.txt` | 间接：摘要要点可同步进个人档案的"工作上下文速查" |
