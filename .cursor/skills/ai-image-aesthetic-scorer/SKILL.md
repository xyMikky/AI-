# AI 生图审美评估（AI Image Aesthetic Scorer）

## 概述

对 AI 生图结果做"好不好看 + 是否场景正确"的双轴评估。前身是能力模块 M9_审美评估器。

## 边界（V7.8 评估三模块严格分工）

| 维度 | 本 Skill | `ai-image-logic-checker` | `non-ai-design-reviewer` |
|---|---|---|---|
| 工作对象 | AI 生图结果图 | AI 生图 Prompt + 结果图 | 非 AI 设计稿 |
| 工作时机 | M16 放行后 | 生图 API 前后 | 用户主动请求 |
| 工作目标 | 9 维审美 + 场景适配 | 常识/逻辑 | 5 维设计规范 |
| 激活方式 | 自动 + 可人工 | 自动，不接受人工 | 仅人工 |

**硬性边界**：
- 不替代 ai-image-logic-checker（评分前必须先过 M16）
- 不评非 AI 稿（Figma/印刷品 → non-ai-design-reviewer）
- 不修改 Prompt（迭代指令交给 ai-image-prompt-builder 执行）

## 三种工作模式

| 模式 | 触发 | 输入 | 输出 |
|---|---|---|---|
| A 默认 | M16 通过后自动 | 结果图 + 评估维度 | 【评估报告卡 C】 |
| B 单独 | "评估这张图""和我风格匹配吗""选哪个" | 用户上传图 | 【评估报告卡 C】|
| C 方案预审 | 对 Prompt 文本预审 | M8 输出的 Prompt | 风险评分 + 改写建议 |

## 9 维审美工艺评分

构图 / 配色 / 光线 / 材质质感 / 主体清晰度 / 留白与层级 / 视觉叙事 / 细节工艺 / 整体气质

每维 1-5 星，附一句话依据。

## 工作流程

### 模式 A：生图后自动评估
1. 接收 ai-image-logic-checker 放行信号
2. 读取结果图 + 评估维度（来自 ai-image-prompt-builder）
3. 加载相关参考档案（F/B/Z 等域）
4. 输出【评估报告卡 C】
5. 主控中心读卡：通过→反馈进化询问；需迭代→指令交回 ai-image-prompt-builder

### 模式 B：用户主动评估
1. 用户上传图 + 说明
2. 加载对应参考档案
3. 输出双轴评分

### 模式 C：Prompt 预审
1. 读 ai-image-prompt-builder 输出的 Prompt 文本（未渲染）
2. 预测潜在风险（如版式过度描述、色彩牵引句、姿势缺七要素）
3. 输出改写建议，**不评尚未存在的图**

## 反馈进化触发

满足全部条件时主动询问用户："是否将本案例写入参考档案 P 域？"
- 审美匹配度 ≥ 4 星
- 含新发现（参考档案中尚未记录的设计规律）
- 用户未明确拒绝写入

## 与其他 Skill 的关系

| Skill | 关系 |
|---|---|
| `ai-image-logic-checker` | 上游：必先过 M16 才能进入本 Skill |
| `ai-image-prompt-builder` | 下游：本 Skill 的迭代指令交回它执行 |
| `non-ai-design-reviewer` | 互斥：仅评 AI 图，非 AI 稿不进 |
| `market-platform-analyzer` | 协同：场景正确性轴的判据来自卡 A |
| 参考库 F/B/Z 域 | 评估时按需加载 |

## 输入输出

**输入**：AI 生图结果图 + 任务上下文 + 参考档案

**输出**：【评估报告卡 C】或预审风险报告

## 注意事项

- 9 维评分基于"普世审美 + 用户参考档案"双重锚点
- 场景正确性轴在纯审美任务中填 N/A
- 详细 9 维评分细则见 `归档/v7.9/能力模块/M9_审美评估器.txt`
