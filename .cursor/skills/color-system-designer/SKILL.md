# 配色系统设计（Color System Designer）

## 概述

配色系统设计的双模式 Skill。前身是能力模块 M2_配色系统设计。

## 双模式触发契约（来自主控中心 V7.9 矩阵 4.1/4.2）

| 模式 | 触发信号 | 工作流 | 主产出 |
|---|---|---|---|
| **独立模式** | "配色方案/色彩系统/色卡/调色板/品牌色规范/深色模式" — 主任务是色彩决策 | Step 1-4 | 完整色彩系统文档（主/辅/点缀/中性/功能/WCAG）|
| **Step 0 协同** | 被 ai-image-prompt-builder / detail-page-designer 声明协同 | Step 0 轻量 | 【配色决策卡 D】供生图 |

**两种模式互斥，由调控中心决策声明，不自仲裁**。

## 模式仲裁原则

1. 用户语义优先："方案/规范/系统/板/手册" → 独立；"生成/样图/出一张" → 协同
2. 主任务归属：色彩决策本身 → 独立；服务生图 → 协同
3. 明确意图："只做配色不做海报" → 独立；"做海报" → 默认协同
4. 疑义：调控中心走 design-requirement-guide Express 1 轮确认

## 独立模式（Step 1-4）

```
Step 1：色彩调研（行业惯例/竞品/品牌调性）
Step 2：色彩策略（情绪映射/对比关系/温度/饱和度）
Step 3：色彩规范（主色/辅色/点缀/中性/功能色 HEX 值 + 占比建议）
Step 4：可访问性 WCAG（对比度 4.5:1+/3:1+ 检查）
```

## Step 0 协同模式（轻量交接卡）

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【配色决策卡 D】　color-system-designer 协同输出
• 色彩来源：[品牌色 / B_配色档案 / 用户指定 / 场景推导]
• 画面配色方案：
  - 背景色：#XXXXXX — 约 X% 面积
  - 主体产品色：[保留产品原色 / #XXXXXX]
  - 强调色：#XXXXXX — 约 X% 面积
  - 文字色：#XXXXXX — 主 / #XXXXXX — 辅
  - 点缀色：#XXXXXX — ≤5% 面积（可选）
• 色彩关系：单色深浅 / 类比 / 互补 / 分割互补
• 决策理由：[一句话]
• 品牌一致性：✅ / ⚠️ / N/A
• 下游衔接：→ ⑥.6 色卡生成门 + 卡 E 对齐文字色值
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 独立 → 协同桥接（关键）

用户先调独立模式产规范，随后说"生成样图"时**不重跑 Step 0**：
- 从 Step 3 主色/辅色/点缀/文字色 HEX 抽取 → 卡 D 的 `color_palette`
- 从 Step 3 占比建议抽取 → 卡 D 的 `ratio`
- 从 Step 2 情绪映射抽取 → 卡 D 的 `mood_keywords`

## 与其他 Skill 的关系

| Skill | 关系 |
|---|---|
| `color-palette-generator` | 工具：本 Skill 决策颜色，前者渲染色卡 PNG |
| `ai-image-prompt-builder` | 下游消费者（Step 0 模式）|
| `detail-page-designer` | 下游消费者（Step 0 模式）|
| `typography-designer` | 协同：卡 E 需读卡 D 对齐文字色值 |
| `brand-logo-designer` | 协同：完整品牌设计时 |
| `brand-spec-learner` | 数据源：品牌色板加载 |

## 输入输出

**输入**：行业 + 调性 + 竞品 + 已有品牌色（可选）+ 调控决策（声明独立 or 协同）

**输出**：
- 独立：完整色彩系统文档
- 协同：【配色决策卡 D】

## 注意事项

- 模块**不做二次仲裁**：调控决策写明协同 → Step 0；未写 → 独立
- 调控漏标时通过 Step 0 开头的【模式自检】反向提示
- 详细 Step 1-4 框架见 `归档/v7.9/能力模块/M2_配色系统设计.txt`
