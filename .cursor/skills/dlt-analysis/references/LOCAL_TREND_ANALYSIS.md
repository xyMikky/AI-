# 局部趋势分析方法论（15 期窗口）

> 与 50 期全局走势不同，**局部趋势**聚焦最近 **15 期**短窗口，捕捉号码在图上的「运动轨迹」——斜向连线、纵向热柱、多线汇聚——并外推下一期落点。对应参考图中**绿色箭头**标注的逻辑。

---

## 1. 与全局分析的区别

| 维度 | 全局走势（50 期） | 局部趋势（15 期） |
|------|-------------------|-------------------|
| 窗口 | 50 期 | **15 期（默认）** |
| 核心 | 冷热、遗漏、区间均衡 | **连线运动方向** |
| 图表 | 基本走势图 | **带绿色趋势线 + 虚线延长** |
| 预测权重 | 回归均值、超期冷号 | **趋势线延长落点、汇聚枢纽** |

两者互补：全局看「欠账」，局部看「动势」。

---

## 2. 读图坐标（15 期局部图）

- **行 = 期号**，上旧下新，最底行为基准期；最下方还有一行虚线框「预测」行。
- **列 = 号码**：前区 01–35（红球），后区 01–12（蓝球）。
- **绿色实线箭头**：已识别的有效趋势线（≥3 期串联）。
- **绿色虚线箭头**：从最新点延长到「预测」行的落点列。
- **橙色空心圈**：多线汇聚的枢纽落点（2 条及以上趋势线指向同一号码）。

---

## 3. 四类局部形态

### 3.1 右下斜带 ↘

连续多期号码**逐期增大**（如 15→17→19），步长大致稳定。延长线指向更大号码区。

### 3.2 左下斜带 ↙

连续多期号码**逐期减小**（如 28→25→22）。延长线指向更小号码区。

### 3.3 纵向热柱 ↓

**同一列**（同号码）在 3 期及以上反复出现，表示该号在短窗内极热，可能延续或透支回调。

### 3.4 汇聚枢纽 ✕

两条及以上趋势线（来自不同起点/方向）的**虚线延长**落在同一号码列 → 该号为局部枢纽，下一期权重上调。

---

## 4. 程序识别规则（`analyze_local_trends`）

| 规则 | 参数 |
|------|------|
| 最小链长 | ≥ **3 期**（3 个红/蓝球） |
| 最大单步 | 相邻行间号码差 ≤ **6** |
| 步长一致性 | 链内各步与首步偏差 ≤ **3** |
| 去重 | 保留更长、更靠近最新期的链，最多 **12** 条/区 |
| 延长落点 | 末点号码 + 链内平均步长，钳制在合法范围 |
| 汇聚 | ≥ **2** 条线落点相同 → 枢纽 |
| 纵向热柱 | 前区同号 ≥ **3** 次；后区同号 ≥ **2** 次 |

---

## 5. Agent 工作流

```
① 运行 local_trend_chart.py（默认 -n 15）
   ↓
② 读取 stdout 中文报告 + JSON（链、落点、候选）
   ↓
③ Read 生成的 PNG → 目视验证绿线/虚线是否与报告一致
   ↓
④ 结合 stats_report.py（50 期冷热）交叉过滤
   ↓
⑤ 输出局部预测：优先「汇聚枢纽 + 高得分候选」，**再经程序四 ±N 偏差带扩展**，不单押纯落点
```

### 定号优先级

1. **汇聚枢纽号**（多线指向 + 高 score）
2. **偏差带扩展候选**（落点 ±N，N 取自 `deviation_profile.json`）
3. **最近 8 期内结束的趋势线**延长落点（作为带中心，非最终号）
4. **纵向热柱**（延续 or 回调需结合遗漏）
5. 与 50 期超期冷号交叉 → 取交集加权

> 纯落点精确率约 20%，**第 2 步偏差带不可跳过**。

### 结构校验（同全局）

- **固定定号 7+3**：前区 7 码、后区 3 码（脚本 `local_trend_chart.py` 报告底部「★ 本期推荐定号」已自动产出）
- 前区 7 码：三区间覆盖 ≥2 区、大小/奇偶不极端、和值合理
- 不可 7 码全来自同一斜带落点 ±1 邻域

## 7. 常见误区

- ❌ 用 50 期大图做局部连线（噪声太多，应用 15 期专用图）
- ❌ 两点连线当趋势（至少 3 期）
- ❌ 忽略汇聚枢纽（多线共振比单线更值得关注）
- ❌ 不做全局交叉（局部动势 + 全局欠账才完整）
